## 决策结论

`park`

`recovery_gate_d12_a3_f75_strong_extreme` 不进入实盘、不作为当前 hard5 回撤深度优化候选，也不继续按原参数族做邻域扩展。

## 这个节点是什么

这个节点处理的是：能否在当前实盘链路 `A2-slope004 + B3 gate + tiered-v2 + hard5` 上，用组合权益回撤状态做第二层仓位预算，降低最大回撤深度。

QKYP 测试的是一个谨慎版本：只在 B3/tiered-v2 的 `strong/extreme` 风险状态里，且组合回撤达到 12% 并连续 3 个状态更新日没有恢复时，才把风险仓再乘一个非线性系数，floor 为 75%。

## 相比上一个节点改变了什么

- FUFW 只确认“有足够样本可以研究”，没有确认“回撤越深越应降仓”。
- QKYP formal 给出进一步结论：这个 `strong/extreme` 窄门控太保守，触发太少，收益损伤虽小但没有换来足够最大回撤改善。
- 当前实盘默认开关保持不变，不加入 QKYP 主变体。

## 支持证据

- QKYP 主变体四段 formal 全部完成，8 个回测格 `exit_status=0`。

## 反对证据

- 主变体实际应用只有 3 次，且受保护样本数为 `0`。
- 2022_2023 final 从 hard5 的 `131214.63` 降到 `130805.90`；2024 final 从 `167005.22` 降到 `166625.94`。

## 边界

这个决策不能说明“权益回撤二级预算完全无效”。它只说明 QKYP 这个 `strong/extreme + 12% + 3日未恢复 + floor75` 主变体不通过，不应实盘化。

若继续研究，必须重新预注册，并优先解释旧 B5 正控为什么在 2022/2024 有保护，同时怎样过滤 2021/2022 的强反弹误伤。

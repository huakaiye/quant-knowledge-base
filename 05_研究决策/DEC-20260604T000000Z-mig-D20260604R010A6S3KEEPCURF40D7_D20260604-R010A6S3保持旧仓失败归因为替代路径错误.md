# D20260604-R010A6S3 保持旧仓失败归因为替代路径错误

## 决策摘要

R010-A6S3 完成对 2024 年 9 次 `filter_all_weak -> 保持旧仓` 触发的只读归因。

结论：A6 保持旧仓路径归档为只读诊断，不再推进交易化。

核心原因不是 A2 与 B0 的目标分歧。9 次触发中：

```text
B0原始目标 == 原A2目标
```

因此 A6 拦截的并不是一个被 A2 改坏的目标，而是原本就会交易的目标。保持旧仓的收益劣化来自替代路径选择错误。

## 当前判断

```text
archive_keep_current_antichurn_keep_readonly
```

允许：

- 保留 A6 默认关闭实现、配置、shadow 日志和归因脚本。
- 把 `filter_all_weak` 作为“追新环境偏弱”的诊断标签。
- 继续做只读研究，寻找旧仓以外的替代目标选择标签。

不允许：

- 不把 `filter_all_weak` 直接映射为保持旧仓。
- 不跑 A6 保持旧仓年度 OOS。
- 不扩大到 A1/A0 或 all_actions。
- 不进入 MiniQMT、实盘开关或实盘 shadow。

## 关键解释

A5 的只读代理链条是：

```text
filter_all_weak 命中新目标后续收益差
```

但真实交易问题是：

```text
如果不买新目标，应该买什么？
```

S3 说明旧仓不是稳定答案。它在 10月30日窗口非常有效，但在 5月、7月、8月、11月窗口造成更大损失，最终抵消优势。

## 下一步

R010-A 主线不再沿“保持旧仓防抖”推进。

下一轮若继续提高双池轮动收益，应预注册为只读替代目标研究：

```text
当A2换仓且filter_all_weak提示追新环境偏弱时，
比较原A2目标、旧仓、风险池Top2-Top5候选、短延迟确认路径，
寻找能事前分离正负触发簇的标签。
```

只有出现跨窗口可解释、非后验的标签后，才允许重新讨论默认关闭交易化。

# D20260604-R010A5 A2过滤层稀薄换仓进入默认关闭Shadow

## 决策摘要

R010-A5 发现 A2-slope004 的真实换仓路径存在一个稳定的坏换仓标签：当 Top5 过滤层很薄时，追新换仓在四个分段中均呈负的 episode 代理收益。

因此 R010-A 主线从已归档的 R010-A4S5 Top1 替换路径，转向 R010-A6 过滤层稀薄换仓防抖。但 A6 只能先做默认关闭 / shadow 验证，不直接进入实盘候选。

## 是什么

本决策关注的是 A2 已经发生真实换仓时的质量控制。

核心现象：

```text
当 r010b3_top5_filter_positive_count <= 1，
或者 r010b3_filter_all_weak == true，
A2 的追新换仓 episode 在四段样本中都呈负贡献。
```

这不是防守降仓规则，也不是小市值方向；它是 ETF 双池进攻轮动内部的换仓防抖候选。

## 证据

宽规则：

```text
rule_switch_thin_filter_positive
= switch episode AND r010b3_top5_filter_positive_count <= 1
```

| 指标 | 数值 |
| --- | ---: |
| 命中 episode | 248 |
| 分段稳定性 | 4/4 为正代理改善 |
| 代理改善 | +139.6381pp |
| 平均 episode 收益 | -0.5631% |
| 捕捉亏损 / 误伤盈利 | 1.6330 |

严格规则：

```text
rule_switch_filter_all_weak
= switch episode AND r010b3_filter_all_weak == true
```

| 指标 | 数值 |
| --- | ---: |
| 命中 episode | 125 |
| 分段稳定性 | 4/4 为正代理改善 |
| 代理改善 | +119.0403pp |
| 平均 episode 收益 | -0.9523% |
| 捕捉亏损 / 误伤盈利 | 2.2714 |

严格规则的绝对代理改善略低，但纯度更高、误伤更小，适合作为 R010-A6 的优先验证候选。

## 当前判断

```text
continue_to_filter_thin_antichurn_shadow_default_off
```

允许：

- 进入 R010-A6 预注册。
- 新增默认关闭 / shadow 观测代码或只读反事实脚本。
- 优先验证严格规则 `filter_all_weak`，再比较宽规则 `top5_filter_positive_count<=1`。
- 观察动作应限定为“追新换仓防抖”，例如保持旧仓、回退 B0 目标或仅记录 shadow target。

不允许：

- 不直接改 MiniQMT 实盘开关。
- 不把该标签解释为防守降仓、清仓或回撤覆盖规则。
- 不恢复 R010-A4S5 的 Top1 替换交易化路径。
- 不接入小市值或多策略切换。

## 竞争性解释

| 解释 | 如何处理 |
| --- | --- |
| 只是 episode 代理，不是真实反事实收益 | A6 必须做 shadow 或默认关闭回测，不把代理收益当净值收益 |
| 宽规则误伤正收益换仓较多 | A6 先测严格版，再测宽版 |
| `filter_all_weak` 可能是风险环境而非选标问题 | A6 不做降仓防守，只比较换仓路径 |
| 旧仓继续持有未必优于新目标 | A6 必须同时记录旧仓保持、B0目标、原A2目标三条路径 |

## 下一步

1. 建立 R010-A6 预注册实验记录。
2. 实现只读 shadow 或默认关闭开关，不影响当前 A2-slope004 实盘默认行为。
3. 验收口径优先看四段收益、误伤正换仓、交易次数、最大回撤，不只看代理改善。

# D20260602-R020 B3 覆盖率缺口复盘完成

## 决策摘要

R020 覆盖率缺口复盘完成。结论不是继续新增一个统一触发器，而是拆分为两类：2021-05 是仍未解决的触发缺口，暂时 `park`；2025-04 不是触发漏报，而是 cap80 强度不足，已经由 R010-B4 tiered-v2 方向覆盖。

## 是什么

本节点只读取既有研究产物，不启动回测、不访问数据库、不修改实盘。复盘对象是路线快照中指定的关键缺口：

- 2021-05-13：R010-B0 最深回撤日为 A0 / `normal_attack`。
- 2025-04-08：R010-B0 最深回撤日为 A1 / `short_risk_watch`。
- 2024-09-11：作为已被 B3 gate 与 tiered-v2 改善的对照样本。

新增平台脚本：

```text
scripts/research/analyze_r020_b3_coverage_gap_review.py
```

输出目录：

```text
results/v2/research/R020/b3_coverage_gap_review/
```

## 相比上个节点

上一节点 [[D20260602-A2-slope004实盘默认行为回归通过]] 确认实盘默认边界没有回归。本节点回到研究路线中的缺口解释：不再把 2021-05 和 2025-04 都视为“B3 漏报”，而是分别判断触发问题和强度问题。

这会改变后续实验顺序：2025-04 不再需要新触发器；2021-05 不能用 `b3_filter_any` 或宽风险信号直接补，因为这些信号过宽且后续平均收益仍为正。

## 当前判断

```text
r020_gap_review_done_2021_park_2025_strength_layer
```

当前可做：

- 将 2025-04 归入 R010-B4 tiered-v2 的强度分层证据。
- 将 2021-05 保留为未解释触发缺口，不急于交易化。
- 若继续研究 2021，只允许预注册慢性失速、波动扩张、集中度衰退等只读规格。

当前不能做：

- 不能因为 2021-05 有 `b3_filter_any` 或 `b3_broad_exhaustion` 就直接新增强防守动作。
- 不能把 2025-04 当作新的漏报触发器样本。
- 不能恢复 R026 硬过滤或宽信号仓位 gate。

## 当时问题

路线快照中仍有一项待复盘任务：2021-05-13、2025-04-08 等最深回撤日为什么未进入强防守。若不拆清楚，会诱导后续把“触发偏晚”和“触发后仓位不够低”混成一个后验阈值组合。

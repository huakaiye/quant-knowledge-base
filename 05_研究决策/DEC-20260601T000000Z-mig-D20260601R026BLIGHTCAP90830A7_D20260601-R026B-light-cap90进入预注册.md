# D20260601-R026B-light-cap90 进入预注册

## 决策摘要

R026 的 B层权益防守信号不直接交易化，但允许进入一个更窄的默认关闭离线预注册实验：只在 B3 gate 未 active 时测试轻量 cap90。

## 是什么

R026-B-light-cap90 是一个受限防守补充实验。它只使用 `signal_B_equity_defense`，尝试在 B3 gate 没有进入 active cap80 的日期，把风险仓轻量限制到 90%，持续 1 个交易日，目标是复核 R026 是否能补足 B3 gate 未覆盖的少数风险日。

它不是：

- R026 直接交易开关。
- R026 清仓或 cap80。
- `signal_any_defense_mem30` 宽信号。
- 旧版 B/C 记忆 15 日切 cap80。
- MiniQMT 实盘开启指令。

## 相比上个节点

上一节点：[[D20260601-R026-strict-marker-only]]

上个节点把 R026 降级为低覆盖风险复核标记，禁止作为主防守 gate。本节点没有推翻该结论，只是在严格边界下允许一个最小权限实验：用最干净但稀疏的 B层信号，验证它是否能做轻量 cap90 复核。

关键变化：

- 从“只记录 marker”增加到“可预注册离线实验”。
- 从宽信号或 B/C 记忆，收窄为 `signal_B_equity_defense`。
- 从 cap80/清仓，收窄为 cap90。
- 从交易上线，限定为默认关闭离线回测。

## 当前判断

```text
observe_pre_backtest
```

当前可做什么：

- 新增默认关闭的策略参数，让平台能表达 R026-B-light-cap90。
- 做配置审计，证明没有混入旧 R026 B/C mem15 cap80。
- 先跑 dry-run 和 2024 单段，结果通过后再四段验证。

当前不能做什么：

- 不能开启实盘 R026。
- 不能用旧 `r010b3_cap90_r026_bc_mem15_switch_*.json` 代替本实验。
- 不能让 R026 覆盖 B3 gate active cap80。
- 不能使用 `signal_any_defense_mem30` 或任意宽信号。

## 当时问题

B3 gate 已成为当前最强防守候选，R026 增量审计显示宽信号噪音大，但 `signal_B_equity_defense` 未来 10 日 -5% 下探精度高于基准，且不覆盖 gate blocked raw。问题是：这个低覆盖信号是否有足够边际价值，能在不破坏 B3 gate 的前提下补充少数漏报风险。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| B层信号较干净 | `signal_B_equity_defense` 未来10日 -5% 精度 35.09%，高于全样本基准 23.87% |
| 不重新放大 gate blocked raw | `signal_B_equity_defense` 对 gate blocked raw 覆盖为 0 |
| 权限更低 | cap90 比 cap80 或清仓更不容易破坏趋势收益 |
| B3 gate 已有主防守 | 本实验只是补充，不承担主撤离职责 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 覆盖太低 | `signal_B_equity_defense` 只有 57 个信号日，non raw future dd5 覆盖仅 5.56% |
| 交易价值不确定 | cap90 可能太轻，无法改变深回撤 |
| 样本脆弱 | 低频信号可能被少数极端日支配 |
| 当前代码不能直接表达 | 现有 R026 switch 固定使用 B/C 记忆并在 B3 全弱下切 cap80 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| B层信号只是风险解释，不是交易信号 | 看 cap90 触发日是否改善后续 5/10 日路径 |
| B层信号只在某几年有效 | 四段分段验收，不看单年 |
| 改善来自少数极端日 | 去极端日和事件级审计 |
| cap90 不足以改变结果 | 对比触发日局部收益、窗口谷底边际和最大回撤 |

## 决策

```text
observe_pre_backtest
```

决策边界：

- 只允许离线预注册和默认关闭实现。
- 只有通过样本外、成本、消融和未来函数审计，才可讨论更高状态。
- 若旧代码无法表达规则，先修参数和审计，不得用近似配置冒充。

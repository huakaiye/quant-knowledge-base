## 决策结论

采用 V2 研究库架构：方向、实验、决策、文献、因子数据灵感、台账、术语和入口驾驶舱分层管理。

## 这个节点是什么

这是新研究库的系统级起点，决定后续所有研究资产如何组织。

## 相比旧库改变了什么

- 不复用旧库目录结构。
- 不使用全局递增编号。
- 每个核心资产有唯一 ID。
- 实验记录必须包含实验前预测和新手总结。
- 策略代码、配置和结果继续放平台，研究库只登记路径和解释。

## 支持证据

- 用户明确指出旧库架构无法满足长期团队研究。
- 多 Agent 和 GitHub 协作需要避免编号冲突。
- 策略研究需要方向、模块、实验和决策之间可追溯。

## 反对证据

- 新架构初期需要更多文档纪律。
- 旧库迁移需要额外工作。

## 边界

本决策只确定研究库结构，不对任何策略有效性做判断。

## 后续动作

- 按新结构迁移旧库有价值资产。
- 后续新增实验必须使用新模板和台账。

# D20260602-R010B5 组合回撤预算动态缩放代理审计完成

## 决策摘要

将“随组合回撤加深机械降仓、回弹补仓”的想法立为 R010-B5 离线研究分支；首轮净值层代理审计显示收益拖累过大，当前结论为 `revise_proxy_return_drag_no_live`，不进入实盘。

## 是什么

R010-B5 是组合层风控研究。它不是新的 ETF 选股逻辑，也不是 B3 gate 的替代品，而是在 B3-gate-tiered-v2 已生成净值路径后，用组合净值相对高点的回撤来决定下一交易日风险仓上限。

本轮只做净值层代理：读取 R010-B4 的 tiered-v2 拼接净值，用前一交易日可得的回撤状态决定下一交易日仓位缩放，缩掉部分按零收益现金处理。它不访问数据库、不跑正式回测、不修改实盘。

## 相比上个节点

上一节点：[[D20260602-R010B4-tiered-v2实盘启用]]

相比 tiered-v2 实盘启用，本节点不是实盘增强，也不是继续调 B3/tiered 参数，而是新增一个独立的离线研究分支，用于验证组合净值路径本身能否作为二级风险预算。

## 当前判断

```text
revise_proxy_return_drag_no_live
```

当前可做：

- 保留 R010-B5 为离线研究分支。
- 使用代理审计结果作为反证，修订下一轮问题。
- 若继续，只允许预注册“B3/tiered 条件化组合回撤预算”。

当前不能做：

- 不能修改 MiniQMT 实盘脚本。
- 不能直接上线组合回撤预算。
- 不能继续做全周期机械回撤阈值扫描来找最优参数。
- 不能把净值层代理结果当作正式策略路径回测。

## 当时问题

用户提出：是否可以通过实时监控，当回撤到一定程度时机械降仓，回弹时补仓。研究问题是：组合净值回撤是否能作为比 B3/tiered 更高层的仓位预算，进一步控制回撤。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| endo_v1 回撤改善 | 最大回撤从 -27.13% 改为 -21.35%，改善 +5.77pp |
| 2024 主回撤窗口 | endo_v1 在 2024-06-24 至 2024-09-11 将窗口回撤从 -25.72% 降至 -14.69% |
| 2021-05 漏报窗口 | endo_v1 / shadow_ref_v1 将窗口回撤从 -7.18% 降至 -3.52% |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 收益保留率不足 | 6 个版本中，最高收益保留率仅 85.87%，低于 95% 门槛 |
| 动作日质量低 | 动作日胜率约 44.69%-47.06%，远低于 75% 门槛 |
| 错过反弹多 | endo_v1 错过反弹日 705 天；极保守 endo_v4 仍有 169 天 |
| 低仓锁定 | 自身回撤驱动版本动作天数最高 1289 天，说明机械规则会长期留在低仓 |
| 2025-04 改善有限 | 极保守版本在 2025-04 没有改善，较强版本改善也有限且压低修复 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 阈值太敏感而非机制失败 | 已加入 10% 后才降仓、15% 后才轻降仓的保守边界，仍未达到 promote |
| 代理模型过保守 | 后续只有在条件化规则通过预注册后，才做真实路径回测区分 |
| B3/tiered 已经吸收主要风险 | 下一轮若继续，应只测试 B3/tiered 风险窗口内的二级预算，而非全周期覆盖 |

## 决策

```text
revise_proxy_return_drag_no_live
```

决策边界：

- R010-B5 保留为观察中研究分支。
- 当前字面版本“全周期机械按组合回撤降仓、回弹补仓”不进入实盘。
- 下一步若继续，必须改为条件化：只在 B3 gate active、tiered strong/extreme 或其他已预注册风险窗口内启用。

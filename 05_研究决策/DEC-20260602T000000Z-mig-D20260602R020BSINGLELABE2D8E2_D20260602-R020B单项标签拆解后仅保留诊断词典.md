# D20260602-R020B 单项标签拆解后仅保留诊断词典

## 决策摘要

R020B 完成 R020A 所有单项只读标签拆解。结论是：`leader_surface_strength_decay` 可保留为诊断标签，但没有交易型风险优势；其余标签只能状态描述、背景解释、晚确认或剔除。R020A/R020B 不再作为触发器路线推进。

## 是什么

本节点读取 R020A 的 2021 首轮审计与独立窗口外推结果，把所有单项标签按外推覆盖、T-10 提前性和后 20 日风险 edge 分层。

平台脚本：

```text
scripts/research/analyze_r020b_single_label_decomposition.py
```

输出目录：

```text
results/v2/research/R020/single_label_decomposition/
```

本节点不启动回测、不访问数据库、不改实盘配置。

## 相比上个节点

上一节点 [[D20260602-R020A-只读标签外推审计后降为案例标签]] 已经证明综合标签不能作为通用触发器。本节点进一步拆单项标签，防止后续绕开综合标签、继续从 `equity_drawdown_pressure`、`leader_surface_strength_decay`、`breadth_decay_watch` 等字段里后验拼阈值。

## 当前判断

```text
keep_diagnostic_labels_no_trigger_research
```

允许：

- `leader_surface_strength_decay` 保留为诊断词条。
- `equity_drawdown_pressure` 与 `equity_slow_decay` 用于描述策略权益压力。
- `breadth_decay_watch` 用作窗口早期广度收缩背景解释。
- `volatility_expansion_watch` 用作谷底附近晚确认解释。

禁止：

- 不把任何 R020A/R020B 标签并入 B3 gate。
- 不用这些标签新增 cap70、cap60 或清仓动作。
- 不修改 MiniQMT 默认开关或 shadow-only 边界。
- 不继续围绕这组标签调阈值。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| `leader_surface_strength_decay` 独立窗口命中 3/3、T-10 命中 2/3 | 具备诊断价值 |
| `leader_surface_strength_decay` 后20日平均回撤差 +0.21pp、亏损率差 -4.01pp | 没有交易型风险优势 |
| `equity_drawdown_pressure` 和 `equity_slow_decay` 均漏掉 2022-10 | 只能描述状态，不能做通用触发 |
| `breadth_decay_watch` 独立窗口命中 3/3，但 T-10 仅 1/3 | 时机不稳定 |
| `volatility_expansion_watch` T-20/T-10 均为 0/3 | 太晚，偏确认 |
| `breadth_gate_near_miss` T-10 为 0/3 | 从触发器研究中剔除 |

## 反对证据

| 证据 | 说明 |

| `breadth_decay_watch` T-20 命中 3/3 | 可作窗口早期背景，但 T-10 不稳定 |

## 决策

```text
keep_diagnostic_labels_no_trigger_research
```

- R020A/R020B 标签体系到此收口。
- 研究库可引用这些标签做归因，但不能作为仓位触发器继续推进。
- 若后续研究 A0 漏报，必须另立新的研究记录和证伪规则。

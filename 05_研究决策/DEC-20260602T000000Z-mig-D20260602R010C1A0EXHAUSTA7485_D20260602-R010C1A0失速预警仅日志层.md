# D20260602-R010C1 A0失速预警仅日志层

## 决策摘要

R010C1 的 A0 假强势和失速预警标签只保留为日志层，不再作为独立防守交易动作推进。

## 是什么

R010C1 研究的是：当 ETF 双池原状态机给出 A0 正常进攻时，是否存在“表面趋势仍整齐，但风险资产池已经失速”的可见迹象。

当前保留两个只记录字段：

```text
r010c1_a0_exhaustion_watch
r010c1_a0_exhaustion_strong
```

其中 `watch` 用于记录“趋势形态仍整齐但池内收益偏弱”，`strong` 用于记录“Top1 中期收益不足且短斜率转弱”。它们只解释 A0 假强势，不直接降仓、不切 511880。

## 相比上个节点

上一条刚收口的是 A2 二级观察门控：它只能做收益修补，不承担深回撤防守。本节点处理 A0 分支的类似问题：C1 标签能解释部分 A0 漏防，但深回撤覆盖不足，因此不能升级成独立防守开关。

## 当前判断

```text
observe_log_only_no_trade_action
```

当前可做：

- 在日志中保留 `r010c1_*` 字段，用于解释 A0 正常进攻下的假强势。
- 在回撤归因时引用 C1 是否提前出现。
- 将 C1 作为后续 A1/A2 已触发风险层的辅助解释字段。

当前不能做：

- 不能把 C1 单独变成 cap90、cap80、清仓或切 511880 规则。
- 不能把 C1 并入 B3 gate 或 tiered-v2。
- 不能把 C1 接入 MiniQMT 默认开关或 shadow-only 观察。
- 不能用 C1 解释 2024 主回撤失败，因为 2024 深回撤覆盖为 0。

## 当时问题

R010-B0 长周期显示，部分深回撤日仍被原状态机标为 A0 正常进攻。R010C1 尝试从 A0 日中识别假强势和失速预警，判断是否能作为新的启动触发器。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| A0 样本 391 天，A0 漏防事件 131 天 | A0 分支确实存在漏防问题 |
| `Top1中期收益不足且短斜率转弱` 命中率 72.73% | 高精度低召回，可解释部分强失速 |
| `趋势形态仍整齐但池内收益偏弱` 触发 80 天，A0 漏防命中率 48.75% | 说明假强势标签有观察价值 |
| 2021-05 的 -15% 回撤窗口被 C1 覆盖 | C1 对部分 A0 触发缺口有局部解释能力 |
| 运行时 smoke 已证明 `r010c1_*` 字段可落盘 | 具备日志层可实施性 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| -15% 深回撤窗口任一 C1 提前覆盖仅 19.05% | 不足以作为通用防守触发器 |
| 2024 年 -10% 与 -15% 回撤窗口提前覆盖均为 0 | 无法解释或修复最关键 2024 主回撤 |
| `watch` 类规则防御 ETF 相对旧仓并不稳定 | 直接切防御可能牺牲收益 |
| 多数深回撤触底日处在 A1 或 A2，而非 A0 | 主问题更像已有风险层动作设计不合适，而不是 A0 缺少标签 |
| 2025-2026 分段多条规则命中率较低 | 固定阈值存在市场结构漂移 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| C1 只是 A0 个别样本的归因标签 | 继续看独立窗口和未来日志，不做交易动作 |
| 深回撤主问题在 A1/A2 动作强度，而不是 A0 识别 | 对比 R010-D 与 B3 gate/tiered-v2 的覆盖和收益代价 |
| C1 可作为已有风险层的加强条件 | 若未来研究，必须只在 A1/A2/B3 已触发时做消融，不能单独触发仓位动作 |

## 决策

```text
observe_log_only_no_trade_action
```

决策边界：

- C1 继续作为 A0 假强势日志标签保留。
- 不进入独立交易动作消融。
- 不改变当前主候选 A2-slope004+B3-gate-median-or-breadth-cap80。
- 后续若再次研究 A0 漏报，必须另立新假设，例如“持仓自身失血”或“换仓路径破坏”，不能复用 C1 固定阈值直接调参。

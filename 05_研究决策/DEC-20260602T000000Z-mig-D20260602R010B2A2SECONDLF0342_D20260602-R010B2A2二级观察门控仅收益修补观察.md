# D20260602-R010B2 A2二级观察门控仅收益修补观察

## 决策摘要

A2 二级观察门控保留为收益修补观察标签，但不再作为回撤防守、B3 gate 增强、tiered-v2 分层或实盘观察开关推进。

## 是什么

A2-slope004 会在旧仓仍有相对优势时保留旧仓，减少追涨换仓。A2 二级观察门控关注的是：当 A2 已经决定保留旧仓时，是否能识别少数“旧仓其实该换、继续保留会损失收益”的日子。

本节点复核的核心候选是：

```text
candidate_old_edge_or_filter_all_weak
```

它不是降仓规则，而是收益修补候选：若未来交易化，合理动作也应是“命中时回退 B0 目标”，不是 cap70、cap60 或清仓。

## 相比上个节点

上一节点 [[D20260602-A2-slope004实盘默认行为回归通过]] 确认 A2-slope004 默认开启行为没有被 B3 gate 或 tiered-v2 默认关闭映射误伤。

本节点不改变 A2 默认行为，而是复核 A2 分支里剩余的二级观察门控：它是否值得继续占用后续研究队列。结果显示，它有收益修补价值，但不具备防守价值。

## 当前判断

```text
observe_profit_patch_label_no_defense
```

当前可做：

- 保留 `candidate_old_edge_or_filter_all_weak` 为收益修补观察标签。
- 在研究解释中引用 2020-07 和 2024-10 的旧仓误保留样本。
- 如果未来明确要验证收益修补，可另写预注册，只做默认关闭短窗口测试。

当前不能做：

- 不能把该标签并入 B3 gate。
- 不能把该标签用于 tiered-v2 cap70/cap60 分层。
- 不能作为深回撤防守、清仓或降仓触发。
- 不能映射到 MiniQMT 默认开关或 shadow-only 观察。

## 当时问题

前序联合审计发现，`candidate_old_edge_or_filter_all_weak` 命中 11 条，捕捉 5 条负贡献、误伤 1 条正贡献，代理改善约 +15.79pp。问题是：这是否说明它能升级为交易规则，还是只是少数样本上的收益修补标签。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| 最强候选代理改善 +15.79pp | `candidate_old_edge_or_filter_all_weak` 仍是收益修补方向最强候选 |
| 捕捉负贡献 5 条，误伤正贡献 1 条 | 说明它确实识别了一部分旧仓误保留日 |
| 2020-07 事件簇代理改善 +8.10pp | 能解释 A2 在 2020-07 的旧仓保留问题 |
| 2024-10 事件簇代理改善 +7.35pp | 能解释 B3 filter 全弱背景下的局部误保留 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 有意义改善只有 2 个事件簇 | 样本稳定性不足，不能直接交易化 |
| 2022-2023 无命中 | 不是跨周期持续发生的通用现象 |
| 2025-20260519 命中 2 条但收益影响约 0 | 外推贡献很弱 |
| -10%、-15%、-20% 深回撤窗口 T-10/T+3 覆盖均为 0 | 不能承担深回撤防守职责 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 只是 2020-07 与 2024-10 的事件后验归因 | 未来若交易化，必须先做默认关闭短窗口预注册，且不得复用这些事件调参 |
| 成本和切换路径会抵消代理改善 | 短窗口验证必须加入交易笔数、换手和实际 B0 回退路径 |
| 它其实是 B3 filter 全弱的衍生标签 | 已验证 `candidate_filter_all_weak` 只有 2024 有意义，不能替代 B3 gate |

## 决策

```text
observe_profit_patch_label_no_defense
```

决策边界：

- A2 二级观察门控从防守候选队列移出。
- 不改变当前主候选：A2-slope004+B3-gate-median-or-breadth-cap80。
- 不改变 tiered-v2 默认关闭和 shadow-only 边界。
- 后续若要研究，只能作为收益修补短窗口实验另立预注册；验收重点是收益保留、误伤反弹和交易成本，不是最大回撤。

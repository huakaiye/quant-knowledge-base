# D20260602-R010B4 tiered-v2实盘启用

## 决策摘要

用户已明确要求启用 B3-gate-[[tiered-v2（分层强度防守）]]。本节点把 R010-B4 tiered-v2 从”默认关闭/[[Shadow-only（影子观察）]]”切换为 MiniQMT 实盘默认启用，并完成静态编译与启用审计。

当前判断为 `live_enabled_user_authorized`：可以按 v19 实盘文件运行，但首个交易日必须复核启动日志和 B3 gate/tiered 分层日志；若日志或执行状态不符合预期，按本节点备份文件回滚。

## 是什么

实盘文件：

```text
E:\xtquant\策略\ETF双池动量轮动_MiniQMT.py
```

启用后的默认组合：

```text
ENABLE_A2_SLOPE004=1
ENABLE_B3_GATE_MOB_CAP80=1
ENABLE_B3_GATE_TIERED_V2=1
ENABLE_B3_GATE_TIERED_V2_SHADOW=0
```

运行逻辑仍是：

```text
[[A2-slope004]] 进攻增强
-> [[B3 gate]] 过滤全弱噪音
-> 普通 gate cap80
-> strong risk cap70
-> extreme risk cap60
```

strong risk：

```text
cum_days >= 10
AND median_ret <= 0
AND top5_exhausted >= 4
```

extreme risk：

```text
strong risk
AND (median_ret <= -0.015 OR ma20_breadth <= 0.25)
AND top5_exhausted >= 4
```

## 相比上个节点

上一节点：[[D20260601-R010B4-tiered-v2影子观察方案建立]]

上一节点只允许 shadow-only：记录 `tiered_candidate_cap`，但实际 `tiered_effective_cap` 保持 80%。本节点因用户明确授权，将 `ENABLE_B3_GATE_MOB_CAP80` 和 `ENABLE_B3_GATE_TIERED_V2` 的默认值改为开启，strong/extreme 触发时会真实改变风险仓上限。

## 当前判断

```text
live_enabled_user_authorized
```

允许：
- MiniQMT v19 默认启用 B3 gate 与 tiered-v2。
- B3 gate active 后按 tiered-v2 输出的 `effective_cap` 调整风险仓上限。
- 保留 `ENABLE_B3_GATE_TIERED_V2_SHADOW=0`，避免交易启用时混淆影子观察口径。

禁止：
- 修改 tiered-v2 阈值或新增后验 guardrail。
- 引入 R026、A1/A3/A4 或其他已证伪动作作为实盘交易开关。
- 在未检查启动日志前假定实盘运行状态已经符合预期。
- 用旧“默认关闭映射审计”判断启用后的状态；启用后应使用新的 live-enabled 审计脚本。

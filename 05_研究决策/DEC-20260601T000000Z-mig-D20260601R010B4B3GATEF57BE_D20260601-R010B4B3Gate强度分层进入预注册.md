# D20260601-R010B4 B3Gate强度分层进入预注册

## 决策摘要

R010-B4 已从 2024 smoke test 推进到四段分段验证。结论从 `promote_to_split_validation` 修订为 `revise_after_split_validation`：固定 cap70/cap60 证明更强防守在部分冲击窗口有效，但不满足动作日胜率纪律；tiered-v1 没有通过四段稳定性验证，下一步只能修订规则，不能上线。

## 是什么

本节点研究的是：当 B3 gate 已经确认 ETF 双池处于较干净的弱势风险状态后，是否可以把风险仓从 cap80 进一步降到 cap70 或 cap60。

已完成三类默认关闭实验：

- 固定 cap70：所有 B3 gate active 日从 cap80 改为 cap70。
- 固定 cap60：所有 B3 gate active 日从 cap80 改为 cap60。
- tiered-v1：普通 gate 仍 cap80；`median_ret<=-0.01` 且 `ma20_breadth<=0.30` 用 cap70；`median_ret<=-0.015` 且 `ma20_breadth<=0.25` 用 cap60。

不作用的边界：

- 不修改 MiniQMT 默认开关。
- 不引入 R026 仓位动作。
- 不替代当前 `A2-slope004 + B3-gate-median-or-breadth-cap80` 主候选。

## 相比上个节点

上一节点：[[D20260601-B3-gate误拦截Top1斜率放松2024压力测试通过]]

上一节点处理的是 B3 gate blocked raw 的漏拦截问题，目标是放松 gate 边界；本节点处理的是 B3 gate 已通过后的强度问题，目标是判断 cap80 是否仍偏轻。

相比本卡片最初状态，新增变化是：四段验证已经完成，2024 单段支持被 2020-2021 和动作胜率反证削弱；强度分层方向从“进入四段验证”变为“修订后再测”。

## 当前判断

```text
revise_after_split_validation
```

当前可做什么：

- 保留 cap70/cap60 作为强度边界对照，不直接采纳。
- 预注册 tiered-v2，重点减少 2022-2023 与 2025-20260519 误伤。
- 继续输出动作日、误伤日和关键回撤窗口审计。

当前不能做什么：

- 不能上线。
- 不能修改实盘默认开关。
- 不能只凭 2024 和 2025-04 冲击窗口采用 cap60。
- 不能把 R026 重新加入仓位动作。

## 当时问题

B3-gate-cap80 四段已通过，复合收益 +1351.04%，最大回撤 -30.00%；但 2024 最大回撤窗口显示 gate 在回撤初期触发有效，提示 cap80 可能仍偏轻。

验证后的问题变为：更强降仓确实能在深回撤和冲击窗口改善表现，但当前规则不能稳定区分“应强降仓的 gate 日”和“可能快速修复的 gate 日”。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| 2024 三组均通过 | cap70、cap60、tiered-v1 相对 B3-gate-cap80 分别改善回撤 +2.45pp、+4.77pp、+2.21pp |
| 2022-2023 固定强度改善 | cap70 收益 +32.82%、回撤 -26.54%；cap60 收益 +34.39%、回撤 -25.34%，均优于 cap80 |
| 2025-20260519 不拖累收益 | cap70、cap60、tiered-v1 总收益分别 +208.75%、+215.32%、+217.31%，均高于 cap80 的 +205.70% |
| 2025-04 冲击窗口支持强防守 | cap60 相对 cap80 窗口收益改善 +4.42pp，窗口回撤改善 +4.14pp |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 2020-2021 固定强度反证 | cap70 收益 -0.83pp、回撤 -0.05pp；cap60 收益 -3.76pp、回撤 -0.18pp |
| 2022-2023 动作胜率偏低 | cap70 64.00%，cap60 68.00%，tiered-v1 50.00%，触发预注册低胜率反证 |
| 2025-20260519 动作胜率偏低 | cap70 55.56%，cap60 55.56%，tiered-v1 50.00%，说明收益改善不是来自稳定高胜率动作 |
| tiered-v1 不稳定 | 2020-2021 完全不升级，2022-2023 升级日累计 log edge 为负，2025 只升级 2 天 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 少数冲击窗口贡献掩盖多数误伤 | 继续输出误伤日聚类、动作日胜率、动作日 log edge 与非动作日路径贡献 |
| 固定 cap60 是强度上界而非交易规则 | 保留为对照组，不进入实盘默认开启 |
| tiered-v1 阈值过窄或过拟合 | 预注册 tiered-v2，加入 Top5 exhausted count、全弱累计天数或恢复条件 |
| 单日 edge 正但路径扰动负 | 对 2020-2021 做动作日后续路径归因，避免只看同日超额 |

## 决策

```text
revise_after_split_validation
```

决策边界：

- 固定 cap70、cap60 不 promote，只作为强度边界参考。
- tiered-v1 不采纳，进入规则修订。
- 当前主候选仍是 `A2-slope004 + B3-gate-median-or-breadth-cap80`。
- 后续若做 tiered-v2，必须先写实验前预测、证伪规则和未来函数审计口径。

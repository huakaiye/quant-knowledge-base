# D20260601-R010B4 tiered-v2进入预注册

## 决策摘要

R010-B4 四段验证后，固定 cap70/cap60 不满足直接采纳条件，tiered-v1 也不稳定。新的方向是 tiered-v2：先用更严格的核心风险条件筛选 cap70，再在极端子集上使用 cap60。该方向只进入默认关闭实现和 2024 smoke 前置审计，不改变当前主候选。

## 是什么

本节点研究的是 B3 gate active 后的分层强度，而不是新的 gate 触发条件。

预注册主规则：

```text
strong_risk:
  B3 gate active
  AND r010b3_all_weak_cum_days >= 10
  AND r010b3_all_weak_persistence_gate_median_ret <= 0
  AND r010b3_top5_filter_exhausted_count >= 4
  -> cap70

extreme_risk:
  strong_risk
  AND (
    r010b3_all_weak_persistence_gate_median_ret <= -0.015
    OR r010b3_all_weak_persistence_gate_ma20_breadth <= 0.25
  )
  -> cap60
```

## 相比上个节点

上一节点：[[D20260601-R010B4-B3Gate强度分层进入预注册]]

上一节点完成 cap70、cap60、tiered-v1 的四段验证，结论是 `revise_after_split_validation`。本节点把修订方向具体化：不继续固定强度，不直接沿用 tiered-v1，而是用全弱持续天数、Top5 滤波衰竭和中位收益构成 core 风险层。

observe_pre_backtest

- 实现默认关闭 tiered-v2 参数。
- 生成 2024 smoke 配置。

禁止：

- 直接上线。
- 修改 MiniQMT 默认开关。
- 跳过 2024 smoke 直接跑四段长周期。
- 用动作日重建结果替代真实路径回测。

~~~markdown
```

```text

dry_run = 4/4 passed
formal_runs = 4/4 completed

all_path_checks_passed = true
```

```text
shadow_logs = 1542
true_triggers = 3
trigger_release_missing = 0
trigger_segments = 1/4
```

3 个真实触发全部发生在 2020 年 1 月：

```text
2020-01-03  512000.XSHG -> 512400.XSHG
2020-01-08  512980.XSHG -> 510410.XSHG
2020-01-10  512980.XSHG -> 510410.XSHG
```

R088 的 51 个只读代理事件在真实路径中被压缩为 3 个可触发事件，说明当前规则触发面不足。

## 允许

- 保留 R010-A10S1 default-off shadow 参数和日志。
- 保留四段配置、审计脚本、分析脚本和结果产物。
- 继续用 S1 日志作为 R010-A10S2 触发面扩展的输入。

## 禁止

- 禁止开启实盘或 guarded-live。
- 禁止把 S1 改成 live/apply 路径。
- 禁止让 shadow 改写真实 targets、weights、orders、trades。
- 禁止在本分支扩展小市值、A0、A1 或 all_actions。

## 下一步

```text
R010-A10S2 readonly trigger-surface expansion
```

优先分析真实路径中：

```text
scope_pass == true
AND switch_pass == true
```

但未触发 strict low_disp_small_gap 的样本，拆解：

```text
dispersion threshold
top1_top5_gap threshold
old target 是否仍在 topN
old target 与 new target 的 ret5/ret20/slope10 差异
delay_days = 1/2/3
```

只有当新规则同时满足触发容量、四分段稳定性和交易隔离要求，才允许进入下一轮 default-off shadow 候选。
~~~

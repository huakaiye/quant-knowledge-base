# D20260602-M006 小盘 SchemaOnly 指标覆盖率审计预注册完成

## 决策摘要

M006 预注册完成。路线从“M005 首轮确认本地代理部分可用”推进到“允许对 8 个 schema-only 指标做受限只读覆盖率查询”。这不是标签构造授权，也不是策略升级授权；它只授权字段覆盖率、日期范围、公告日/可得日、陈旧度和缺口分布审计。

## 是什么

本节点为小盘专用状态标签的数据库输入层建立查询边界。待审计指标包括小盘池成分、日频成交/换手/停牌/涨跌停、质量风险财报字段、ST 状态、09:30 容量和拥挤踩踏字段。审计脚本若连接 ClickHouse，必须使用平台 `ch_config`，只能执行 `SELECT` 聚合查询。

## 相比上个节点

上一节点：[[D20260602-M005-小盘专用状态标签输入审计首轮完成]]

上一节点证明 8 个本地代理指标覆盖率达标，但质量风险、市场宽度、实际流动性和拥挤踩踏仍是 schema-only。本节点把下一步从泛泛“需要覆盖查询”收敛为可执行边界：允许查哪些表、输出哪些文件、怎样判定 promote / observe / revise / kill，以及哪些行为直接使结果作废。

## 当前判断

```text
schema_only_coverage_audit_preregistered_readonly_query_ready
```

当前可做什么：

- 编写 `scripts/research/audit_m006_small_cap_schema_only_coverage.py`。
- 只读查询 `jq_trade_days`、`jq_stock_info`、`index_constituent`、`jq_valuation`、`jq_fina_indicator`、`jq_income`、`jq_balance`、`jq_bar_daily`、`jq_bar_minute_v2`、`jq_st_status`。
- 输出表字段覆盖率、四段覆盖率、公告日/可得日、ST 状态陈旧度、09:30 容量时点说明和未来函数审计。

当前不能做什么：

- 不能构造正式小盘状态标签。
- 不能生成日频标签序列。
- 不能运行回测。
- 不能自动更新数据。
- 不能修改策略、配置或实盘文件。
- 不能生成路由、权重、仓位或交易信号。

## 当时问题

M005 已经证明本地代理指标可以支持部分只读诊断，但它们多来自策略净值、持仓和成交结果，不等同于小盘全市场状态。若直接用这些代理构造标签，容易把策略结果误当作市场状态。下一步必须先验证 schema-only 字段是否真实可用。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| M005 输入审计 | 8 个本地代理覆盖率达标，但 8 个 schema-only 指标未连库验证 |
| 平台数据字典 | `jq_valuation`、`jq_fina_indicator`、`jq_income`、`jq_balance`、`jq_bar_daily`、`jq_bar_minute_v2`、`jq_stock_info` 等表有字段说明 |
| data_routing | `financials`、`income`、`balancesheet`、`st_status`、`index_constituent`、`daily_bar`、`minute_bar` 均标记 active |
| 小市值策略依赖 | M004 小市值配置声明 `valuation`、`financials`、`st_status`、`index_constituent`，策略使用 `399101.XSHE` 成分和前一交易日基本面 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 表存在不等于标签可用 | 仍需实际覆盖率、日期范围和缺口分布 |
| 财报字段有公告日风险 | 质量风险必须按 `ann_date <= signal_date` 或平台可得日审计 |
| 小盘池可能有成分未来污染 | 必须用 `effective_date <= signal_date`，不能用结束日成分 |
| 09:30 容量不能做开盘前标签 | 该字段只能进入执行容量审计 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 小盘本地代理已足够 | 先运行 M006；若 schema-only 覆盖不足，只能降为诊断词典，不进入完整标签 |
| 质量风险覆盖不足但不重要 | 后续标签归因前不能假设不重要；本轮只判断它是否可审计 |
| 小盘状态主要来自风格相对强弱 | 若质量和流动性字段缺口大，后续可修订为窄版相对强弱标签 |

## 决策

```text
schema_only_coverage_audit_preregistered_readonly_query_ready
```

决策边界：

- 只授权受限只读覆盖率审计。
- 审计通过只表示可以进入下一轮标签构造预注册，不表示可以交易化。
- 审计失败必须保留字段缺口和失败原因，不得用本地代理硬替代。

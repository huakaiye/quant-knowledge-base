# D20260603-R010B5S1 非线性仓位公式组件映射完成

## 决策摘要

用户提出的非线性仓位公式保留为研究启发，但当前只能进入组件级规格和只读审计，不进入 R010-B5R6、完整乘法公式回测或实盘。

## 是什么

R010-B5S1 审计的公式是：

```text
w_t = (sigma_target / sigma_t) * tanh(k * m_t) * exp(-a * D_t) * S_t
```

它把仓位控制拆成四个分量：

- 波动率控制：`sigma_target / sigma_t`。当前 `realized_vol` 只是池内风险代理，不等同组合级 `sigma_t`，`sigma_target` 不是现有 R010-B5 字段。
- 趋势过滤：`tanh(k * m_t)`
- 回撤反馈：`exp(-a * D_t)`
- 资金曲线状态：`S_t`

本节点只回答一个问题：这些分量在当前 R010-B5 / B3-gate-tiered-v2 研究链中哪些已有证据，哪些缺字段，哪些被前序反证限制。

## 相比上个节点

上一节点：[[D20260603-R010B5R5R4-低MedianRet样本外复核预注册完成]]

R5R4 把低 `median_ret` 过滤线索停靠到等待前向或独立 OOS，不允许进入 R010-B5R6。R010-B5S1 不绕开这个限制，而是另开“公式规格化支线”：把完整公式拆成组件级审计对象，避免直接组合四项做后验参数扫描。

## 当前判断

```text
observe_formula_component_spec_only_no_live_no_backtest
```

当前可做什么：

- 保留公式作为研究规格。
- 使用 R010-B5S1 输出的组件映射表，设计组件级只读审计。
- 为未来公式审计预注册日志字段，包括 `vol_scale`、`trend_tanh_scale`、`capital_curve_state`、`component_product_cap` 和 `component_cap_reason`。

当前不能做什么：

- 不能进入 R010-B5R6。
- 不能回测完整乘法公式。
- 不能把 `median_ret` 分支绕过 R5R4 的独立样本外限制。
- 不能把波动率项恢复为常开缩放。
- 不能修改 MiniQMT 或实盘默认开关。

## 当时问题

R010-B5 早期显示非线性回撤预算 overlay 在真实路径和敏感性阶段有离线优势，但严格 counterfactual 发现多数降仓事件在 5日/10日维度没有 edge。低 `median_ret` 过滤看起来能改善局部事件级表现，但 R5R4 确认没有独立样本外。

因此，若继续研究非线性仓位控制，必须先确认新公式是不是只是在叠加未验证组件，还是能拆成可证伪模块。

## 支持证据

| 证据 | 说明 |
| --- | --- |
| 回撤反馈项已有实现 | 平台源码包含 `_apply_r010b5_nonlinear_drawdown_budget` 与 `math.exp(-decay * excess)`，shadow 事件表包含 `portfolio_drawdown`、`drawdown_threshold`、`risk_scale`、`nonlinear_suggested_cap` |
| 环境字段已有 | R010-B5R5R2 字段审计确认 `realized_vol`、`median_ret`、`ma20_breadth`、Top1/Top5 等环境字段进入日志；其中 `realized_vol` 只能作为风险代理，不能直接等同组合级 `sigma_t` |
| 公式分解能降低混杂 | 将波动率、趋势、回撤反馈、资金曲线状态分开后，可以避免把 B3/tiered 风险识别和回撤反馈误混成一个新参数扫描 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 低 `median_ret` 没有独立样本外 | R5R4 审计 10 个现有资源后 `eligible_independent_resource_count=0`，R2 true path 动作日与 R5R3 27/27 重合 |
| 完整公式未实现 | 当前只有回撤反馈项已有实现；`tanh(km_t)` 和 `S_t` 不是已验证模块 |
| 常开缩放已被削弱 | BLF003 早期波动率管理结论不支持常开宽基波动率缩放，必须条件化 |
| [[资金曲线状态]]缺失 | 当前没有专用 `capital_curve_state`，不能用回撤本身替代 `S_t`，否则会重复低仓锁定问题 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 公式本身很强，只是旧 R010-B5 表达不完整 | 先做组件级日志和只读审计，不能直接完整公式回测 |
| 低 `median_ret` 已经足够作为趋势项 | 等待前向或独立 OOS，按 R5R4 固定阈值复核 |
| 波动率项能补上缺口 | 先审计 `realized_vol` 在 B3/tiered 风险窗口内的条件 edge，并对比常开负控制 |
| `S_t` 可以简单使用组合回撤 | 必须单独定义恢复、锁仓解除和资金曲线斜率，不能复用 `D_t` |

## 决策

```text
observe
```

决策边界：

- R010-B5S1 是规格和审计节点，不是新回测节点。
- 完整乘法公式当前状态为 `not_ready_for_backtest`。
- 后续已由 [[D20260603-R010B5S2-波动率项与资金曲线状态组件级只读审计预注册完成]] 接续；S2 也只证明输入可得，不允许完整公式回测。
- 任意使用低 `median_ret` 的部分必须继承 R5R4 的 `park` 边界。

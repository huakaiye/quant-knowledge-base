# D20260602-R010B5R4 默认关闭影子观察预注册完成

## 决策摘要

R010-B5R4 进入默认关闭影子观察规格阶段，只记录非线性建议 cap 与实际 tiered cap 的差异，不改变实盘交易。

当前判断：

```text
observe_shadow_only_no_live
```

## 是什么

本节点把 R010-B5R3 四段通过后的下一步落成可执行观察规格。

观察对象不是“是否上线非线性预算”，而是：

```text
在 B3 gate active 且 tiered-v2 已给出实际 cap 后，
非线性回撤预算会建议什么 cap，
该建议与实际 tiered cap 有何差异，
以及该差异是否有后验保护价值。
```

## 相比上个节点

上一节点：[[D20260602-R010B5R3-四段敏感性与成本扰动通过]]

上个节点证明参数和成本稳健，允许设计影子观察。本节点新增：

- R010-B5R4 研究记录。
- R010-B5R4 影子观察方案。
- 必备字段、停止条件、晋级条件和禁止项。

## 当前判断

```text
observe_shadow_only_no_live
```

当前可做：

- 只读审计平台现有字段。
- 设计只读日志抽取和字段审计脚本。
- 记录 `actual_tiered_cap`、`nonlinear_suggested_cap`、`cap_delta`。

当前不能做：

- 不能修改 MiniQMT 默认开关。
- 不能修改真实 target weights 或订单。
- 不能让非线性预算绕过 B3 gate 或 tiered-v2。
- 不能进入实盘。
- 不能后验切换到 `d10_a5_f65`。

## 当时问题

R010-B5R3 的四段结果支持“非线性预算可以继续研究”，但它还不能证明该模块可以安全进入实盘。最大不确定性已经从历史收益转向工程隔离和实盘可观察性：

```text
是否能只记录 suggested cap，而不改变真实交易？
```

## 支持证据

| 证据 | 说明 |
| --- | --- |
| R010-B5R3 四段通过 | 参数邻域和成本扰动均通过 |
| R010-B4 模板可复用 | tiered-v2 已有 shadow-only 观察方案作为结构参考 |
| 平台已有部分字段 | 研究策略已有 R010-B5 非线性状态/预算日志和 tiered-v2 action 字段 |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 平台字段未完全确认 | `records.json` 为空，`allocation_actions.csv` 可能为空，现有日志偏应用型 |
| 工程隔离风险 | suggested cap 若复用 effective cap 变量，可能误改真实目标仓 |
| 风险识别不可替代 | R010-B5 不能解决 B3 gate 漏报，只能在既有风险窗口内工作 |
| 重复 run 证据风险 | R010-B5R3 仍需随结论保留 duplicate_runs 审计文件 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 非线性建议只是重复 tiered-v2 | 统计非零 `cap_delta` 事件比例 |
| 非线性建议会误伤反弹 | 复核后续 1/5/10 日风险资产收益 |
| 非线性建议会削弱强风险防守 | 检查建议更高 cap 后是否回撤加深 |
| 影子观察实际改变交易 | 对比 target weights 和订单，要求 100% 不变 |

## 决策

```text
observe_shadow_only_no_live
```

决策边界：

- R010-B5R4 只建立观察规格。
- 下一步先做平台字段审计。
- 若字段不足，只允许补研究策略日志或只读抽取脚本。
- 不允许修改 MiniQMT 默认开关或实盘交易逻辑。
- 只有影子观察满足样本、胜率、edge 和交易隔离条件后，才允许预注册 R010-B5R5。

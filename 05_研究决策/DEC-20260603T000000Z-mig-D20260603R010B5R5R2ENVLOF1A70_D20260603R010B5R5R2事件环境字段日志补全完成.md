## 是什么

本节点只解决 R010-B5R5/R5R1 事件审计的可观测性问题。

新增字段会随未来 `R010-B5R4 SHADOW` 日志进入事件 CSV，包括：

```text
r010b5r5r2_env_log_version = R010-B5R5R2-env-v1
```

## 相比上个节点

上个节点：[[D20260603-R010B5R5R1-窄条件严格复核观察级完成]]

上个节点确认：

```text
A strong+extreme 只保留观察级，且被 5日分段冲突削弱。
B 只是 10日延迟防守观察。

不进入 R010-B5R6。
```

同时暴露出一个约束：

```text
旧事件 CSV 缺少环境字段，无法判断事件失败是否来自 median_ret、breadth、dispersion 或 Top5 exhaustion 的差异。
```

本节点的改变是：

```text
补齐日志与 CSV 字段通道。
固化静态字段审计与 payload 扫描工具。
把旧日志明确归类为 legacy/version_missing，而不是新样本验证通过。
不改变交易逻辑。
不改变 R5R1 的观察级结论。
```

## 当前判断

```text
env_logging_fields_ready_no_live_wait_new_samples
```

允许：

- 等待下一批带环境字段的 R010-B5R4 SHADOW 事件；
- 在新样本上复跑 R010-B5R4 事件聚合、R010-B5R5 counterfactual、R010-B5R5R1 子集复核；
- 将新增字段用于失败解释和候选筛查。

- 运行新回测，除非用户之后明确要求；
- 修改 MiniQMT；
- 修改实盘默认开关；
- 把日志字段补全写成策略通过；
- 宣称 strong/extreme 已经可替代固定 cap70/cap60。

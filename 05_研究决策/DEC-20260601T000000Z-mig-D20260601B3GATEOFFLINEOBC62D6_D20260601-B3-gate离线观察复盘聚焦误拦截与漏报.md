# D20260601-B3-gate 离线观察复盘聚焦误拦截与漏报

## 决策摘要

B3 gate 默认关闭观察不再把 R026 交易化作为下一步，而是先用历史离线复盘清单聚焦三类事件：gate active 是否有效、gate blocked raw 是否误拦截、non raw future dd5 是否是 B3 尚未识别的漏报风险。

## 是什么

本节点不是新交易规则，也不是实盘开启指令。它把已完成的 B3 gate 四段结果和 R026 增量审计逐日面板，整理成默认关闭观察时需要人工复核的事件清单。

平台脚本：

```text
scripts/research/analyze_b3_gate_observation_replay.py
```

输出目录：

```text
results/v2/research/R010-B3/gate_live_observation/offline_replay/
```

核心输出：

```text
b3_gate_observation_event_list.csv
b3_gate_observation_episode_summary.csv
b3_gate_observation_class_summary.csv
b3_gate_observation_segment_summary.csv
b3_gate_observation_manual_watchlist.csv
summary.json
```

## 相比上个节点

上一节点 [[D20260601-R026B-light-cap90首段反证]] 证明 `signal_B_equity_defense` 自动 cap90 在 2024 首段收益代价过高。本节点把 R026 从“可能轻量交易动作”进一步降到“解释字段”，并把研究资源转向 B3 gate 自身的观察质量：

- active/trigger 日是否真的跟随后续下探。
- gate blocked raw 是否包含被误拦截的风险日。
- non raw future dd5 日期里是否有可用解释变量。

## 当前判断

```text
observe
```

当前可做什么：

- 使用离线 watchlist 做人工复核。
- 在未来模拟盘、小资金或演练日志中按同一字段复核 raw、gate blocked、trigger、active。
- 把 R026 B层信号作为解释字段记录。

当前不能做什么：

- 不能据此主动开启 `ENABLE_B3_GATE_MOB_CAP80`。
- 不能恢复 R026B-light-cap90 四段长测。
- 不能让 R026 单独触发 cap90、cap80 或清仓。

## 当时问题

B3 gate 已经是当前主防守候选，但默认关闭观察阶段需要知道观察资源应放在哪里。R026B-light 的自动仓位动作首段失败后，剩余问题不是“R026 能否交易化”，而是：

```text
B3 gate 的历史失败或不确定样本集中在哪些事件类型上？
```

## 支持证据

| 证据 | 说明 |
| --- | --- |
| 事件总量 | 四段历史中整理出 290 个观察日、144 个连续事件段 |
| gate active 有效性 | trigger/active 合计 66 天，其中 27.27% 后续 10 日出现 -5% 下探 |
| gate active 误伤风险 | 18 天被标记为 `active_possible_trend_cost`，需要人工判断是否误伤趋势延续 |
| gate blocked 误拦截 | blocked raw 44 天，其中 8 天后续 10 日出现 -5% 下探 |
| non raw 漏报 | non raw future dd5 共 180 天，说明仍有 B3 raw 尚未覆盖的早期风险日 |
| R026 B层覆盖不足 | non raw future dd5 中，`signal_B_equity_defense` 只覆盖 10 天，覆盖率 5.56% |

## 反对证据

| 证据 | 说明 |
| --- | --- |
| 离线标签包含事后未来下探 | `future_dd5_10d` 只用于复盘分类，不能直接作为交易信号 |
| active 命中率不是很高 | 27.27% 后续 -5% 下探说明 gate 仍有大量模糊样本 |
| wide R026 覆盖高但噪声大 | `signal_any_defense_mem30` 覆盖 non raw future dd5 58.89%，但此前已证明覆盖 gate blocked raw 也高，不能直接交易化 |
| 缺少持仓级复盘 | 当前事件表还没有把当日 Top1、实际目标权重和防御 ETF 替代损益完整拆开 |

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| B3 gate 已经足够，误拦截只是少数噪声 | 人工复核 8 个 blocked false negative 日是否集中在同一类 gate 变量边界 |
| non raw 漏报来自更早的池内弱化变量 | 复核漏报日前的 median_ret、ma20_breadth、ret20_breadth、Top1 slope |
| R026 wide memory 有解释力但不能交易 | 分离 `missed_only_wide_R026` 与 `missed_without_R026` 的后续路径，不直接改变仓位 |
| active 误伤来自 limited2 退出或替代资产选择 | 对照 active 后原风险 ETF 与 511880 的相对路径 |

## 决策

```text
observe
```

决策边界：

- B3 gate 仍是当前主防守候选，保持默认关闭观察。
- 后续观察重点改为 `blocked_false_negative_dd5`、`non_raw_future_dd5_10d` 和 `active_possible_trend_cost` 三类样本。
- R026 只保留为 `r026_marker`，不得作为独立仓位动作。

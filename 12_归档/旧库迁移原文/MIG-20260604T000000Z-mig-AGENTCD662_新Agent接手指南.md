# 新Agent接手指南

本文用于让任意 Agent 在新窗口、新对话或无旧上下文情况下接手研究库。接手时不要依赖历史聊天记录，以本文和研究库内文档为准。

## 5分钟接手流程

> ⚠️ **新：先读术语词典。** 遇到任何不认识的缩写（B3、tiered-v2、cap80、R010-B5…）直接 Ctrl+F 搜词典。

| 步骤 | 动作 | 目的 |
| --- | --- | --- |
| 1 | 阅读 [[术语词典]] | **新增。** 5 分钟扫完 100+ 个术语，后续阅读不再被缩写挡路 |
| 2 | 阅读 [[📊 驾驶舱]] | 1 页看全局当前状态，这是唯一当前入口 |
| 3 | 阅读 [[Agent操作规范]] | 掌握研究库硬规则 |
| 4 | 阅读 [[研究库格式规范标准]] | 明确新版本研究库的文件、字段、状态和命名规范 |
| 5 | 从驾驶舱进入对应方向页 | 在 `🧭 方向/` 中阅读当前方向总结、结论、阻塞和下一步 |
| 6 | 阅读 [[决策链路说明]] | 理解决策卡片格式、路线判断和边界 |
| 7 | 阅读 [[研究资产清单]] | 需要复原实验、配置、结果和 manifest 时使用 |
| 8 | 阅读 [[Agent研究方法论交接]] | 掌握研究方法和实验纪律 |
| 9 | 阅读 [[子代理分层调度研究方案]] | 需要并行调用子代理、运行回测或分工审计时使用 |

## 如果用户只说"继续"

默认按以下优先级处理：

1. 查看 [[📊 驾驶舱]] 的"当前最高优先级"。
2. 若用户上一句明确指定文档或专题，继续该专题。
3. 若平台仍不可用，优先推进研究库工作：补充方法库、完善研究信念、写实验前假设和证伪设计。
4. 若平台已恢复，先补齐研究记录，再按平台规则在 WSL 内运行回测。

## 子代理使用提醒

- 子代理命名中文优先，统一使用 `子查`、`子研`、`子执`、`主控` 四类前缀。
- 委派提示词第一行必须写任务代号，例如 `任务代号：子查-001-实验记录检索`。
- 子代理最终回复必须复述任务代号、使用模型、读取文件、修改文件、执行命令和结论边界。
- 运行回测的子代理必须提供实时日志路径和 `tail -f` 进度查看命令。
- 不要让子代理独立决定 `kill / revise / promote / park`、实盘边界或未来函数最终结论。

## 如何开展一项新研究

| 阶段 | 必须产物 | 说明 |
| --- | --- | --- |
| 灵感 | 文献笔记或模块想法 | 不直接进入回测 |
| 机制 | 方向页或方向页中的当前信念章节 | 写清楚相信什么、为什么、在哪里会失效 |
| 实验前 | 研究记录草稿 | 写假设、预测、竞争性解释、证伪测试 |
| 执行 | 平台配置和结果目录 | 配置和大结果表放平台项目 |
| 实验后 | 研究记录更新 | 写实际观察、偏差诊断、信念更新 |
| 决策 | 研究决策卡片 | 按 [[决策链路说明]] 新建卡片 |
| 收束 | 方向页更新 | 方向收束时更新 `🧭 方向/` 下的当前总结和边界 |
| 资产 | 刷新 [[研究资产清单]] | 实验完成后更新配置、结果、manifest 和决策索引 |
| 驾驶舱 | 更新 [[📊 驾驶舱]] | 方向状态变化时更新对应方向页 frontmatter |

## 如何接续已有研究

先打开 [[📊 驾驶舱]]，找到该方向的状态行，再进入 `🧭 方向/` 下的方向页。然后回答：

- 当前研究信念是什么。
- 实验前预测是什么。
- 已经有哪些支持证据和反对证据。
- 当前决策是 `promote`、`revise`、`park` 还是 `kill`。
- 下一轮实验是为了减少什么不确定性。

不要直接从"上一次参数结果"继续调参。若缺少实验前预测，先补预测和证伪测试。

## 常见错误

- 只看回测收益，不写实验前预测。
- 看到结果好就直接采纳。
- 看到结果差就直接大规模扫参数。
- 忽略交易成本、换手、样本外、消融和未来函数。
- 在研究库保存大型 CSV 或交易明细。
- 忘记在方向页 frontmatter 中更新状态，导致 [[📊 驾驶舱]] 不一致。
- 方向收束后忘记更新 `🧭 方向/` 下的当前总结。
- 做了方向判断但没有写研究决策卡片。
- 完成实验后没有刷新 [[研究资产清单]]。

# 决策链路说明

本目录保存会改变研究路线的关键判断节点。决策卡片不替代实验记录，而是把长实验记录中的关键判断抽成可浏览、可追溯、可放入 Canvas 的研究路径。

## 阅读规则

- 每张决策卡应能在 3 分钟内读完。
- 卡片中关键术语首次出现时，应链接到概念页或方向页。
- 决策标签必须使用统一枚举，避免同义词混用。
- Canvas 只引用决策节点，不在 Canvas 中保存唯一研究事实。

## 标准结构

```markdown
# D2026XXXX-XXX-简短标题

## 是什么
用白话说明本节点判断的对象。

## 相比上个节点
说明上一节点结论，以及本节点要解决的不确定性。

## 证据
列出支持证据和反对证据，尽量指向实验记录、结果目录或审计文件。

## 决定
写明 promote、kill、revise、park、observe 或 live，并说明允许动作和禁止动作。

## 下一节点
写明下一张决策卡，或标注“链尾”。
```

## 决策类型

| 标签 | 含义 | 后续 |
| --- | --- | --- |
| promote | 晋级，进入下一验证阶段 | 继续推进 |
| kill | 终止，不再恢复该假设 | 不恢复 |
| revise | 修订方向，保留核心思路但改变实现方式 | 新分支 |
| park | 暂停，等待新数据或条件 | 不主动推进 |
| observe | 只观察，不交易 | 日志层保留 |
| live | 已按用户授权进入实盘或默认启用 | 必须保留审计链 |

## 写作要求

- 不用“感觉有效”这类模糊判断。
- 必须写清证据来源、样本范围和禁止动作。
- 改变实盘开关的决策必须写明用户授权、审计结果和回滚边界。
- 决策卡可以引用实验记录，但实验数据、CSV、JSON、日志仍保存在平台结果目录。

~~~

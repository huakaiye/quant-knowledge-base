## 状态判定草案

```text
if crash_rebound_risk:
    state = "crash_rebound"
elif median20 < 0 and top5_power_count == 0:
    state = "weak_no_leader"
elif breadth_high and median20 > 0 and broad_index_trend > 0:
    state = "broad_uptrend"
elif dispersion_high and top1_power_strong:
    state = "narrow_leader"
elif dispersion_mid and top5_power_count >= 2:
    state = "rotational_momentum"
elif dispersion_low and top1_top5_gap_low:
    state = "low_dispersion_chop"
else:
    state = "neutral_observe"
```

## Top5 动力确认逻辑

当 Top1 动力不足时，不应立刻防守，应继续检查 Top2 到 Top5：

| 情况 | 动作 |
| --- | --- |
| Top1 强 | 正常交易 Top1 或 TopN |
| Top1 弱但 Top2/Top3 强 | 可以买入仍有动力的候选，降低仓位或提高确认 |
| Top1 到 Top5 至少 2 个强 | 维持轮动，但降低追涨惩罚 |
| Top1 到 Top5 全弱 | 进入防守状态 |
| Top1 到 Top5 分差很小且均弱 | 降低换手，优先保留旧仓或防守 |

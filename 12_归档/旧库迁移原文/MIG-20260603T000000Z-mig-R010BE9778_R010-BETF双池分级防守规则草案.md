# R010-B ETF双池分级防守规则草案

## 目标

R010-B 的目标不是再次做 510300 二元 KillSwitch，而是把 R019 的长周期状态日志转化为可证伪的分级防守规则。

核心原则：

- 使用策略池内生信号，不依赖单一外部大盘基准。
- 先做分级动作，不直接全仓切 511880。
- 区分“短期风险升高”“过热拥挤”“真正弱势无主线”三类状态。
- 所有信号必须只使用交易日前已可见数据。

## R019 证据约束

| 证据 | 对 R010-B 的约束 |
| --- | --- |
| `top5_power_count` 1542 条记录中有 1482 条等于 5 | 删除或弱化该字段，不能再作为防守核心触发 |
| 原分类器 T-10 只覆盖 6/15 个主要回撤段 | 不能直接把 R010-A 的 `caution/defensive` 当交易开关 |
| `ma20_breadth < 0.30`、`median_ret < 0` 对 5/10 日风险有信息 | 优先作为短期降风险信号，而不是长期看空信号 |
| `overheat/crowded` 有时仍对应强趋势 | 只能限制追高和换手，不能简单清仓 |

## 输入信号

| 信号 | 来源 | 用途 |
| --- | --- | --- |
| `ma20_breadth` | R019 状态日志 | 判断短中期趋势广度是否塌陷 |
| `ret20_breadth` | R019 状态日志 | 判断全池正动量数量 |
| `ma20_ma60_breadth` | R019 状态日志 | 判断中期趋势背景 |
| `median_ret` / `median_ret20` | R019 状态日志 | 判断全池中位收益是否为负 |
| `top1_ret5` | R019 状态日志 | 判断最强候选是否近端失速 |
| `top1_ret20` | R019 状态日志 | 区分“强趋势回撤”和“无主线弱势” |
| `top1_slope10` | R019 状态日志 | 判断短斜率是否转负 |
| `dispersion20` | R019 状态日志 | 判断横截面是否有轮动价值 |
| `top1_top5_gap` | R019 状态日志 | 判断最强者优势是否极端或不明显 |
| `confirmed_state` | R019 状态日志 | 作为辅助解释，不作为唯一触发 |

## 状态分层

| 分层 | 候选条件草案 | 解释 | 初始动作 |
| --- | --- | --- | --- |
| `normal_attack` | `ma20_breadth >= 0.40` 且 `median_ret >= 0` | 池内趋势健康 | 不干预 |
| `short_risk_watch` | `ma20_breadth < 0.35` 或 `median_ret < 0`，但 Top1 仍有正 20 日动量 | 短期风险升高，不等于趋势结束 | 降低新开仓积极度 |
| `leader_exhaustion` | `top1_ret5 < 0` 且 `top1_slope10 <= 0`，同时 `ma20_breadth < 0.40` | 最强候选近端失速 | 降低换手，避免追 Top1 |
| `weak_no_leader` | `ma20_breadth < 0.30` 且 `median_ret < 0`，且 Top1 近端失速 | 池内弱势无主线 | 降低风险资产上限 |
| `crowded_overheat` | `confirmed_state in [crowded, overheat]` 或 Top1 分差极端，同时 Top1 近端转弱 | 强势后拥挤，可能反转 | 限制追高，不主动清仓 |
| `defense_confirmed` | `weak_no_leader` 连续确认，且 `ret20_breadth < 0.35` 或 `ma20_ma60_breadth < 0.35` | 真正进入防守候选 | 半仓或部分防御，不默认全仓货币 |

## 动作分级

| 动作 | 说明 | 允许触发状态 |
| --- | --- | --- |
| A0 不干预 | 保持原 ETF 双池逻辑 | `normal_attack` |
| A1 降低追高 | 新买入需要 Top1/Top2 近端收益为正；否则继续看 Top3-Top5 | `short_risk_watch`、`leader_exhaustion` |
| A2 降低换手 | 若旧持仓未触发卖出，且旧持仓近端动量仍不差，则不因微弱排名差主动换仓 | `leader_exhaustion`、`crowded_overheat` |
| A3 风险资产上限降至 50%-70% | 只降低目标暴露，不全仓撤出 | `weak_no_leader` |
| A4 部分防御仓 | 风险资产上限降至 30%-50%，剩余进 511880 或可用防御资产 | `defense_confirmed` |

R010-B 第一轮不测试 A4 全仓防御。A4 也必须保留至少一部分风险资产，以避免 R015 那种长期防守导致收益坍塌。

## Top1 到 Top5 递补规则

用户提出的核心思路应保留：如果 Top1 没有上涨动力，不应立刻防守，应继续检查 Top2 到 Top5。

候选规则：

1. 按原策略排序得到 Top1 到 Top5。
2. 对每个候选检查 `ret5 > 0` 或 `slope10 > 0`。
3. 若 Top1 失速但 Top2-Top5 中仍有候选满足近端动力，则允许买入最强的有效候选。
4. 若 Top1-Top5 均近端失速，再结合 `ma20_breadth` 和 `median_ret` 判断是否进入 A3/A4。

这一路径优先解决“Top1 不强但 Top2/Top3 仍强”的问题，避免过早防守。

## 确认与退出

| 机制 | 规则草案 |
| --- | --- |
| 进入确认 | `weak_no_leader` 需要连续 2 个交易日；`defense_confirmed` 需要连续 3 个交易日或两类广度同时塌陷 |
| 退出确认 | `ma20_breadth > 0.40` 且 `median_ret >= 0`，或 Top1/Top2 近端动力恢复 |
| 冷却期 | 状态动作切换后至少 3 个交易日不反向切换 |
| 强趋势保护 | 如果 `top1_ret20` 很强且 `dispersion20` 高，只允许 A1/A2，不允许直接 A3/A4 |

## 回测拆分

| 阶段 | 交易动作 | 目的 |
| --- | --- | --- |
| R010-B0 | 只在日志中输出建议动作 | 检查动作是否覆盖 R018 主要回撤前 5/10/20 日 |
| R010-B1 | Top1 失速后递补 Top2-Top5 | 验证“继续看前五名”是否优于直接防守 |
| R010-B2 | A1/A2：限制追高和换手；A2需要当前持仓动量约束 | 检验低侵入式风控是否改善回撤或收益路径 |
| R010-B3 | A3：风险资产上限降至 50%-70% | 检验条件降仓 |
| R010-B4 | A4：部分防御仓 | 最后测试，必须和收益损失同时评估 |

## 验收标准

| 维度 | 标准 |
| --- | --- |
| 解释力 | R018 主要回撤前 T-10/T-5 的覆盖率高于 R019 原分类器 |
| 收益损失 | 年化收益不能因长期防守显著坍塌 |
| 回撤改善 | 最大回撤和连续回撤长度至少有一项改善 |
| 稳健性 | 不能只改善 2022 或 2024 单一年份 |
| 可交易性 | 不依赖当日收盘后才知道的数据 |

## 决策口径

R010-B 的第一目标是减少错误进攻，不是追求最高收益。若 B1/B2 能减少回撤但不明显损害收益，应优先于 B3/B4。只有在 B1/B2 无效时，才进入降仓和防御仓测试。

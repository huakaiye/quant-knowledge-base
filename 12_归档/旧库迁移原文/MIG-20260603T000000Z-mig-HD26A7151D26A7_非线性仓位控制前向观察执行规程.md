# 非线性仓位控制前向观察执行规程

## 目的

本规程用于 R010-B5S4 之后的前向样本重开判断。它解决一个具体问题：未来如果出现新的 R010-B5 shadow-only 事件，任何 Agent 都必须按同一套标准判断是否可以重开 rolling sigma mid 参考分组确认，不能把旧样本、同日反事实、聚合汇总或后验扫描当成独立样本外。

当前方向页：[[非线性仓位控制]]

当前决策：[[D20260603-R010B5S4-前向样本可用性审计完成]]

## 当前边界

R010-B5S4 的结论是：

```text
park_forward_confirmation_until_new_independent_samples_no_formula_backtest
```

这表示：

- 当前没有可用新增独立事件样本。
- 完整乘法公式不得回测。
- 不得进入 R010-B5R6。
- 不得扫描目标波动率、`sigma_target` 或 `vol_scale`。
- 不得生成 cap、目标权重、订单或实盘开关。
- 不得声称可顶替 B3 gate、tiered-v2 或固定 cap70/cap60。

## 重开条件

只有同时满足以下条件，才允许新建下一轮只读确认实验。

| 条件 | 固定要求 | 不满足时处理 |
| --- | --- | --- |
| 事件日期 | 新事件日期不在 S3 的 27 个事件日内 | 继续 park |
| 新增事件数 | 不少于 8 个 | 只记录观察，不确认 |
| 分段覆盖 | 不少于 2 个分段 | 只记录观察，不确认 |
| 样本来源 | 必须是新增前向或独立 shadow-only 事件 | 不得使用 |
| 事件级字段 | 必须能识别 `sigma20_rank_state` 或 `sigma60_rank_state` | 先补只读字段审计 |
| 交易隔离 | 必须证明 shadow 不影响实际订单和目标权重 | 先做隔离审计 |

## 可确认对象

只允许确认 S3 留下的两个参考分组：

| 参考分组 | 允许问题 | 禁止问题 |
| --- | --- | --- |
| `sigma20_rank_state=mid` | 新样本中 5日与10日 edge 是否仍偏正 | 不得改阈值，不得扫描 low/mid/high 边界 |
| `sigma60_rank_state=mid` | 新样本中 5日与10日 edge 是否仍偏正 | 不得转成目标波动率缩放 |

不得新增 `sigma_target`、`vol_scale`、波动率阈值扫描、资金曲线补仓规则或完整乘法公式。

## 首轮执行步骤

### 1. 样本资格审计

先运行 S4 只读审计脚本：

```powershell
[Console]::OutputEncoding=[System.Text.UTF8Encoding]::new($false)
$OutputEncoding=[System.Text.UTF8Encoding]::new($false)
python E:\量化平台_V1.4.0\scripts\research\audit_r010b5s4_forward_sample_readiness.py
```

检查输出：

```text
E:\量化平台_V1.4.0\results\v2\research\R010-B5\r010b5s4_forward_sample_readiness\r010b5s4_forward_sample_readiness_summary.json
```

只有当 `current_forward_sample_available=true` 且 `eligible_independent_resource_count>=1` 时，才允许进入下一步。

### 2. 新建下一轮预注册

若样本资格通过，先新建研究记录，不直接运行确认脚本。

建议命名：

```text
R2026MMDD-xxx_R010B5S5_滚动波动率中位参考分组新增样本只读确认预注册.md
```

必须固定：

- 新样本来源路径。
- 新样本事件日期清单。
- 与 S3 27 个事件的重叠审计。
- 5日与10日 edge 的计算方式。
- 负控制。
- 决策规则。

### 3. 只读确认

确认实验只允许回答：

```text
新增样本中 rolling sigma mid 参考分组是否仍表现为 5日与10日偏正，且负控制不污染。
```

不得回答：

```text
是否可以上线。
是否可以生成仓位公式。
是否可以替代 tiered-v2。
是否可以进入 R010-B5R6。
```

## 决策规则

| 结果 | 决策 | 下一步 |
| --- | --- | --- |
| 无新增独立样本 | park | 等待样本 |
| 新样本不足 8 个或不足 2 分段 | observe | 只记录，不确认 |
| 5日和10日均值为正，正率不低于 60%，负控制不污染 | observe | 允许继续下一轮只读确认，不得回测 |
| 5日或10日不通过 | kill | 停止 rolling sigma mid 参考分组 |
| 负控制同样偏正 | kill | 视为分组污染或数据挖掘 |

本分支最高只能到 `observe`，不能直接 `promote` 到回测、R010-B5R6 或实盘。

## 禁止把以下对象当作新增样本

| 对象 | 原因 |
| --- | --- |
| R4/R5/R5R2 同日期 shadow 事件 | 与 S3 27 个事件重叠 |
| R3 参数敏感性汇总 | 聚合表，没有事件级 rolling sigma mid 成员 |
| R2 true path 汇总 | 同策略路径聚合结果，不是事件级独立样本 |
| S2/S3 事件面板 | 直接输入和诊断样本 |
| R5R3 低 median 成员表 | 旧事件子集，不是独立样本 |
| 回测收益变好 | 不是事件级反事实证据 |

## 交接要求

每次新样本资格审计后，必须写回：

- [[非线性仓位控制]]
- 对应 R010-B5S* 研究记录
- 对应 R010-B5S* 决策卡
- [[研究库健康检查]]

若只是继续 park，更新方向页的 `updated_at`、`blocker` 和 `next_step` 即可；不得为了“有进展”新增交易规则。

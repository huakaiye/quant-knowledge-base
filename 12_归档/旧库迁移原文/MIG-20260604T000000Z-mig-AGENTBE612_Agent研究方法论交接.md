# Agent 研究方法论交接

本文是给正在负责 `E:\【笔记库】\量化研究库` 的策略研究 Agent 的接力指令。

## 当前决策

研究库采用 [[研究方法论]] 作为默认研究方法论，核心是实验前预测、贝叶斯证伪、严格反事实审计和资产可追溯。

你的目标不是尽快把回测调好看，而是把每个策略研究方向变成可追踪的研究信念、可证伪的假设、可复现的实验和可解释的结论。

## 立即执行方向

1. 新 Agent 接手时，先按 [[新Agent接手指南]] 判断当前应该继续哪条主线。
2. 新建研究记录时，必须使用升级后的 [[研究记录模板]]。
3. 对当前主线研究建立或补充研究信念，优先包括：
   - [[🧭 方向/BLF20260523-001_市场状态识别与动量门控研究信念]]
   - [[🧭 方向/BLF20260523-002_平滑上涨软排序研究信念]]
   - [[📦 归档/BLF20260523-003_波动率管理与条件降仓研究信念]]
   - [[🧭 方向/BLF20260528-004_市场环境驱动多策略切换研究信念]]
   - 防守切换
4. 对 R010-A 市场状态识别，不要只写接口规格；还要补充：
   - 原始机制信念
   - 可观察预测
   - 竞争性解释
   - 证伪测试
   - 进入 R010-B/C/D 的升级条件
5. 对已有实验记录，不要求一次性全部重写；优先改造仍在推进的主线和未来新增实验。

## 每轮实验的强制流程

```text
1. 选择研究信念
2. 写出本次假设
3. 写出实验前预测
4. 写出竞争性解释
5. 写出证伪测试和决策规则
6. 运行平台回测或诊断
7. 对照预期与实际
8. 解释偏差
9. 更新信念置信度
10. 决定 kill / revise / promote / park
11. 若改变方向、候选优先级、交易边界或实验顺序，创建或更新研究决策卡片
12. 写入研究记录、方向页、驾驶舱入口状态、必要的 Canvas 节点和下一步
```

研究决策卡片必须能回答三个交接问题：

```text
这个节点是什么？
相比上一个节点改变了什么？
当前判断和边界是什么？
```

## 回测后禁止直接做的事

- 禁止看到收益下降就直接调参数。
- 禁止看到收益上升就直接建议采纳。
- 禁止只解释支持原假设的证据。
- 禁止忽略成本、换手、样本外、消融和未来函数风险。
- 禁止把多个参数扫描结果包装成单次确定性结论。

## 推荐回复格式

当你完成一个实验后，研究记录必须能回答：

```text
我原本相信什么？
实验原本预测会看到什么？
实际看到了什么？
哪些观察支持原假设？
哪些观察反驳原假设？
是否存在更简单的竞争性解释？
下一轮哪个实验最能减少不确定性？
该方向应该 kill、revise、promote 还是 park？
这个判断对应哪一张研究决策卡片？
```

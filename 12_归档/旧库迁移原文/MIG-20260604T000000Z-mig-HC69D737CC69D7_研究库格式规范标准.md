# 研究库格式规范标准

> 本文件定义研究库所有 Markdown 文件的统一格式标准。新增文件必须遵守，存量文件逐步治理对齐。

## 五、Tags 格式规范

### 5.1 统一使用多行 list 格式

```yaml
tags:
  - 类别/子类
  - 状态/xxx
```

**禁止使用**：
- ~~`tags: [a, b]`~~（inline-array）
- ~~`tags: "a, b"`~~（string）

### 5.2 Tags 命名规范

采用 `大类/小类` 层级结构：

| 前缀 | 含义 | 示例 |
|------|------|------|
| `研究/` | 研究体系相关 | `研究/实验`, `研究/决策`, `研究/信念` |
| `策略/` | 策略相关 | `策略/趋势跟踪`, `策略/横截面动量` |
| `概念/` | 概念分类 | `概念/策略组件`, `概念/防守端` |
| `状态/` | 当前状态 | `状态/观察中`, `状态/active` |
| `来源/` | 收益来源 | `来源/趋势跟踪`, `来源/事件驱动` |
| `执行/` | 执行方式 | `执行/ETF轮动`, `执行/风控叠加` |
| `资产/` | 资产类别 | `资产/ETF`, `资产/股票` |
| `市场/` | 市场相关 | `市场/A股`, `市场/美股` |

### 5.3 通用体系标签

所有文件至少包含一个体系标签：
- `研究库治理` — 规范和治理类文件
- `研究/实验` — 实验记录
- `研究/决策` — 决策卡片
- `研究/信念` — 信念文件
- `研究/文献` — 文献笔记
- `研究/想法` — 模块想法

## 七、实验记录 Frontmatter 标准

实验记录必须满足以下规则：

1. 不允许重复顶层键。
2. `tags` 必须放在 frontmatter 最后。
3. `status` 必须使用标准状态值。
4. `tags` 中必须包含 `研究/实验` 和与 `status` 一致的 `状态/{status}`。
5. 具体实验结论写入 `decision`、`decision_action`、`result_status`，不要写进 `status`。

推荐字段顺序：

```yaml
---
type: 实验
research_id: RYYYYMMDD-001
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
direction_id: 待填
topic_id: 待填
result_status: preregistered_no_run
governance_review_status: 待确认
related_strategies:
  - 待填
belief_id: BLFYYYYMMDD-001
hypothesis_id: HYYYYMMDD-001
source_tags:
  - 待填
execution_tags:
  - 待填
status: 草稿
decision: 待判断
decision_action: 待判断
research_stage: 灵感收集
progress: 0
blocker:
next_action: 待填
related_factor:
prior_confidence: 中
posterior_confidence: 待判断
config_path: configs/research/RYYYYMMDD-001/
result_path: results/v2/research/RYYYYMMDD-001/
decision_ids: 待填
tags:
  - 研究/实验
  - 状态/草稿
---
```

## 八、文件命名规范

### 8.1 实验记录

```
R{YYYYMMDD}-{序号}_{简要描述}.md
```

示例: `R20260601-027_ETF双池防守模块预注册.md`

- 序号在同一日期下必须唯一
- 描述使用中文，以下划线分隔

### 8.2 决策卡片

```
D{YYYYMMDD}-{主体}_{简要描述}.md
```

示例: `D20260601-B3-gate误拦截Top1斜率放松预注册.md`

### 8.3 文献笔记

```
{年份}_{第一作者}_{关键词}.md
```

示例: `2014_Da_FrogInThePan_平滑上涨动量.md`

### 8.4 信念文件

```
BLF{YYYYMMDD}-{序号}_{主题}.md
```

示例: `BLF20260523-001_市场状态识别与动量门控研究信念.md`

统一使用 Obsidian wiki-link 格式，例如 `[[研究记录模板]]`。

### 9.3 数据表格

统一使用 Markdown 表格，对齐格式一致。

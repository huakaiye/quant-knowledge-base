## 系统边界

| 系统 | 职责 |
| --- | --- |
| 研究库 | 文献、研究信念、假设、实验设计、结果解释、失败记录、路线决策 |
| 量化平台 | 数据、配置、回测、工程实现、结果文件、实盘接口 |

研究库不要保存大型结果表。大型 CSV、交易明细、净值、参数扫描结果放在平台：

```text
E:\量化平台_V1.4.0\results\v2\research\<direction_id>\<research_id>_<topic_id>\
```

```text
E:\量化平台_V1.4.0\configs\research\<direction_id>\<research_id>_<topic_id>\
```

存量配置和结果目录可能仍使用旧路径。查找存量资产时先看 [[研究资产清单]]，不要凭目录名猜测。

## 研究方法硬规则

- 默认使用 [[研究方法论]] 中的假设驱动、贝叶斯证伪和反事实审计纪律。
- 每个正式实验必须先有研究信念、假设、可观察预测、竞争性解释、证伪测试和决策规则。
- 回测结果好时，先找反证，再考虑升级。
- 回测结果差时，先解释偏差，再考虑改参数。
- 没有样本外、消融、成本和未来函数审计的研究，最多标记为 `观察中`。
- 决策使用 `kill / revise / promote / park`，不要用模糊措辞替代。

## 可视化与决策卡片硬规则

- 每一次方向决策必须新建或更新一张 `type: 决策` 卡片，存放在 `🔀 决策/`，格式参照 [[决策链路说明]]。
- “方向决策”包括：改变研究方向、候选优先级、交易边界、实盘观察边界、后续实验顺序，或产生 `kill / revise / promote / park / observe / promote_candidate / strict_marker_only` 等路线判断。
- 决策卡片必须让完全不了解研究的人看懂：该节点是什么、相比上一个节点改变了什么、当前判断是什么、证据和反证是什么、下一步做什么。
- 只改脚本路径、补充单个结果表、修正文档错字或不改变路线的小参数说明，不需要单独建卡片；但如果这些改动改变了路线边界，必须建卡片。
- 方向决策写入后，必须同步更新相关方向页；若该决策改变主链、分支或当前候选，还应更新 `📊 研究全局路线.canvas`。
- Canvas 是展示层，不是唯一事实来源；事实、证据和边界必须保存在 Markdown 决策卡片、研究记录或专题页中。

- 不自动更新数据。
- 不自动运行回测，除非用户明确要求或当前任务已进入实验执行阶段。
- CLI 回测必须按平台规则在 WSL 内运行。
- 运行任何平台命令前，先读 `E:\量化平台_V1.4.0\docs\AGENT_RULES.md`。
- 当前研究库任务若只是文献、方法、路线、状态设计，不访问数据库。

## 子代理调用规则

- 需要并行检索、格式检查、资产盘点、证据链归纳或受控执行时，先读 [[子代理分层调度研究方案]]。
- 子代理任务代号统一使用中文优先命名：`子查-001-研究记录检索`、`子研-001-证据链归纳`、`子执-001-可见进度回测`、`主控-001-路线终审`。
- 旧格式 `S53/M54/CTL` 只作为历史别名，不作为新委派首选。
- 委派提示词第一行必须写任务代号，最终回复必须复述任务代号、使用模型、读取文件、修改文件、执行命令和结论边界。
- 子代理有当前任务范围内的完整访问权限，但不得绕过研究治理、自动更新数据、破坏性 Git 操作、修改实盘默认开关或独立做最终路线判断。
- 子代理运行回测时必须使用可见进度命令，并提供日志路径、PID 和 `tail -f` 查看命令。

## Obsidian 结构规则

- 表格中不要使用带别名的 wiki 双链，例如不要写“页面加竖线别名”的双链。
- 模板中的占位符不要写成 wiki 链接，避免产生未解析链接。
- 新增收益来源、执行方式、策略名或概念名时，应同步建立对应词条页。
- 新增文件后，必须运行全库格式检查和双链检查，并更新 [[研究库健康检查]]。
- 修改当前生效规范、接手指南、实验模板或决策说明后，必须运行规范专项检查。

## 接手后的默认动作

如果用户只说“继续”“推进”“接着做”，不要凭旧对话猜测。按以下顺序判断：

1. 打开 [[📊 驾驶舱]]。
2. 根据驾驶舱进入对应 `🧭 方向/` 页面。
3. 打开 [[决策链路说明]] 和最近决策卡，确认当前主链和边界。
4. 如需复原实验资产，打开 [[研究资产清单]]。
5. 查找方向页 `next_step` 明确且未阻塞的主线。
6. 若平台不可回测，优先推进文献、方法、规格、实验前预测和资产治理。
7. 若平台可回测，先建立或补齐研究记录中的实验前预测，再按平台规则运行回测。

## 完成任务后必须写回

- 如果新增或修改研究资产，更新相关方向页、资产清单或驾驶舱。
- 如果完成实验，更新研究记录、研究信念和下一步。
- 如果产生方向决策，新增或更新研究决策卡片，并同步相关方向页；影响主链时同步更新 Canvas。
- 如果发现失败方向，也必须保留失败原因。
- 如果修改了 Obsidian 链接结构或规范模板，更新 [[研究库健康检查]]。
- 收尾前至少根据改动范围运行：基础健康检查、实验 schema 检查、规范 schema 检查。

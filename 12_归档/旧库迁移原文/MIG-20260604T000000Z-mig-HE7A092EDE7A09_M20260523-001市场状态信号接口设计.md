# M20260523-001 市场状态信号接口设计

## 设计原则

接口要做到三件事：

- 可单独测试，不依赖回测引擎。
- 可审计数据时点，不产生未来函数。
- 可扩展到 R010-B/C/D，但 R010-A 只输出日志。

## 数据结构

### `MarketStateInput`

```text
MarketStateInput:
  trade_date: date
  signal_date: date
  etf_universe: list[str]
  broad_proxy_symbol: str
  daily_bars: DataFrame
  ranking_snapshot: DataFrame
  previous_state: MarketStateMemory
```

字段要求：

| 字段 | 说明 |
| --- | --- |
| `trade_date` | 计划交易日 |
| `signal_date` | 信号截止日，默认上一交易日 |
| `etf_universe` | 信号日可见 ETF 池 |
| `broad_proxy_symbol` | 宽基代理 |
| `daily_bars` | 截止 `signal_date` 的日线 |
| `ranking_snapshot` | 截止 `signal_date` 的策略排序快照 |
| `previous_state` | 前一交易日状态记忆 |

### `MarketStateSignals`

```text
MarketStateSignals:
  trend_breadth: float
  median_ret20: float
  broad_ret20: float
  top1_abs_mom: float
  top5_power_count: int
  top1_top5_gap: float
  dispersion20: float
  realized_vol20: float
  crash_rebound_score: float
  valid_count: int
```

### `MarketStateDecision`

```text
MarketStateDecision:
  raw_state: str
  confirmed_state: str
  suggested_mode: str
  confidence: float
  reason: str
  signal_date: date
  diagnostics: dict
```

## 函数接口

```text
build_market_state_input(context, ranking_snapshot, config) -> MarketStateInput
compute_market_state_signals(input, config) -> MarketStateSignals
classify_raw_state(signals, config) -> RawStateResult
confirm_market_state(raw_state, previous_state, config) -> ConfirmedStateResult
build_market_state_log(input, signals, decision) -> dict
```

## 信号函数定义

| 函数 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- |
| `calc_trend_breadth` | ETF池日线 | float | `close > MA20 > MA60` 占比 |
| `calc_median_ret20` | ETF池日线 | float | ETF池20日收益中位数 |
| `calc_broad_ret20` | 宽基日线 | float | 宽基20日收益 |
| `calc_top_abs_momentum` | 排序前5和日线 | float | Top1绝对动量 |
| `calc_top5_power_count` | 排序前5和日线 | int | 满足绝对动量的候选数量 |
| `calc_top1_top5_gap` | 排序快照 | float | Top1分数减Top5中位分数 |
| `calc_dispersion20` | ETF池日线 | float | ETF池20日收益截面离散度 |
| `calc_realized_vol20` | 宽基或ETF池等权收益 | float | 20日实现波动率 |
| `calc_crash_rebound_score` | 宽基日线 | float | 急跌后急反弹风险评分 |

## 配置草案

```text
market_state:
  enabled: true
  log_only: true
  signal_lag_days: 1
  min_history_days: 120
  confirm_days: 2
  crash_rebound_enter_days: 1
  crash_rebound_exit_days: 2
  windows:
    short: 20
    mid: 60
    long: 120
    rebound_short: 5
  thresholds:
    breadth_high: 0.65
    breadth_low: 0.35
    median_ret20_positive: 0.0
    top_abs_mom_positive: 0.0
    top5_power_min: 2
    dispersion_low_quantile: 0.30
    dispersion_high_quantile: 0.70
    vol_high_quantile: 0.80
```

阈值第一阶段只作为默认草案。真正进入回测前，必须在研究记录中固定训练期和样本外区间。

## 状态判定伪代码

```text
if crash_rebound_score >= crash_rebound_threshold:
    raw_state = "crash_rebound"
elif median_ret20 < 0 and top5_power_count == 0:
    raw_state = "weak_no_leader"
elif trend_breadth >= breadth_high and median_ret20 > 0 and broad_ret20 > 0:
    raw_state = "broad_uptrend"
elif dispersion20_is_high and top1_abs_mom > 0 and top5_power_count >= 1:
    raw_state = "narrow_leader"
elif dispersion20_is_mid_or_high and top5_power_count >= 2:
    raw_state = "rotational_momentum"
elif dispersion20_is_low and abs(top1_top5_gap) <= gap_low_threshold:
    raw_state = "low_dispersion_chop"
else:
    raw_state = "neutral_observe"
```

## 离线测试场景

| 场景 | 构造数据特征 | 预期状态 |
| --- | --- | --- |
| 全面上涨 | 多数 ETF 上穿 MA20/MA60，中位收益为正 | `broad_uptrend` |
| 窄主线 | 少数 ETF 大涨，离散度高，广度一般 | `narrow_leader` |
| 轮动市 | 多个 Top5 有正动量，离散度中高 | `rotational_momentum` |
| 低离散震荡 | ETF 收益接近，Top1-Top5 分差小 | `low_dispersion_chop` |
| 弱势无主线 | 中位收益为负，Top5 全弱 | `weak_no_leader` |
| 急跌反弹 | 先大跌后短期急反弹，波动率高 | `crash_rebound` |

## 实现边界

R010-A 的函数可以先作为独立模块实现，但不能在平台升级期间接入交易链路。接入前必须完成 [[未来函数审计清单]]。

# 实验表格管理规范

实验结果采用三层表格管理：`Bases 台账`、`Markdown 关键对比表`、`CSV/XLSX 大型结果表`。

## 第一层：Bases 台账

每篇研究记录必须填写 YAML Properties。当前聚合入口以 [[📊 驾驶舱]]、[[研究资产清单]]、方向页反向链接和平台资产清单为准；不再依赖旧台账入口。

必填字段：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `type` | 文档类型 | `实验` |
| `research_id` | 研究编号 | `R20260521-001` |
| `direction_id` | 研究方向编号 | `R010` |
| `topic_id` | 研究主题编号 | `R010B4` |
| `result_status` | 结果资产状态 | `completed_with_result` |
| `related_strategies` | 相关策略 | `[[ETF双池动量轮动]]` |
| `source_tags` | 收益来源标签 | `[[趋势跟踪类策略]]` |
| `execution_tags` | 执行方式标签 | `[[ETF轮动]]` |
| `status` | 研究状态 | `草稿`、`观察中`、`已采纳`、`已废弃` |
| `decision` | 当前判断 | `kill`、`revise`、`promote`、`park`、`observe` |
| `decision_ids` | 关联决策卡片 | `D20260601-R010B4-tiered-v2四段验证通过` |
| `baseline_return` | 基准总收益，小数口径 | `0.8746` |
| `test_return` | 实验总收益，小数口径 | `0.9210` |
| `baseline_mdd` | 基准最大回撤，小数口径 | `-0.1928` |
| `test_mdd` | 实验最大回撤，小数口径 | `-0.1680` |
| `config_path` | 配置目录或文件 | `configs/research/R20260521-001/` |
| `result_path` | 结果目录 | `results/v2/research/R20260521-001/` |

## 第二层：Markdown 关键对比表

单篇研究记录只放关键对比，避免长表污染阅读。

| 版本 | 年化收益 | 总收益 | 最大回撤 | 夏普 | 换手率 | 结论 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| 基准版 | 待填 | 待填 | 待填 | 待填 | 待填 | 对照 |
| 实验版 | 待填 | 待填 | 待填 | 待填 | 待填 | 待判断 |

## 第三层：大型结果表

参数扫描、逐日净值、交易明细、因子 IC、分层收益等大表不写入 Obsidian 正文。

- CSV/XLSX 放在平台结果目录，新增实验建议使用 `results/v2/research/<direction_id>/<research_id>_<topic_id>/`。
- 研究记录只引用路径，并摘录关键结论。
- 需要透视表或图表时，使用项目已有 spreadsheet 能力处理，再把摘要写回研究记录。

## 口径要求

- 收益和回撤在 YAML 中统一使用小数口径，例如 `87.46%` 写作 `0.8746`。
- Markdown 表格可以用百分数展示，便于阅读。
- 同一研究下所有 run 必须使用同一资金、费率和样本区间，除非该研究本身就是测试这些变量。
- 没有基准对照的实验不得标记为 `已采纳`。
- 没有样本外或滚动验证的实验最多标记为 `观察中`。

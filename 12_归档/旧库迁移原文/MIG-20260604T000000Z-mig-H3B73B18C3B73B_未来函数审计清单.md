# 未来函数审计清单

本清单用于所有因子、状态识别、风控和组合实验。R010-A 必须逐项满足。

## 基础规则

| 项目 | 要求 | 通过标准 |
| --- | --- | --- |
| 信号日 | 明确 `signal_date` | 日志中可看到信号截止日 |
| 交易日 | 明确 `trade_date` | `signal_date` 不晚于交易可见时点 |
| 日线数据 | T 日交易前只能用 T-1 及以前数据 | 查询或缓存必须按 `signal_date` 截断 |
| 成分池 | 只能使用信号日可见标的 | 不能用未来上市 ETF 或未来股票池 |
| 排序结果 | 排序快照必须来自信号日 | 不能用交易后收益重排 |
| 阈值 | 使用训练期固定阈值或滚动阈值 | 禁止全样本分位数 |
| 标签 | 状态标签不能由未来收益定义 | 只能由当时可见信号定义 |
| 外部数据 | 记录发布时间和修订规则 | 无发布时间的数据不能交易使用 |

## 状态识别专项

| 信号 | 主要风险 | 审计要求 |
| --- | --- | --- |
| ETF池趋势广度 | 使用 T 日收盘判断 T 日状态 | 默认只用 T-1 收盘 |
| ETF池中位20日收益 | 未来成分池泄漏 | 成分池按信号日截断 |
| Top1绝对动量 | 排序和绝对动量时点不一致 | 排序和动量都用同一 `signal_date` |
| Top5动力确认 | 事后挑选表现好的候选 | 只能检查当时 Top5 |
| 横截面离散度 | 全样本分位数泄漏 | 使用滚动分位或固定阈值 |
| 实现波动率 | 使用未来窗口波动率 | 只使用过去窗口 |
| 急跌后急反弹 | 事后确认反弹成功 | 只用已发生的短期反弹，不用后续收益 |
| HMM状态概率 | 使用平滑概率 | 只能使用过滤概率 |

## 代码审计问题

实现时逐项回答：

- 函数是否显式接收 `signal_date`。
- 所有历史数据查询是否传入 `end_date=signal_date`。
- 是否存在 `date.today()` 参与历史数据查询。
- 是否存在全量历史数据后再按结果切片。
- 是否存在用完整样本计算分位数、均值或标准差。
- 是否存在用未来收益创建状态标签。
- 单元测试是否覆盖信号日和交易日错位。

## 研究记录要求

每篇涉及交易信号的研究记录都应写明：

- 信号生成时点。
- 数据截止时点。
- 是否使用滚动阈值。
- 样本内和样本外如何划分。
- 是否有未来函数风险残留。

# 研究台账

本页使用 Obsidian 原生 Bases 汇总 `05-实验记录` 下的研究记录。新增研究记录时，只要按 [[研究记录模板]] 填写 YAML Properties，就会自动进入下方表格。

```base
filters:
  and:
    - file.inFolder("05-实验记录")
    - type == "研究记录"
formulas:
  return_delta: if(test_return, (test_return - baseline_return).toFixed(4), "")
  mdd_delta: if(test_mdd, (test_mdd - baseline_mdd).toFixed(4), "")
properties:
  file.name:
    displayName: 文件
  research_id:
    displayName: 研究编号
  target_strategy:
    displayName: 目标策略
  source_tags:
    displayName: 收益来源
  execution_tags:
    displayName: 执行方式
  status:
    displayName: 状态
  decision:
    displayName: 决策
  baseline_return:
    displayName: 基准收益
  test_return:
    displayName: 实验收益
  formula.return_delta:
    displayName: 收益差
  baseline_mdd:
    displayName: 基准回撤
  test_mdd:
    displayName: 实验回撤
  formula.mdd_delta:
    displayName: 回撤差
  config_path:
    displayName: 配置路径
  result_path:
    displayName: 结果路径
views:
  - type: table
    name: 全部研究
    order:
      - file.name
      - research_id
      - target_strategy
      - source_tags
      - execution_tags
      - status
      - decision
      - baseline_return
      - test_return
      - formula.return_delta
      - baseline_mdd
      - test_mdd
      - formula.mdd_delta
      - config_path
      - result_path
    columnSize:
      file.name: 530
  - type: table
    name: 观察中
    filters:
      and:
        - status == "观察中"
    order:
      - file.name
      - research_id
      - target_strategy
      - source_tags
      - execution_tags
      - decision
      - formula.return_delta
      - formula.mdd_delta
  - type: table
    name: 已采纳
    filters:
      and:
        - status == "已采纳"
    order:
      - file.name
      - research_id
      - target_strategy
      - source_tags
      - execution_tags
      - decision
      - formula.return_delta
      - formula.mdd_delta

```

## 维护规则

- 本页不手工维护表格行，数据来自研究记录的 YAML Properties。
- 若某篇研究没有出现在表格中，优先检查它是否位于 `05-实验记录`，以及 `type` 是否为 `研究记录`。
- 若数值列无法计算，检查收益和回撤是否使用小数口径。

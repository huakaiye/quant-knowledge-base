# 涨停事件研究方法库

本方法库定义[[涨停事件]]类策略的研究顺序。核心原则是：先建立事件基准，再验证价量结构增量，最后加入执行容量约束；不允许直接复刻社区文章中的高收益窗口。

## 研究顺序

| 阶段 | 名称 | 是否交易 | 目的 |
| --- | --- | --- | --- |
| S1 | 涨停事件基准 | 否 | 检查全部非一字涨停 T+1 买入、持有 2 日是否有正期望 |
| S2 | 交易约束基准 | 否 | 加入 ST、停牌、一字板、涨跌停不可成交和基础滑点 |
| S3 | 单结构因子 | 否 | 分别验证强度、缩量、波动收敛、突破等单因子 |
| S4 | 多结构组合 | 否 | 检查组合因子是否稳健，而不是历史拟合 |
| S5 | 成交容量 | 否 | 加集合竞价或开盘成交容量约束 |
| S6 | 小资金仿真 | 是，观察 | 只有通过样本外和容量审计后才进入观察 |

## 默认事件定义

| 项目 | 初始口径 |
| --- | --- |
| 事件日 | T 日收盘价触及涨停价 |
| 买入日 | T+1，最早在集合竞价或开盘后执行 |
| 持有期 | 默认 2 个交易日，可做 1/2/3 日敏感性 |
| 剔除 | ST、停牌、上市过短、一字涨停、极低成交额 |
| 收益口径 | 成本前、成本后、容量约束后三组同时输出 |

## 因子验证表

| 因子 | 单独验证 | 与基准比较 | 证伪观察 |
| --- | --- | --- | --- |
| 涨停前强度 | 是 | 是否提高均值收益 | 若只提高波动不提高收益则降级 |
| 缩量整理 | 是 | 是否降低左尾 | 若样本外方向反转则废弃 |
| 波动收敛 | 是 | 是否降低回撤 | 若只在少数年份有效则观察 |
| 涨停突破 | 是 | 是否改善胜率 | 若与涨停事件完全重合则无增量 |
| 流动性过滤 | 是 | 是否提高可成交性 | 若收益完全来自低流动性票则谨慎 |

## 必做对照

| 对照 | 目的 |
| --- | --- |
| 全部非一字涨停 | 判断事件本身是否有正期望 |
| 随机涨停样本 | 判断形态是否优于随机挑选 |
| 同行业同市值匹配 | 控制行业和市值暴露 |
| 延迟一日买入 | 判断收益是否依赖难成交的开盘优势 |
| 去极端收益样本 | 判断结果是否由少数票驱动 |
| 年份滚动样本外 | 判断是否过拟合 |

## 未来函数审计

| 风险点 | 审计要求 |
| --- | --- |
| 涨停判定 | T 日收盘后才可用于 T+1 信号 |
| 价量窗口 | 只能使用 T 日及以前数据，不能用 T+1 高低价定义形态 |
| 涨跌停成交 | 一字涨停不可买，一字跌停不可卖 |
| 集合竞价成交 | 不能默认无限按开盘价成交 |
| 样本筛选 | 禁止用全样本历史胜率筛选窗口 |
| 参数扫描 | 必须记录搜索空间和失败组合 |

## 决策规则

| 观察 | 决策 |
| --- | --- |
| 事件基准无正期望，单结构无稳定增量 | kill |
| 单结构有增量但容量后消失 | park，作为形态研究保留 |
| 成本后有效但样本外弱 | revise，降低复杂度或改为风险提示 |
| 多结构组合有效但单结构无效 | 默认判定过拟合，除非有强机制和样本外证据 |
| 成本、容量、样本外均通过 | promote，建立正式研究记录和小资金观察方案 |

## 不允许的做法

- 不允许把社区文章给出的收益当作本平台结论。
- 不允许直接建立 20 个以上价量模板并用历史胜率筛选。
- 不允许忽略涨跌停不可成交。
- 不允许缺少容量约束就讨论实盘价值。
- 不允许只记录成功窗口，不记录失败窗口。

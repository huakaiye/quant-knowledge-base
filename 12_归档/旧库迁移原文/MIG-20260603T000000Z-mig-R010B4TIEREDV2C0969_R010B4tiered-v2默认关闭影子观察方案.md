# R010B4 tiered-v2默认关闭影子观察方案

## 一句话结论

R010-B4 tiered-v2 已完成默认关闭映射，并补齐影子观察开关；本页不是开启实盘交易指令。当前只允许在用户明确授权的模拟盘、小资金或演练环境中记录 tiered-v2 候选等级，不改变实际风险仓上限。

## 当前边界

| 项目 | 当前规则 |
| --- | --- |
| 默认状态 | `ENABLE_B3_GATE_MOB_CAP80=0`、`ENABLE_B3_GATE_TIERED_V2=0`、`ENABLE_B3_GATE_TIERED_V2_SHADOW=0` |
| 主体候选 | 当前主体仍是 `A2-slope004 + B3-gate-median-or-breadth-cap80` |
| 影子观察前提 | 只有用户明确授权模拟盘、小资金或演练日志时，才允许显式打开 B3 gate 与影子观察 |
| 影子观察推荐组合 | `ENABLE_B3_GATE_MOB_CAP80=1`、`ENABLE_B3_GATE_TIERED_V2=0`、`ENABLE_B3_GATE_TIERED_V2_SHADOW=1` |
| 交易禁止项 | 不得把 `ENABLE_B3_GATE_TIERED_V2` 改为默认开启；不得绕过 B3 gate active 独立触发 tiered-v2 |
| 实际仓位边界 | 影子观察下 `tiered_candidate_cap` 可为 70% 或 60%，但 `tiered_effective_cap` 仍保持 B3 gate 的 80% |

## 规则口径

| 观察等级 | 判定口径 | 影子候选 |
| --- | --- | --- |
| normal | B3 gate active，但不满足 strong | `candidate_cap=80%` |
| strong | B3 gate active，且 `cum_days>=10`、`median_ret<=0`、`top5_exhausted>=4` | `candidate_cap=70%` |
| extreme | strong 成立，且 `median_ret<=-0.015 OR ma20_breadth<=0.25`，并满足 `top5_exhausted>=4` | `candidate_cap=60%` |

字段口径以 MiniQMT 日志为准：

```text
cum_days = r010b3_all_weak_cum_days
median_ret = r010b3_all_weak_persistence_gate_median_ret
ma20_breadth = r010b3_all_weak_persistence_gate_ma20_breadth
top5_exhausted = r010b3_top5_filter_exhausted_count
```

## 必看日志与状态字段

| 来源 | 字段或日志片段 | 用途 |
| --- | --- | --- |
| 启动日志 | `[启动] B3-gate-tiered-v2:` | 复核交易开关和影子观察开关是否符合预期 |
| 启动日志 | `shadow=开/关` | 确认是否处在只观察模式 |
| B3 gate 日志 | `[B3-gate] 持久状态` | 复核 raw、active、cum、consec、quiet、gate_pass、median、breadth |
| 触发日志 | `tiered_enabled=` | 必须为 `False` 才是影子观察，不是交易启用 |
| 触发日志 | `tiered_shadow=` | 记录是否执行 tiered-v2 候选评估 |
| 触发日志 | `tiered_level=` | 区分 `normal`、`strong`、`extreme` |
| 触发日志 | `tiered_candidate_cap=` | 记录 tiered-v2 理论候选仓位 |
| 触发日志 | `tiered_effective_cap=` | 影子观察下必须仍为 80%，否则先 `revise` 映射 |
| 触发日志 | `tiered_reason=` | 解释分层由 cum、median、breadth、top5 exhausted 中哪些条件驱动 |

## 人工复核字段

每次出现 B3 gate active 时记录摘要；完整日志和明细放在平台结果目录，不放入研究库。

| 字段 | 说明 |
| --- | --- |
| `observe_date` | 观察交易日 |
| `event_type` | `shadow_normal_cap80` / `shadow_strong_cap70` / `shadow_extreme_cap60` |
| `tiered_enabled` | 影子观察阶段必须为 `False` |
| `tiered_shadow` | 观察阶段应为 `True` |
| `tiered_level` | `normal`、`strong` 或 `extreme` |
| `tiered_candidate_cap` | tiered-v2 理论候选风险仓 |
| `tiered_effective_cap` | 实际生效风险仓；影子观察必须保持 80% |
| `cum_days` / `median_ret` / `ma20_breadth` / `top5_exhausted` | 分层解释变量 |
| `target_weights` | 最终目标权重，确认未因 shadow 降到 70% 或 60% |
| `next_1d` / `next_5d` / `next_10d` | 离线复盘时补充后验表现，用于评估误伤和漏报 |
| `manual_judgement` | 有效防守、误伤趋势、冲击保护、无法判断 |

## 复核问题

| 问题 | 判断依据 |
| --- | --- |
| strong/extreme 是否集中出现在真实风险扩张日 | 对照当日组合回撤、后续 5/10 日净值和持仓收益 |
| `candidate_cap=60%` 是否错过快速修复 | 观察原风险资产后续 1/5/10 日收益是否显著为正 |
| shadow 是否完全不改仓位 | `tiered_enabled=False` 且 `tiered_effective_cap=80%` |
| normal/strong/extreme 是否可人工解释 | `tiered_reason` 是否能回溯到当日可得字段 |
| 是否仍有 B3 gate 漏报 | 对照 [[R010B3_B3-gate默认关闭观察方案]] 中的 non raw future dd5 清单 |

## 升级与停止规则

| 结果 | 决策动作 |
| --- | --- |
| 日志缺失、状态跨日不连续、`tiered_effective_cap` 与影子边界不一致 | `revise`，先修映射和日志 |
| 影子 strong/extreme 多次发生在趋势延续并明显误伤 | `revise` 或 `kill` tiered-v2 实盘候选 |
| 影子 strong/extreme 样本过少，无法判断 | 继续 `observe`，不得用少量样本 promote |
| 至少 20 个 B3 gate active 观察日且至少 5 个 strong/extreme 事件，误伤可解释且冲击保护仍成立 | 仍只能 `observe`，进入下一张决策卡片讨论模拟盘或小资金验证 |
| 任何人尝试把 `ENABLE_B3_GATE_TIERED_V2` 改成默认开启 | 立即 `revise` 研究边界并回退为默认关闭 |

# 术语词典

> **怎么用**：遇到不认识的缩写，Ctrl+F 搜一下。每条术语只有一行中文解释，读完就懂。详细定义点击链接跳转。

> **维护规则**：新术语首次在实验记录或决策卡片中出现时，同时追加到本词典。不要等"以后再说"。

## 一、研究方向编号

| 编号 | 含义 |
|------|------|
| **R + 日期 + 序号** | 实验记录编号。如 `R20260601-027` = 2026年6月1日第27号实验 |
| **D + 日期 + 主体** | 方向决策卡片编号。如 `D20260602-R010B5-...` = 研究路线改变的关键判断 |
| **BLF + 日期 + 序号** | 研究信念文件。如 `BLF20260523-001` = 一个研究方向的核心信念声明 |
| **M001-M007** | 多策略研究子编号。M = Multi-strategy |
| **R010** | 市场状态识别与分级防守研究方向。下含 A/B/C/D/E 子线 |
| **A 系列**（A0-A2） | ETF 双池进攻端组件 |
| **B 系列**（B0-B5） | ETF 双池防守端组件 |
| **C/D/E 系列** | R010 其他子线（均已证伪或降级） |

## 二、ETF 双池动量轮动 —— 策略组件

| 缩写                                   | 一句话解释                                                          |
| ------------------------------------ | -------------------------------------------------------------- |
| **A0**                               | 基础进攻：原始动量轮动，无任何防守干预                                            |
| **A2-slope004**                      | 当前进攻底座：拥挤过热时保留有动量的旧仓，减少追高换仓。`slope004` 是触发阈值                   |
| **B3 全弱信号**                          | 池内 Top1-Top5 ETF 的短/中/长动量全线走弱时发出的原始风险信号                        |
| **B3 gate**                          | B3 信号 + 二次确认门控（`中位收益 ≤ 0` 或 `广度 ≤ 40%`），过滤假风险信号                |
| **B3-cap80 / cap90 / cap70 / cap60** | 风险仓位上限。cap80=最多80%仓位在风险资产，以此类推。B3 gate 通过后默认 cap80             |
| **tiered-v2**                        | B3 gate 的强度分层版：普通风险 → cap80，强风险 → cap70，极端风险 → cap60。**当前实盘中** |
| **limited2**                         | 旧版试探方案：B3 触发后最多只降仓 2 天。已废弃                                     |
| **R026**                             | 防守池吸血信号研究——防守型 ETF 被涌入是否能提前预警。结论：不能做主开关，保留为只读标记                |

## 三、风险仓位（cap）数值

| 数值 | 含义 |
|------|------|
| **cap100** | 不限制，正常满仓 |
| **cap90** | 风险资产最多 90%（轻量防守） |
| **cap80** | 最多 80%（中等防守，B3 gate 默认档） |
| **cap70** | 最多 70%（强防守，tiered-v2 strong） |
| **cap60** | 最多 60%（极端防守，tiered-v2 extreme） |
| **floor 70%** | 非线性仓位的地板：最多降 30%，不全部清仓 |

## 四、市场状态诊断指标

| 术语 | 一句话解释 |
|------|-----------|
| **median_ret** | 候选 ETF 池的中位收益率。正值 = 大部分 ETF 在涨，负值 = 大部分在跌 |
| **ma20_breadth** | 池内有多少比例的 ETF 价格在 20 日均线之上。低于 40% = 广度衰竭 |
| **ret20_breadth** | 池内 20 日正收益 ETF 的比例 |
| **dispersion20** | ETF 之间的 20 日收益差异有多大。高 = 选对了有超额，低 = 买谁都差不多 |
| **cum_days** | B3 全弱信号的累计出现天数 |
| **top5_exhausted** | Top5 ETF 中动量衰竭的数量。≥4 个衰竭 = 强势 ETF 也在变弱 |
| **top1_slope10** | 排第一的 ETF 近 10 日价格斜率。接近零或负 = 最强 ETF 也在失速 |
| **avg_r2** | 趋势的拟合优度。高 = 涨得稳，低 = 乱涨乱跌 |
| **sigma_t20 / sigma_t60** | 组合 20 日/60 日年化波动率。用于波动率缩放组件 |

## 五、非线性仓位控制（R010-B5 系列）

| 缩写                      | 一句话解释                                   |
| ----------------------- | --------------------------------------- |
| **risk_scale**          | 仓位缩放系数。1=不缩放，0.7=最多降到70%                |
| **drawdown_threshold**  | 触发缩放的回撤门槛。当前为 10%                       |
| **decay**               | 指数衰减速度。值越大降仓越快。当前为 5                    |
| **叠加版（overlay）**        | 非线性缩放挂在 tiered-v2 之后，两者协同               |
| **替代版（replacement）**    | 关掉 tiered-v2 只用非线性缩放。**已证实失败**          |
| **Counterfactual（反事实）** | 同日比较"如果用了非线性仓位"和"实际用了固定 cap"的净值差        |
| **R5**                  | 严格反事实审计节点——非线性仓位在此被整体否定了                |
| **非线性公式四组件**            | 波动率缩放 × 趋势过滤 × 回撤反馈 × 资金曲线状态。目前仅回撤反馈已实现 |

## 六、多策略 / 小市值系列

| 术语            | 一句话解释                                   |
| ------------- | --------------------------------------- |
| **M001-M004** | 多策略输入准备系列：从"能不能做"到"小市值四段回测通过"           |
| **M005-M007** | 小盘专用状态标签系列：从"定义标签"到"面板归因"。当前卡在标签修订      |
| **小市值 T0**    | 小市值股票日内做T策略。非 ETF 策略，四段回测 +1705%/-16.7% |
| **状态归因**      | 不改交易，只分析"不同市场状态下策略表现差异"。纯诊断用途           |

## 七、研究方法术语

| 术语 | 一句话解释 |
|------|-----------|
| **预注册** | 实验前先写清楚假设、证伪条件、验收标准——禁止先看结果再编故事 |
| **消融** | 逐个移除模块看每个模块贡献多少的实验方法 |
| **Shadow-only** | 只记录候选 cap 建议不改变实际交易——做实验的"安全模式" |
| **Smoke** | 先在一个时间段快速跑一次，通过再四段长测。不替代四段 |
| **四段** | 2020-2021 / 2022-2023 / 2024 / 2025-20260519 四个历史窗口的全量压力测试 |
| **未来函数** | 在信号生成时使用了当时不存在的数据（比如今天用明天的收盘价）。致命错误 |
| **后验** | 先看了结果再挑参数——高过拟合风险。和预注册对立 |
| **OOS** | Out-of-Sample，样本外验证。用没见过的新数据验证 |
| **外部数据** | 平台行情、财务和回测结果之外的数据来源。使用前须确认发布时间、修订规则和可复现性 |
| **实验治理** | 管理样本内外、消融实验、参数扫描、多重检验和研究记录质量，防止研究过程退化为参数挖掘 |

## 八、决策术语

| 术语 | 含义 |
|------|------|
| **✅ promote** | 晋级：通过验证，进入下一阶段 |
| **❌ kill** | 杀死：永久终止，禁止恢复 |
| **🔧 revise** | 修订：核心思路保留，实现方式需改 |
| **📌 park** | 暂停：等待新数据或新条件，不主动推进 |
| **👀 observe** | 观察：只记录不交易，纯诊断用途 |
| **no_live** | 禁止实盘。出现在决策标签尾部 |

# R010-A 市场状态识别规格书

## 目标

R010-A 只做市场状态识别和日志记录，不改变交易、不改变持仓、不改变排序、不改变仓位。

它的目标是先回答三个问题：

- 当前 ETF 双池动量轮动处在什么市场环境。
- 当前环境更适合原始动量、R9 平滑软排序、低换手保留，还是防守。
- 这些状态判断是否稳定、可解释、无未来函数。

## 非目标

- 不做参数扫描。
- 不做收益优化。
- 不引入 HMM、机器学习分类器或外部宏观数据。
- 不直接使用状态结果控制交易。
- 不根据未来收益给历史日期打状态标签。

## 信号时点

| 场景 | 信号可用数据 | 说明 |
| --- | --- | --- |
| T 日开盘前决策 | T-1 及以前日线数据 | 默认口径 |
| T 日盘中决策 | T 日盘中可见价格和成交数据 | 必须在日志中标记为盘中信号 |
| T 日收盘后复盘 | T 日收盘数据 | 只能用于 T+1 或之后，不能用于 T 日交易 |

R010-A 默认使用第一种：T 日交易前，只用 T-1 及以前数据生成状态。

## 输入数据契约

| 输入 | 字段 | 要求 |
| --- | --- | --- |
| ETF 日线 | `date, symbol, close, high, low, volume, amount` | 至少覆盖 120 个交易日 |
| ETF 池 | `symbol, start_date, end_date` | 成分必须在信号日可见，不能使用未来上市 ETF |
| 宽基代理 | `symbol` | 优先使用沪深300ETF或主要宽基指数 ETF |
| 策略排序结果 | `symbol, rank, score` | 使用当日策略同口径排序结果 |
| 交易日历 | `date` | 用于确定 T-1 和滚动窗口 |

## 核心信号

| 信号 | 计算口径 | 作用 |
| --- | --- | --- |
| ETF池趋势广度 | ETF 池中 `close > MA20 > MA60` 的比例 | 判断全面上涨或弱势 |
| ETF池中位20日收益 | ETF 池过去 20 日收益中位数 | 判断整体方向 |
| 宽基20日收益 | 宽基代理过去 20 日收益 | 判断大盘背景 |
| Top1绝对动量 | 当前 Top1 的 20/60 日收益或回归斜率 | 判断最强者是否真的强 |
| Top5动力数量 | Top1 到 Top5 中满足绝对动量的数量 | Top1 衰竭时继续检查候选 |
| Top1-Top5分差 | Top1 分数减 Top5 中位分数 | 判断排名优势是否显著 |
| 横截面离散度 | ETF 池 20 日收益截面标准差或四分位差 | 判断轮动是否有选标价值 |
| 实现波动率 | 宽基或 ETF 池等权 20 日波动率 | 控制风险状态置信度 |
| 急跌后急反弹 | 过去 20/60 日下跌后 3/5 日快速反弹且波动高 | 识别动量崩盘风险 |
| 全池20日正收益广度 | ETF 池中 20 日收益为正的比例 | 判断池内是否仍有可追动量 |
| 全池MA20广度 | ETF 池中 `close > MA20` 的比例 | 判断短中期趋势是否健康 |
| 全池MA20/MA60广度 | ETF 池中 `MA20 > MA60` 的比例 | 判断中期趋势背景 |
| Top1到Top5近端收益 | 实际候选 Top1 到 Top5 的 20日/5日收益、10日短斜率、R²、得分 | 判断最强者是否短期衰竭 |

## 状态定义

| 状态 | 条件草案 | 行为解释 |
| --- | --- | --- |
| `broad_uptrend` | 广度高、ETF池中位20日收益为正、宽基20日收益为正 | 全面上涨，原始动量大概率优于过度平滑 |
| `narrow_leader` | 离散度高、Top1强、Top5至少有1个强、广度不必高 | 窄主线，动量有效但需防尾部反转 |
| `rotational_momentum` | 离散度中高、Top5动力数量不少于2、Top1分差不极端 | 轮动有价值，适合软排序或TopN分散 |
| `low_dispersion_chop` | 离散度低、Top1-Top5分差小、ETF池中位收益不强 | 排名噪声高，后续可考虑低换手 |
| `weak_no_leader` | ETF池中位20日收益为负、Top5动力数量为0 | 弱势无主线，后续可考虑防守 |
| `crash_rebound` | 前期急跌、短期急反弹、波动率高 | 动量崩盘风险，旧赢家可能反杀 |
| `neutral_observe` | 以上状态不明确 | 只观察，不做强判断 |

## 状态优先级

状态判断按风险优先：

```text
crash_rebound
weak_no_leader
broad_uptrend
narrow_leader
rotational_momentum
low_dispersion_chop
neutral_observe
```

风险状态优先于进攻状态。若同一天同时满足 `crash_rebound` 和 `broad_uptrend`，记录为 `crash_rebound`，同时在日志中保留宽基转强信号。

## 状态切换确认

R010-A 只记录原始状态和确认状态：

| 字段 | 含义 |
| --- | --- |
| `raw_state` | 当日信号直接判出的状态 |
| `confirmed_state` | 经过连续确认后的状态 |
| `days_in_state` | 当前确认状态持续天数 |
| `pending_state` | 正在观察但尚未确认的新状态 |
| `pending_days` | 候选状态连续出现天数 |

默认确认规则：

- 常规状态连续 2 个交易日出现后确认。
- `crash_rebound` 可单日进入，但退出需要连续 2 日不满足。
- `weak_no_leader` 连续 2 日进入，退出需要 Top5 动力恢复。

## 日志输出

R010-A 每个交易日输出一行状态日志。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `trade_date` | date | 计划交易日 |
| `signal_date` | date | 信号截止日，默认 T-1 |
| `raw_state` | string | 原始状态 |
| `confirmed_state` | string | 确认状态 |
| `suggested_mode` | string | 建议模式，仅记录不交易 |
| `confidence` | float | 状态置信度 |
| `trend_breadth` | float | ETF池趋势广度 |
| `median_ret20` | float | ETF池中位20日收益 |
| `broad_ret20` | float | 宽基20日收益 |
| `top1_abs_mom` | float | Top1绝对动量 |
| `top5_power_count` | int | Top5动力数量 |
| `top1_top5_gap` | float | Top1与Top5分差 |
| `dispersion20` | float | 20日横截面离散度 |
| `realized_vol20` | float | 20日实现波动率 |
| `crash_rebound_score` | float | 急跌后急反弹风险评分 |
| `ret20_breadth` | float | 全池20日正收益广度 |
| `ma20_breadth` | float | 全池 `close > MA20` 广度 |
| `ma20_ma60_breadth` | float | 全池 `MA20 > MA60` 广度 |
| `top1_code` 到 `top5_code` | string | 实际候选前五代码 |
| `top1_ret20/top1_ret5/top1_slope10/top1_r2/top1_score` | number | Top1 细分动量指标，Top2 到 Top5 同口径输出 |
| `full_pool_top5` | list | 全池按20日收益排序的前五明细，用于区分交易候选和全池最强资产 |
| `reason` | string | 状态原因 |

## 建议模式映射

R010-A 只写日志，不执行建议。

| 状态 | `suggested_mode` |
| --- | --- |
| `broad_uptrend` | `raw_momentum` |
| `narrow_leader` | `raw_momentum_light_quality` |
| `rotational_momentum` | `smooth_soft_rank` |
| `low_dispersion_chop` | `low_turnover_hold` |
| `weak_no_leader` | `defensive` |
| `crash_rebound` | `reduced_exposure` |
| `neutral_observe` | `observe` |

## 验收标准

平台恢复后，R010-A 先跑日志，不看收益优化，只验收：

- 每个交易日都有状态日志。
- `signal_date` 永远早于或等于可用信号日。
- 状态切换次数不过度频繁。
- 弱势无主线、急跌后急反弹、宽基共振上涨能被直观看到。
- 日志能解释 [[R20260522-010_ETF双池轮动适用市场环境归因]] 中的主要市场环境。

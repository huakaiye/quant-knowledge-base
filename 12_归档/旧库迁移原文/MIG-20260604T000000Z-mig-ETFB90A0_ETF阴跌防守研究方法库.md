# ETF阴跌防守研究方法库

本方法库定义 ETF 动量轮动中[[阴跌状态]]和防守切换的验证流程。核心原则是：先证明阴跌状态能解释策略失效，再比较防守动作；不直接把社区参数接入现有策略。

## 研究顺序

| 阶段 | 名称 | 是否交易 | 目的 |
| --- | --- | --- | --- |
| S1 | 阴跌状态日志 | 否 | 逐日记录是否满足阴跌条件 |
| S2 | 状态内收益归因 | 否 | 检查 ETF 动量在阴跌状态内是否显著变差 |
| S3 | 单信号消融 | 否 | 分离 MA20、20 日回撤、20 日收益的贡献 |
| S4 | 防守动作对照 | 是，回测 | 比较货币 ETF、国债 ETF、半仓、保留旧仓 |
| S5 | 参数稳健性 | 否 | 检查阈值和最短防守期是否稳定 |
| S6 | 样本外和执行审计 | 是，观察 | 加成本、溢价、分钟执行、延迟交易 |

## 默认状态变量

| 变量 | 说明 |
| --- | --- |
| `risk_benchmark_close` | 风险基准收盘价 |
| `risk_benchmark_ma20` | 风险基准 20 日均线 |
| `risk_benchmark_ret20` | 风险基准 20 日收益 |
| `risk_benchmark_dd20` | 风险基准相对 20 日高点回撤 |
| `below_ma20` | 是否位于 MA20 下方 |
| `bear_slowdown_state` | 是否进入阴跌状态 |
| `bear_state_days` | 阴跌状态持续天数 |
| `exit_rebound_from_low20` | 从 20 日低点反弹幅度 |

## 必做对照

| 对照 | 目的 |
| --- | --- |
| 原策略不防守 | 基准 |
| 只用 MA20 下方防守 | 判断均线破位是否过宽 |
| 只用 20 日回撤防守 | 判断回撤信号贡献 |
| 只用 20 日收益防守 | 判断负动量信号贡献 |
| MA20 + 回撤或收益 | 复核社区规则 |
| 延迟一日执行 | 检查时点稳健性 |
| 不同防守资产 | 检查收益改善是否来自资产选择 |

## 参数扫描边界

| 参数 | 候选范围 |
| --- | --- |
| 回撤进入阈值 | 8%、10%、12%、15% |
| 20 日收益进入阈值 | -5%、-8%、-10%、-12% |
| 最短防守期 | 3、5、10 个交易日 |
| 退出低点反弹 | 3%、4%、5%、8% |
| 退出均线要求 | 需要站回 MA20 / 不需要 |

参数扫描只能用于稳健性检查。若只有单一参数组合有效，不升级为交易模块。

## 未来函数审计

| 风险点 | 审计要求 |
| --- | --- |
| 盘中触发 | 明确 13:04 使用哪些已发生数据 |
| 日线数据 | 若用当日未收盘数据，需要改名为盘中快照，不得伪装成收盘信号 |
| 调仓顺序 | 状态信号必须早于卖出和买入 |
| ETF池 | 历史 ETF 池必须使用当时可交易列表或明确固定池偏差 |
| 溢价率 | 净值发布时间和缺失处理必须审计 |
| 成交成本 | 单仓切换必须加滑点和冲击成本复核 |

## 决策规则

| 观察 | 决策 |
| --- | --- |
| 阴跌状态不能解释 ETF 动量失效 | kill |
| 只在 2021 有效 | park，保留为慢跌案例 |
| 能降回撤但收益损失过大 | revise，改为半仓或风险预算降权 |
| 延迟一日后仍能降低回撤 | promote 到状态防守实验 |
| 多个相邻参数方向一致 | 提高信念置信度 |

## 不允许的做法

- 不允许直接复制社区参数作为生产规则。
- 不允许只看总收益，不看阴跌状态内归因。
- 不允许忽略 ETF 固定池和溢价过滤差异。
- 不允许用全样本最优参数解释未来。
- 不允许在没有成本和延迟测试时采纳防守切换。

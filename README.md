# 量化研究库 V2.0.0

本库是团队量化研究的中文事实源，负责保存研究方向、研究方法、文献灵感、实验记录、决策卡片、台账、术语和交接信息。

回测平台位于：

```text
E:\量化平台_V1.4.0
```

平台负责策略代码、回测配置、数据、结果和质量门禁；本研究库负责解释“为什么做、怎么验证、结果说明了什么、下一步为什么这样走”。

## 5 分钟接手

新 Agent 或新同事接手时按顺序阅读：

1. `AGENTS.md`
2. `00_入口/研究驾驶舱.md`
3. `00_入口/当前状态.md`
4. `08_方法论/命名与编号规范.md`
5. `08_方法论/研究方法论.md`
6. `08_方法论/研究质量审计规范.md`
7. `08_方法论/平台协作规范.md`
8. `08_方法论/子代理调度规范.md`
9. `08_方法论/Obsidian使用规范.md`
10. `08_方法论/Obsidian双向链接规范.md`

读完后应能回答：

- 当前主研究方向是什么。
- 每个方向处于什么状态。
- 最新有效结论来自哪些实验和决策。
- 下一轮实验要减少哪一个不确定性。
- 需要到平台读取哪些配置和结果。

## 编码约定

本库文本文件统一优先使用 UTF-8。Markdown、CSV、JSON、Canvas、YAML、PowerShell 脚本和 Git 配置文本必须能被 UTF-8 正常解码。除非兼容 Windows PowerShell 5.1 等工具确有必要，新增文本文件优先使用 UTF-8 无 BOM。

## 编号原则

本库不使用全局顺序号，因为多人并行开发和 GitHub PR 合并时容易冲突。

统一 ID 格式：

```text
<类型前缀>-<UTC时间戳>-<agent标识>-<随机码>
```

示例：

```text
RD-20260605T115651Z-main-DP00_双池轮动策略.md
EX-20260605T115651Z-main-BASE_双池轮动基线资产盘点预注册.md
DEC-20260605T115651Z-main-SYS0_研究库V2架构立项通过.md
```

详细规则见 `08_方法论/命名与编号规范.md`。新建文件优先使用：

```powershell
powershell -ExecutionPolicy Bypass -File tools/New-ResearchItem.ps1 -Type Experiment -Title "实验标题" -ParentId "RD-..."
```

## 研究流程

每一轮正式实验都必须经历：

```text
文献或想法 -> 研究方向 -> 实验前假设 -> 回测或诊断 -> 实验记录 -> 研究决策 -> 方向页和驾驶舱更新
```

实验记录必须写给完全的新手看，至少解释：

- 我们原本相信什么。
- 这次实验想验证什么。
- 实验前预计会看到什么。
- 实际结果是什么。
- 为什么这个结果支持或反驳原假设。
- 下一步为什么这样走。

实验还必须完成未来函数、过拟合、样本外、成本扰动、消融或负控审计。详细规则见 `08_方法论/研究质量审计规范.md`。

## 与旧库关系

旧库路径：

```text
E:\【笔记库】\量化研究库
```

旧库有可迁移的数据、记录和经验，但架构不复用。迁移时只搬运经过复核的事实、路径、结果摘要和决策链，不复制旧目录结构。迁移规则见 `08_方法论/研究库迁移规范.md`。

## Obsidian

本库可以直接作为 Obsidian vault 打开，并已复用旧库的外观、插件和主题配置。私密插件配置、workspace、缓存和 token 不入库，规则见 `08_方法论/Obsidian使用规范.md`。

核心研究资产应充分使用 Obsidian 双向链接。双向链接用于人类导航和反向链接，ID 与台账用于结构化治理，规则见 `08_方法论/Obsidian双向链接规范.md`。

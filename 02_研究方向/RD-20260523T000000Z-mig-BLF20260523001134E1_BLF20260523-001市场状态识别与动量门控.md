# BLF20260523-001 市场状态识别与动量门控

## 信念陈述

ETF 双池动量轮动的收益质量依赖市场状态。用 T-1 及以前可见的趋势广度、横截面离散度、TopN 动量强度和风险状态，可以把“原始动量、平滑软排序、低换手保留、防守”分配到更合适的环境。

2026-06-02 更新：长周期证据支持“状态日志先于交易动作”的纪律。A2-slope004 已成为进攻增强底座，B3 gate 与 tiered-v2 证明池内状态可以形成防守候选；但 R020、A2二级、R010C1 等标签也证明许多状态描述只有诊断价值，不能直接交易化。

## 机制解释

动量策略需要两个条件：有足够趋势延续，以及标的之间有足够可交易差异。全面上涨时，原始动量能快速跟随强趋势；中等广度和中等离散时，平滑质量有助于降低噪音；弱势无主线和低离散震荡中，继续轮动容易制造无效换手和回撤。

## 适用边界

| 维度 | 当前判断 |
| --- | --- |
| 适用资产 | ETF 双池及后续趋势跟踪类策略 |
| 适用频率 | 日频，默认 T 日开盘前使用 T-1 及以前数据 |
| 适用市场状态 | 趋势明确、主线分化、轮动有离散度的市场 |
| 可能失效环境 | 状态切换过快、极端政策脉冲、低离散噪声、信号阈值过拟合 |

## 可观察预测

如果该信念成立，应当观察到：

- `broad_uptrend` 中原始动量优于或不弱于平滑软排序。
- `rotational_momentum` 中平滑软排序相对原始动量有正主动收益。
- `weak_no_leader` 中原始动量和平滑软排序都偏弱，防守或降仓应改善回撤。
- `low_dispersion_chop` 中降低换手或提高分差门槛应优于频繁轮动。
- 状态日志可以解释 R009 和 R20260522-010 中主要月份的胜负差异。

## 竞争性解释

| 解释 | 如何区分 |
| --- | --- |
| 月度环境分组只是事后归因 | R010-A 先只记录逐日状态，不改变交易，再做样本外解释 |
| 状态阈值存在未来函数或全样本分位数污染 | 所有阈值使用滚动窗口或训练期固定阈值，并记录 `signal_date` |
| R9 主动收益来自少数月份偶然性 | 年份切分、月份去极端值、滚动样本外 |
| 状态门控本质是参数过拟合 | 先做少量规则状态机，不引入 HMM 或机器学习分类器 |

## 证据记录

### 支持证据

- [[R20260522-010_ETF双池轮动适用市场环境归因]] 显示宽基/ETF 池共振上行时 R9 平滑软排序拖累，弱势无主线中动量策略明显亏损。
- [[R010-A_市场状态识别规格书]] 已经把状态层设计为只记录日志、不改变交易，适合作为无未来函数的第一阶段验证。
- [[R20260530-019_ETF双池池内动量衰竭状态日志]] 的 2024Q3 短窗口显示，全池 `ret20_breadth`、`ma20_breadth`、`median_ret` 在主要回撤前已经偏弱，池内状态记录方向有价值。
- [[D20260601-R010B3-gate成为当前主候选]] 支持池内 `median_ret` 与 MA20 广度作为低噪音启动确认门。
- [[D20260601-R010B4-tiered-v2四段验证通过]] 支持状态强度分层，而不是单一防守开关。

### 反对证据

- 当前状态定义还没有完成长周期逐日日志验证。
- R019 短窗口也显示，原分类阈值偏乐观：2024-09-11 谷底前仍多判为 `steady/normal`，不能直接作为交易开关。
- 现有归因主要基于 R009 结果，仍可能受样本切分和月份分组影响。
- [[D20260602-R020B-单项标签拆解后仅保留诊断词典]]、[[D20260602-R010B2-A2二级观察门控仅收益修补观察]] 与 [[D20260602-R010C1-A0失速预警仅日志层]] 均提示：可解释标签不等于可交易触发器。

### 尚未解决的问题

- 状态切换确认规则是否会过慢，导致错过急速转强或转弱。
- 是否需要区分“全面上涨早期”和“全面上涨末期”。
- 状态门控对交易次数、换手和成本的影响尚未量化。
- 已有状态标签如何在实盘中长期保持只读、默认关闭或 shadow-only 边界，仍需要审计纪律约束。

## 当前决策

决策：active

理由：机制合理，且 A2-slope004、B3 gate、tiered-v2 已经证明“状态先记录、再消融、再默认关闭映射”的路线有效。但大量标签只能诊断，说明市场状态层必须严格区分解释、观察和交易动作。

下一步最有信息量的行动：维持当前主候选与默认关闭边界；若继续新增状态标签，必须先预注册其用途是解释、收益修补还是防守触发，并用分段覆盖、动作日质量和成本审计决定是否升级。

# B3 Gate + Tiered-v2 —— 防守系统研究总结

> **一句话**：我们花了 9 天时间，从"市场弱了就降点仓"这个模糊想法出发，经过 5 轮淘汰（limited2 → cap90 → gate → tiered-v1 → tiered-v2），最终造出了一个三层分级的防守系统，现在正在实盘中保护你的账户。四段回测中它把最大回撤从 -35% 压到了 -27%，同时收益反而增加了 +137pp。

---

## 一、这是什么？

### 直觉

动量轮动策略靠"追强势资产"赚钱。但当市场整体转弱时，追强势的策略会反复换仓、反复亏损。这时候需要的不是选出更好的 ETF，而是**降低风险暴露**。

问题在于：什么时候降？降多少？

如果每根阴线都降——你会错失大量反弹。
如果只在崩盘时降——来不及了。
如果一直低仓位——收益没了。

B3 gate + tiered-v2 就是把"什么时候降"和"降多少"这两个问题拆开，分别用不同的信息源来回答。

### 它是怎么挂在策略上的

```
每一天的决策流程：

A2-slope004 选出要买什么（进攻端，不改）
        ↓
B3 全弱信号检查：池内 Top1-Top5 ETF 的趋势是不是都在走弱？
  没有走弱 → 正常满仓交易
  走弱了 → 进入 gate 确认
        ↓
Gate 确认：市场整体是不是也在弱？
  (中位收益 ≤ 0？ 或者 广度 ≤ 40%？)
  不满足 → gate 拦截了（"这次走弱可能是噪音，不降仓"）
  满足 → gate 通过，进入 tiered-v2 分级
        ↓
Tiered-v2 判断：风险有多强？
  普通风险 → 仓位降到 80%
  强风险 → 降到 70%
  极端风险 → 降到 60%
```

**核心设计哲学**：B3 负责"要不要防守"，tiered-v2 负责"防守多重"。两者用的是不同的信息——B3 看趋势是否走弱，gate 看市场广度是否恶化，tiered-v2 看持续时间和衰竭程度。三个信息源互相独立，一个失效不会导致全盘崩溃。

---

## 二、演进过程（5 轮淘汰）

### 第 0 轮：我们尝试过的东西（都失败了）

在 B3 gate 之前，研究者尝试了：

| 方案 | 为什么被淘汰 |
|------|-------------|
| 外部 510300 阴跌开关 | 沪深 300 阴跌和你手中的 ETF 池不一定同步——保护来自"长期趴着不动"，不是精准识别 |
| R026 防守池吸血 | 防守 ETF 涌入信号太稀疏（2024 年只触发了 3 天），没法做主开关 |
| 常开波动率缩放 | 连续缩放反而拖累收益，误伤反弹 |

**教训**：外部基准不如池内状态，稀疏信号不能做主开关，连续缩放不如条件触发。

### 第 1 轮：limited2 是否比 cap90 更好？

**问题**：B3 全弱信号触发后，是把仓位从正常的 100% 降到 90%（cap90），还是进一步降到 80% 但只持续 2 天（limited2）？

**结果**：
- limited2 的回撤改善确实更好
- 但四段总收益低于 cap90——防守太猛，牺牲了太多进攻
- **淘汰**。limited2 的设计理念被保留（"防守动作应该有限制"），但方案本身不采纳

### 第 2 轮：能不能加一层"确认门"来过滤假信号？

**问题**：B3 全弱信号本身噪音大——有时候 Top5 ETF 确实在走弱，但只是暂时的，整个市场没有在恶化。能不能在这中间加一层 filter？

**方案**：在 B3 原始信号之后加 `gate`——用 `中位收益 ≤ 0` 或 `池内广度 ≤ 40%` 作为确认条件。只有确认了才执行 cap80。

**结果**：
- 四段复合收益 +1351%，最大回撤 -30.0%（vs cap90 的 -32.5%）
- Gate 成功过滤了 44 天假信号，只保留了 66 天真风险日
- ✅ **推广为主候选**。B3 gate 成为新的防守基准。

| | gate blocked（被拦截） | gate active（确认风险） |
|---|---|---|
| 天数 | 44 天 | 66 天 |
| 后续 10 日出现 -5% 下跌 | 8 天（18%） | 18 天（27%） |
| 含义 | 低风险，拦截正确 | 高风险，gate 识别正确 |

Gate 出来的 66 天确实是更高风险的天。这道门卡得有理有据。

### 第 3 轮：cap80 对所有风险日都一样吗？

**问题**：gate 确认了风险，但有的风险日只是"市场情绪偏弱"，有的风险日则是"池内全面崩溃"。一律降到 80% 够不够？会不会有些天应该降更多？

**方案**：在 gate 之上做"强度分层"——tiered-v1：强风险 → cap70，极端风险 → cap60。

**结果**：
- 2020-2021 年一次都没触发升级（条件太窄）
- 2022-2023 年触发了但 log edge 是负的
- **淘汰**。v1 的升级条件用的是双重 AND 逻辑（`median_ret ≤ -0.01 AND ma20_breadth ≤ 0.30`），太严格了。

### 第 4 轮：tiered-v2（最终版本）

**改进**：放宽升级条件，增加更多维度的判断。

```
strong_risk (cap70)：
  B3 gate active
  + cum_days ≥ 10             ← "全弱信号持续够久"（vs v1 没有这个）
  + median_ret ≤ 0            ← "中位收益不转正"（放宽了 v1 的 ≤-0.01）
  + top5_exhausted ≥ 4        ← "Top5 ETF 中至少 4 个在衰竭"（独立于 gate 的新信息）

extreme_risk (cap60)：
  strong_risk 成立
  + (median_ret ≤ -0.015 OR ma20_breadth ≤ 0.25)  ← OR 而非 AND（放宽）
```

**四段回测结果**：

| | 复合收益 | 最大回撤 | 相对 B3-gate-cap80 | 动作日胜率 |
|---|---|---|---|---|
| B3-gate-cap80（基准） | +1351% | -30.00% | — | — |
| tiered-v2 | **+1468%** | **-27.13%** | **+117pp, +2.88pp** | **77.3%** |

**分段看**：
- 2020-2021：收益 +3.5pp，回撤几乎不动（这段 window 里没有深度风险）
- 2022-2023：回撤改善 +0.89pp（深熊里最稀缺的改善）
- 2024：收益 +5.0pp + 回撤改善 +3.0pp（主回撤窗口 -27% vs -30%）
- 2025-20260519：收益 +12.3pp，回撤持平

✅ **晋升为默认关闭候选，进入实盘映射。**

### 第 5 阶段：实盘启用（2026-06-02）

经过用户明确授权，tiered-v2 从"默认关闭可观察"切换为 MiniQMT v19 固定启用版：

```
A2-slope004：进攻端，默认开启
B3-gate：防守确认门，默认开启
B3-gate-tiered-v2：强度分层，默认开启（交易开，影子关）

启动审计 21/21 项通过。首个交易日需核验启动日志。
```

如果首个交易日出现异常——按启用前备份立即回滚。

## 三、实盘运行时的具体规则

每天 13:10 交易决策：

```
Step 1: 计算 B3 全弱状态（cum_days、consecutive_days）
Step 2: 如果 cum_days≥8 或 consecutive_days≥3 → raw 全弱成立
Step 3: Gate 检查 → median_ret≤0 或 ma20_breadth≤0.40？
        不满足 → gate_blocked，正常满仓
        满足 → gate_active
Step 4: Tiered-v2 检查：
        cum_days≥10 且 median_ret≤0 且 top5_exhausted≥4？
          否 → normal，最终 cap = 80%
          是 → 继续检查：
                median_ret≤-0.015 或 ma20_breadth≤0.25？
                  否 → strong_risk，最终 cap = 70%
                  是 → extreme_risk，最终 cap = 60%
Step 5: 实际持仓 = A2-slope004 选出的目标仓位，但以最终 cap 为上界
Step 6: 超出的仓位转入 511880（货币 ETF）
```

**各字段来源（T-1 收盘后可得，无未来函数）**：
- `cum_days`：历史全弱信号累计天数
- `median_ret`：池内 ETF 的中位收益率
- `ma20_breadth`：池内 ETF 中收盘在 20 日均线上方的比例
- `top5_exhausted`：Top5 ETF 中动量衰竭的数量

## 四、为什么这个方案有效？

1. **三层独立信息**：B3 看趋势走弱、gate 看广度恶化、tiered-v2 看持续时间和衰竭程度。三个信息源提供了冗余安全边际。

2. **比例降仓而非清仓**：cap80/cap70/cap60 保留了 60-80% 的风险暴露，避免完全错过反弹。这与 R015（外部 510300 二元清仓）形成鲜明对比——后者因"长期趴着不动"而丧失收益。

3. **条件触发而非连续缩放**：只在确认风险时才降仓，非风险日保持满仓。这避免了常开波动率缩放在反弹日的误伤。

4. **不替代进攻端**：tiered-v2 完全不改变 A2-slope004 的选股逻辑——它只在选完之后限制仓位上限。选股对错是进攻端的事，仓位轻重是防守端的事，两者职责分离。

---

## 五、它不做什么（硬边界）

| 不做的事 | 为什么 |
|----------|--------|
| 不替代 A2-slope004 的选股 | 选股和仓位是两个独立决策 |
| 不在 gate 未触发时降仓 | 防止反弹日误伤（机械降仓的核心失败原因） |
| 不降到 cap60 以下 | 当前证据不支持更深降仓的边际优势 |
| 不与 R026 叠加 | R026 已证明不能改善回撤 |
| 不与非线性仓位叠加（实盘中） | R5 严格反事实审计已否定了整体替代路径 |

---

## 六、已知局限

1. **2021-05 缺口**：B3 全弱信号在这个深回撤窗口之前没有提前触发——这是一个漏报。R020 复盘后 park，当前没有解决方案。
2. **动作日少，极端层依赖窄样本**：extreme cap60 在 2020-2021 只触发了 0 天，2022-2023 触发了 2 天。四段验证中的 extreme 表现基于小样本，如果未来极端风险日特征变化，cap60 的效果未知。
3. **不解决进攻端问题**：tiered-v2 是纯防守模块。如果 A2-slope004 在特定市场状态下选错了 ETF，tiered-v2 无法纠正——它只会降低仓位，不会换仓。

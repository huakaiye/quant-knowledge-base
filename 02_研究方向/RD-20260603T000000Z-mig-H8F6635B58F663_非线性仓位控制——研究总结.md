# 非线性仓位控制 —— 研究总结

> **一句话**：我们试图让仓位随回撤深度连续缩放而不是固定三档，回测数字漂亮（+1535%），但严格反事实审计揭穿了它——降仓决策的局部正确性只有 37%，还不如抛硬币。现在暂停，等新数据。

---

## 一、这是什么？

### 你的想法

tiered-v2 的仓位是三档硬切换：cap80 / cap70 / cap60。你提出了一个更精细的想法：

> 仓位不应该是三档开关，而应该是**连续函数**——回撤越深仓位越低，回弹时自动恢复。

后来你进一步提出了一个完整公式：

```
仓位 = 波动率缩放 × tanh(趋势强度) × exp(-衰减 × 回撤深度) × 资金曲线状态
```

### Agent 把它拆成验证路线

- **R 支线**：先把最核心的"回撤反馈"部分跑通（`exp(-a × 回撤)`），验证非线性缩放能否替代固定 cap
- **S 支线**：再把完整四组件公式拆开逐个审计，看看每个组件是否可落地

---

## 二、R 支线：回撤反馈验证（14 个节点）

### 阶段 1：机械降仓 → ❌ 被快速否定

**R010-B5**：先在净值代理层测试最简单的"回撤到 15% → 降到 70%，回撤到 25% → 降到 50%"。

| 版本 | 收益保留率 | 回撤改善 | 动作天数 | 问题 |
|------|-----------|---------|---------|------|
| 激进版 | 37% | +5.8pp | 1289天 | 长期低仓位锁死 |
| 保守版 | 86% | +0.9pp | 323天 | 反弹日误伤严重 |

→ ❌ **revise**。机械降仓有两个致命问题：反弹日误伤 + 长期低仓锁定。必须改为条件触发。

### 阶段 2-4：非线性连续缩放 → ✅ 逐层通过

**R010-B5R1 → R010-B5R3**：引入连续指数函数 + 双层门控。

```
risk_scale = max(0.70, exp(-5 × max(回撤 - 10%, 0)))
```

核心改进：只在 [[B3 gate]] active 时启用 + 连续函数代替硬分档 + 动作日从数百天压缩到 27 天。

| 测试层 | 结果 |
|--------|------|
| 代理审计 | 收益保留率 107.73%，胜率 81.48% ✅ |
| 四段真实路径（叠加版） | 收益 +1535%，回撤 -25.3%，相对基准 +66.6pp ✅ |
| 六参数扰动 | 6/6 优于基准 ✅ |
| 成本压力 | 佣金+滑点后仍优于同成本基准 ✅ |
| 替代版（不叠加 tiered-v2） | 收益更差 ❌ **永久杀死** |

### 阶段 5：影子观察 → ✅ 安全性通过

**R010-B5R4**：1542 条记录，242 字段全覆盖，交易隔离 100% 通过，27 天出现"非线性建议降仓"。

### 阶段 6：严格反事实审计 → ❌ **核心转折点**

**R010-B5R5**：逐笔重建那 27 天"如果用了非线性仓位"和"实际用了固定 cap"的净值差。

| 指标 | 结果 | 门槛 | 判断 |
|------|------|------|------|
| 10日正率 | **37.04%** | ≥70% | ❌ |
| 10日累计 log edge | **-0.029** | >0 | ❌ |
| 5日累计 log edge | -0.025 | >0 | ❌ |

→ ❌ **整体替代路径被否证**。回测里 +66.6pp 的额外收益不是从那 27 天降仓来的——它是运气。

### 阶段 7：失败归因 → 找到残余线索

**R010-B5R5R1-R4**：

| 失败来源 | 事件数 | 10日 log edge |
|----------|--------|---------------|
| **normal 日** | 15 | **-0.033** ← 主拖累 |
| 压到 floor | 13 | -0.031 ← 深回撤降仓误伤 |
| extreme-only | 7 | +0.010 ← 微弱正 |

低 [[median_ret（中位收益）]] 条件下（`median_ret ≤ -0.01` 且 B3 active）有局部正信号：13 事件，10 日 log edge +0.015。但只覆盖 2 个分段，缺独立样本外。

→ 📌 **park**。等待前向或独立样本外。

---

## 三、S 支线：完整公式组件审计（4 个节点）

**R010-B5S1 → R010-B5S4**：把用户的完整公式拆成四个组件逐个审计。

| 组件 | 状态 |
|------|------|
| `σ_target / σ_t`（波动率缩放） | ❌ sigma_target 缺失。有宽基波动率缩放反证历史 |
| `tanh(k × m_t)`（趋势过滤） | ⚠️ 低 median_ret 线索存在但缺独立样本。tanh 未实现 |
| `exp(-a × D_t)`（回撤反馈） | ⚠️ 已实现，但 R5 严格反事实未通过 |
| `S_t`（[[资金曲线状态]]） | ❌ 完全缺失。27 事件中 24 个是"深回撤未恢复" |

→ 📌 **完整公式禁止回测**。四组件中只有一个可用且是观察级。

---

## 四、最终状态

| 结论 | 含义 |
|------|------|
| 非线性不能替代固定 cap70/cap60 | ❌ R5 严格反事实已否证 |
| 非线性叠加 tiered-v2 离线可行 | ✅ R2-R3 已证明 |
| 低 median_ret 条件有局部正信号 | ⚠️ 线索，缺独立确认 |
| 完整乘法公式不可回测 | ❌ 四组件仅一个可用 |
| 非线性仓位不进入实盘 | 🚫 |

## 五、重开条件

| 条件 | 要求 |
|------|------|
| 新增事件 | ≥ 8 个 |
| 分段覆盖 | ≥ 2 个新分段 |
| 日期要求 | 不得与 S3 的 27 个事件日重叠 |
| 可确认对象 | 仅 sigma20/sigma60_rank_state=mid 参考组 |
| 禁止项 | 不完整公式回测、不生成 cap、不扫描 sigma_target、不实盘 |

## 六、前向观察执行规程

后续任何重开都必须先按 [[非线性仓位控制前向观察执行规程]] 做样本资格审计。

关键执行顺序：

1. 先运行 `audit_r010b5s4_forward_sample_readiness.py`，确认是否有新增独立事件。
2. 若没有新增独立事件，继续 park，不得解释旧样本。
3. 若新增事件不少于 8 个且覆盖不少于 2 个分段，先新建 R010-B5S5 预注册。
4. S5 仍只能只读确认 rolling sigma mid，不得生成仓位公式、cap、目标权重或实盘动作。

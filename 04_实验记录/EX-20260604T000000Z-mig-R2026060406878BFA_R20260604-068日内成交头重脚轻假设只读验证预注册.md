# R20260604-068 日内成交头重脚轻假设只读验证预注册

## 研究信念

关联对象：[[小市值日内做T]]、[[动量趋势日内做T]]、[[集合竞价成交模型]]、[[集合竞价成交容量模型]]、[[小盘流动性压力]]、[[未来函数审计清单]]。

当前相信的机制：

日内做 T、涨停事件和集合竞价类策略的可实现收益，首先受日内成交容量和价格冲击约束。若多数标的的成交和波动都集中在开盘初段，那么开盘阶段更适合做容量审计和冲击风险估计；若午盘 13:00-14:00 确实相对稳定，则该窗口可能更适合做低冲击调仓、回补或做 T 平仓候选。

先验置信度：中。A 股常见日内成交呈开盘和尾盘活跃的 U 型结构，但研究库当前只记录了 09:30 容量字段覆盖不足和分钟线依赖缺口，尚未对全市场、小盘池、ETF 池和事件样本做过系统只读验证。

## 本次研究假设

主假设分为强弱两版：

| 版本 | 假设 | 本轮处理 |
| --- | --- | --- |
| 强版本 | 所有标的都呈现开盘成交和波动显著更高、午盘 13:00-14:00 更稳定 | 作为异质性审计对象，不直接 promote |
| 弱版本 | 多数可交易标的、多数交易日和至少 3/4 分段呈现开盘成交/波动更强，午盘窗口相对开盘更稳定 | 本轮主要验证对象 |

本轮只做只读事实审计，不构造交易信号、不回测、不修改策略、配置或实盘文件。

## 窗口定义

默认交易分钟数按 A 股连续竞价 240 分钟处理。若数据源缺少 09:30 行或存在 09:30/09:31 聚合差异，必须在 S0 覆盖审计中单独输出，不得静默补齐。

| 窗口 | 时间 | 用途 |
| --- | --- | --- |
| `open_15` | 09:30-09:44 | 开盘即时冲击 |
| `open_30` | 09:30-09:59 | 开盘初段主窗口 |
| `morning_mid_30` | 10:30-10:59 | 上午中段对照 |
| `pre_lunch_30` | 11:00-11:29 | 午前对照 |
| `lunch_open_2` | 13:00-13:01 | 若用户指“午盘刚开 1-2 分钟”，用此窗口 |
| `lunch_30` | 13:00-13:29 | 午盘开盘半小时 |
| `lunch_60` | 13:00-13:59 | 若用户指“1 点到 2 点”，用此窗口 |
| `close_30` | 14:30-14:59 | 尾盘活跃对照 |

所有成交量和成交额指标必须按分钟数归一化，避免把 60 分钟窗口与 30 分钟窗口直接比较。

## 可观察预测

如果弱版本假设成立，预期应当观察到：

- 预测 1：`open_30` 的每分钟成交量/成交额强度显著高于 `lunch_30` 和 `lunch_60`，并高于全日平均强度。
- 预测 2：`open_30` 的分钟收益绝对值、实现波动率和高低价振幅显著高于 `lunch_30` 与 `lunch_60`。
- 预测 3：`lunch_30` 或 `lunch_60` 相对 `open_30` 的价格冲击指标更低，即同等成交额对应的绝对价格变动更小。
- 预测 4：上述关系在全市场、399101 小盘池、ETF 池、小市值 T0 实际交易标的和涨停事件候选中方向一致；如果只在部分池成立，则结论降为条件假设。
- 预测 5：2020-2021、2022-2023、2024、2025-2026 至少 3/4 分段方向一致；否则不得作为稳定执行模型前提。

## 指标定义

### 成交头重脚轻指标

| 指标 | 定义 | 判断方向 |
| --- | --- | --- |
| `volume_share_w` | 窗口成交量 / 全日成交量 | 开盘窗口高 |
| `turnover_share_w` | 窗口成交额 / 全日成交额 | 开盘窗口高 |
| `volume_intensity_w` | `volume_share_w / 窗口分钟数` | 开盘窗口高 |
| `turnover_intensity_w` | `turnover_share_w / 窗口分钟数` | 开盘窗口高 |
| `open_vs_lunch_turnover_ratio` | `turnover_intensity_open30 / turnover_intensity_lunch30` | 大于 1 |

### 价格剧烈程度指标

| 指标 | 定义 | 判断方向 |
| --- | --- | --- |
| `rv_w` | 窗口分钟 close-to-close 收益平方和开方 | 开盘窗口高 |
| `absret_sum_w` | 窗口分钟绝对收益之和 | 开盘窗口高 |
| `range_w` | 窗口内 `max(high) / min(low) - 1` | 开盘窗口高 |
| `open_to_close_abs_w` | 窗口首末价绝对收益 | 开盘窗口高 |
| `impact_proxy_w` | `absret_sum_w / sqrt(turnover_w)`，成交额为 0 时置缺失 | 午盘窗口低 |

## 竞争性解释

| 竞争性解释 | 为什么可能成立 | 如何区分 |
| --- | --- | --- |
| U 型结构来自尾盘而非开盘 | 尾盘集合竞价和被动调仓也会放量 | 单独比较 `open_30`、`close_30`、`lunch_30` |
| 开盘成交高但价格未更剧烈 | 大成交可能只是集合释放，不一定带来更大波动 | 成交指标与 `rv_w`、`range_w` 分开验收 |
| 午盘稳定只是低成交假象 | 成交少会降低观测波动，但冲击成本可能更高 | 增加 `impact_proxy_w` 和成交容量下限 |
| 小盘与 ETF 不同 | 小盘盘口薄，ETF 流动性更连续 | 按 universe 分层，不用全市场均值覆盖异质性 |
| 涨跌停污染开盘波动 | 一字板、涨跌停附近价格被限制 | 剔除涨跌停、单独输出涨跌停样本 |
| 09:30 数据口径不稳定 | 既有文档提示 09:30 覆盖和聚合差异 | S0 单独输出 09:30/09:31 覆盖和缺口 |

| 测试 | 目的 | kill / revise / promote 规则 |
| --- | --- | --- |
| 覆盖率审计 | 判断分钟数据是否足以支撑结论 | 关键 universe 覆盖低于 95% 则 `park_data_gap` |

### promote 条件

满足全部条件，才允许把弱版本假设升级为“执行模型前提候选”：

- S0 覆盖和时点审计通过，且不使用未来数据。
- `open_30` 相对 `lunch_30` 成交额强度中位比值不低于 1.25。
- `open_30` 相对 `lunch_30` 实现波动中位比值不低于 1.15。
- 上述两个方向在至少 65% 标的日成立，并在至少 3/4 分段成立。
- 小盘池和小市值 T0 实际交易标的均不反向。
- 去极端日后结论不反转。

### revise 条件

出现任一情况则修订：

- 成交头重脚轻成立，但价格波动不显著更高。
- 午盘低波成立，但成交容量太低，无法支持订单。
- 只在小盘、ETF 或事件股某一类样本成立。
- 结论依赖 09:30 口径，切换到 09:31 后显著变化。

### kill 条件

出现任一情况则停止把该假设作为执行前提：

- 开盘成交强度和开盘波动均不高于午盘。
- 结果在多数分段或多数 universe 方向相反。
- 必须使用未来收益、事后波动窗口或涨跌停后验过滤才能成立。

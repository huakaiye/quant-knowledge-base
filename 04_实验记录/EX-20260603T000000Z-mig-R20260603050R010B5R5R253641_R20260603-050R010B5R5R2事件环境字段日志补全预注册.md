# R20260603-050 R010B5R5R2 事件环境字段日志补全预注册

## 定位

本节点是 R010-B5R5R1 之后的日志可观测性修复，不是 R010-B5R6，不是回测预注册，不是实盘候选。

R010-B5R5R1 的窄条件复核只得到观察级结论：

```text
observe_subset_candidate_weakened_by_segment_conflict_no_live
```

继续推进前，必须先解决一个可复现问题：

```text
旧 R010-B5R4 SHADOW 事件 CSV 只有 cap、回撤、权重和 reason。
median_ret、ma20_breadth、dispersion、avg_r2、top5_exhausted_count 等环境字段没有平铺进入事件行。
R5R1 只能从少量中文 reason 字符串里解析环境，样本严重不足。
```

## 研究假设

主假设：

```text
R010-B5R5/R5R1 后续是否还能解释，取决于事件发生时的环境字段可观测性。
补齐日志字段可以减少归因盲区，但不会提高既有 27 个事件的交易证据等级。
```

竞争假设：

```text
即使补齐字段，strong/extreme 子集仍可能只是小样本噪声；
环境字段只会帮助解释失败，不会让非线性回撤预算自动成为可回测或可实盘候选。
```

## 字段规格

新增字段只从 `_log_state()` 已生成的 `action_record` 复制，不新增行情查询、不重新计算信号、不读未来收益。

关键字段族：

| 字段族 | 代表字段 | 用途 |
| --- | --- | --- |
| 市场环境 | `median_ret`, `ma20_breadth`, `ret20_breadth`, `dispersion`, `realized_vol`, `avg_r2` | 解释事件是否发生在广度坍塌、分化或高波动环境 |
| Top候选 | `top1_code`, `top1_ret20`, `top1_ret5`, `top1_slope10`, `top1_r2`, `top1_score`，并展平至 Top5 | 判断是否旧强者失速、候选仍有动量 |
| B3滤波 | `r010b3_filter_all_weak`, `r010b3_top5_filter_exhausted_count`, `r010b3_top1_filtered_slope10` | 区分全弱、leader exhaustion 与滤波衰竭 |
| B3持久化与gate | `r010b3_all_weak_cum_days`, `r010b3_all_weak_consecutive_days`, `r010b3_all_weak_persistence_gate_median_ret`, `r010b3_all_weak_persistence_gate_ma20_breadth` | 复核 gate 通过和持续天数 |
| tiered-v2 | `r010b3_gate_tiered_strength_level`, `r010b3_gate_tiered_strength_effective_cap`, `r010b3_gate_tiered_strength_reason` | 对齐 strong/extreme 子集 |
| cap来源 | `r010b3_strength_effective_cap`, `r010b3_strength_cap_reasons`, `r026_strength_switch_active`, `r026_b_light_cap_active` | 区分 cap80/cap70/cap60 或 R026 叠加影响 |

# R20260605-084 R010A9S5 预可见容量护栏 Shadow 四分段验证

## 研究目的

承接 [[R20260605-083_R010A9S3_预可见容量护栏只读评估]]。

上一轮只读评估发现，`prior5_window_avg_5pct_full` 可以在不使用触发日未来分钟信息的前提下，剔除 2 个容量不足的 S3 Top3 替代事件，并把 15 个事件的只读扣费后贡献从 `+20648.88` 提高到 `+23372.35`。

本轮目标不是打开交易，而是把该规则接成 R010-A9S5 default-off shadow，并验证：

```text
1. 容量护栏字段能够在正式 V2 回测中稳定记录；
2. 只使用触发日前 5 个交易日的 13:01-13:30 分钟成交额；
3. shadow-only 运行不改变真实 targets / weights / orders / equity；
4. 四分段结果和 R083 只读候选一致。
```

## S5 规则定义

S5 = S3 + 预可见容量护栏：

```text
base S3 trigger:
  action == A2
  AND r010b3_filter_all_weak == true
  AND top2_ret5 > 0
  AND median_ret <= 0.03
  AND top3_code exists
  AND top3_code != current A2 target

capacity guard:
  min(
    avg_turnover_13:01_13:30_prior5(original_target) * 5%,
    avg_turnover_13:01_13:30_prior5(top3_target) * 5%
  ) >= portfolio_notional
```

治理边界：

```text
r010a9_filter_thin_top3_replace_enabled = false
r010a9_filter_thin_top3_replace_shadow_enabled = true
r010a9_filter_thin_top3_replace_capacity_guard_enabled = true

只允许 shadow 记录；
不允许改变真实交易；
不允许进入 MiniQMT 默认开关；
不扩展小市值、A0/A1/all_actions。
```

## 实现与数据口径修正

本轮实现新增了容量护栏参数、prior window 分钟成交额读取、`capacity_guard_pass/reason/pair_fill_ratio` 日志字段，并把 S5 配置拆成四个正式分段。

验证中发现一个数据口径问题：V2 dataset portal 在日内回放时，`get_price(..., frequency="1m")` 对非当前交易日的分钟窗口只查 `_current_minute_frame`，导致触发日前 5 日分钟窗口为空。

已修复为：

```text
当前日分钟请求：继续读取 _current_minute_frame
非当前日分钟请求：改走 dataset.minute_loader(day, symbols, None)，再按 start/end 过滤
```

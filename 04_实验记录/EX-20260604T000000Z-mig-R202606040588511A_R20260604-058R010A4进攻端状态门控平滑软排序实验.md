# R20260604-058 R010A4 进攻端状态门控平滑软排序实验

## 研究信念

关联信念：[[BLF20260523-002_平滑上涨软排序研究信念]]

先验置信度：中

R9 在 2025 的样本外失败说明“常开软排序”会拖慢极端强趋势。若将软排序仅在适配窗口开启，并保留 Top1 极强情形豁免，有望在不伤害原有主线的前提下提升长期收益。

## 研究目的

在不改变防守链路（`B3 gate + tiered-v2`）的前提下，提升进攻端 A2-slope004 的择时质量，重点验证：

- 是否能提升 A2-slope004 在 `2022-2025` 长窗口的回撤质量与换手效率；
- 是否能减少 2025-09、2026-03、2025-12 这类在全面强趋势下的拖累；
- 是否能保留 R009 在 `rotational_momentum` 场景的优势。

## 研究假设

`H1:` 在 `R010-A confirmed_state == rotational_momentum` 且 `broad_uptrend == false` 时启用平滑软排序，其他状态回退原始动量排序，可显著提高 A2 的进攻收益质量。

`H2:` 增加 Top1 强势豁免（Top1 斜率/强度足够强时不参与软排序衰减）可消除在全面上涨下的代表性滞后。

`H3:` 采用“排序加权 + 轻量惩罚”的连续权重形式（而非硬门禁）比固定硬阈值更稳健。

## 竞争性解释

| 解释 | 可能成立原因 | 区分方式 |
| --- | --- | --- |
| 市场状态只是样本重叠效应 | R9 好样本和坏样本天然集中在少数年份 | 年度/月份滚动验证 |
| 改善来自交易频率下降 | 软排序本身降低了换仓 | 同时比较 2025/2026 的交易次数与交易成本 |
| 改善来自特定斜率阈值偶然 | Top1豁免阈值主观 | 3档阈值 + 0.5pp 收益/换手扰动测试 |
| 仍是防守状态驱动而非进攻改造 | 状态门控把 B3/tiered 的效应掩盖了 | 回测固定默认 A2-slope004 与 B3/tiered-v2 两组对照 |

## 实验设计

> 2026-06-04 接手审计更新：本节仍保留原始预注册假设，但不能直接执行。详见 [[R20260604-059_R010A4_进攻软排序字段与实现可用性审计]]。当前平台状态日志没有 `rotational_momentum / broad_uptrend` 输出，`_apply_offense_soft_rank()` 也尚未接入 `filter_etfs()` 交易路径；下一版必须先改为信号级门控和默认关闭实现审计。

### 统一前提

- 进攻核心仍为 `A2-slope004`。
- 防守保持现有默认：`B3 gate + tiered-v2`。
- 仅改写候选分数计算，禁止新增未来字段。

### 对照组

| 版本 | 说明 |
| --- | --- |
| 基准组 | 当前 `A2-slope004 + B3 gate + tiered-v2` |
| v1 状态门控组 | `rotational_momentum` 状态打开 R9 软排序，其它状态回退 raw momentum |
| v2 + Top1豁免组 | v1 + `top1_strong_exemption` |
| v3 全连续组 | v2 + 质量-斜率双重软加权 |

### 核心逻辑

1. 先验软排序：

```text
score_soft = raw_score * (1 + k * smooth_quality)
```

2. 状态门控：

```text
if confirmed_state == "rotational_momentum" and not broad_uptrend:
    use score_soft
else:
    use raw_score
```

3. Top1 豁免（v2、v3）：

```text
if top1_slope10 >= 0.004 and top1_raw - top2_raw >= top_gap_min:
    use raw_score  // 避免强势新起动失速
```

4. 连续修正（v3）：

```text
score_final = score_soft * (1 + beta * normalized(momentum_gap))
```

其中 `momentum_gap` 为 Top1 与 Top2 的短期强度差；`beta` 用于控制追涨偏好，越高越保留 Top1。

## 参数扫描

| 维度 | 取值 |
| --- | --- |

| top_gap_min（豁免门槛） | 0.004, 0.006, 0.008 |
| confirmed_state 映射 | `rotational_momentum` only / `rotational_momentum + cautious` |
| top1_strong_exemption 生效日 | all / only when `raw_top1_score > q80` |
| beta（连续修正） | 0.00, 0.05, 0.10 |

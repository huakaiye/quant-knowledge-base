~~~markdown
# R20260605-089 R010A10S1 A2 换仓延迟确认 Default-Off Shadow 四分段验证

## 研究目的

承接 [[R20260605-088_R010A10_A2换仓延迟确认只读发现]]。

R088 在 R010-A5 的 726 个 A2 switch episode 中发现一个只读候选：

```text
action == A2
AND dispersion < 0.04
AND top1_top5_gap < 0.05
=> 延迟 3 个交易日确认换仓，在只读代理中四分段正 edge

## 关键解释

R088 的 51 个只读事件不等于真实路径中的 51 个可用触发。

R088 使用 R010-A5 switch episode 做 close-to-close 代理，回答的是：

```text
如果已知某次 A2 switch 发生，旧标的继续持有 N 日是否跑赢新标的？
```

R010-A10S1 放进真实路径后，还必须同时满足：

```text
当前真实动作仍为 A2
当前真实风险仓位存在旧目标
最终目标确实从旧目标切到新目标
dispersion/top1_top5_gap 当天字段齐全
shadow 只记录，不改写真实订单
```

这些真实路径约束把候选从 51 个代理事件压缩到 3 个触发事件。3 个事件全部集中在 2020 年 1 月，无法支撑收益增强交易化。

## 结论

```text
keep_default_off_shadow_no_live
```

R010-A10S1 可以保留为 default-off shadow 观察层，因为实现隔离已经通过；但不能进入 live、apply smoke 或 guarded-live。

当前 strict low_disp_small_gap 规则的触发容量太低，收益提升能力不来自“直接上线 S1”，而来自继续研究如何在双池 A2 内部扩大真实可用触发面。
~~~

~~~markdown
# R20260605-085 R010A9S5 放行事件 Live-Readiness 只读审计

## 研究目的

承接 [[R20260605-084_R010A9S5_预可见容量护栏Shadow四分段验证]]。

R084 证明了 S5 容量护栏可以 default-off shadow 运行，且不会污染真实交易路径。但这只说明“能记录、能隔离”，不能说明“适合交易化”。本轮只读审计专门检查 S5 放行的 13 个事件是否达到 guarded-live 的前置质量门槛。

新增只读脚本：

```text

```

```text

results/v2/research/R010-A9/filter_thin_top3_replace/s5_top2_ret5_positive_top3_shadow_a2_median003_prior5win5pct_*/logs.jsonl

输出：

## 审计口径

从 S5 shadow 日志中抽取所有 `capacity_guard_checked=true` 的 15 个事件，再和 S3 逐事件成交可用性复核 CSV 合并。

分类规则：

```text
blocked_by_capacity_guard:
  capacity_guard_pass == false

ready_initial:
  capacity_guard_pass == true
  AND interval_delta_after_fee >= 0
  AND pair_daily_tradeable == true
  AND pair_m1310_tradeable == true
  AND pair_window_tradeable == true
  AND same_day_5pct_capacity_full == true
  AND no adjacent trigger within 1 trading day

review_required:
  放行后仍命中任一风险旗标
```

风险旗标：

```text
negative_after_fee
same_day_5pct_capacity_not_full
adjacent_trigger_path_dependency
daily_pair_not_tradeable
m1310_pair_not_tradeable
window_pair_not_tradeable
```

- `2025-03-10` 与 `2025-03-11` 是连续触发，真实交易化后可能改变下一次触发起点，不能按独立事件简单相加。
- `2021-08-10` 虽然 prior5 预可见容量通过，但触发日同日窗口 5% 容量不足，只能作为可交易性复核问题，不能反过来当信号。

## 治理结论

```text
keep_shadow_no_guarded_live
```

允许：

- S5 继续保留为 default-off shadow 观察候选。
- 后续可以拆解 5 个复核事件，寻找更稳定的二级护栏。
- 继续围绕 ETF 双池轮动内部做进攻收益增强研究。

不允许：

- 不允许把 S5 推进为 guarded-live。
- 不允许直接实盘开启。
- 不允许用同日 13:01-13:30 窗口容量作为交易信号。
- 不扩展小市值、A0/A1/all_actions。
~~~

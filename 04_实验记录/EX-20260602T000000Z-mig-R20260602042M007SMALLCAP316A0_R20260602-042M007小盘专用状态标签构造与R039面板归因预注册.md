# R20260602-042 M007 小盘专用状态标签构造与 R039 面板归因预注册

## 研究信念

关联信念：[[BLF20260528-004_市场环境驱动多策略切换研究信念]]

当前相信的机制是：小市值 T0 的失效不应只用 ETF 池内状态解释。R039 已证明小市值有低相关互补价值，但 R010-A 状态对小市值解释力偏弱；M005 证明本地代理可做部分诊断；M006 证明 schema-only 输入在 universe、日频/估值、财报质量和 ST as-of 层具备进入标签构造预注册的覆盖基础。因此下一步应把这些输入转成可复现、可证伪、只读的小盘专用状态标签，再用 R039 面板检验它们是否比 R010-A 状态更能解释小市值收益、坏日和关键窗口。

先验置信度：中。M006 支持输入可得性，但尚未证明任何标签有收益解释力或组合路由价值。

## 本次研究假设

如果小市值 T0 的收益和风险主要受小盘自身状态驱动，那么基于小盘相对强弱、流动性压力、质量风险和拥挤踩踏构造的只读标签，应当在 R039 日收益面板中呈现可解释差异：

- 小盘有利状态下，小市值日收益、胜率和相对 ETF 主候选超额应更好。
- 小盘压力状态下，小市值 -1%/-2% 坏日率、回撤和同跌风险应更高。
- 关键窗口中，标签应能解释风险释放、低暴露互补或质量/流动性压力，而不是只在事后低点附近出现。

本轮只做预注册，不构造正式标签、不查询数据库、不回测、不路由、不修改策略、配置或实盘文件。

## 可观察预测

如果假设成立，预期应当观察到：

- 预测 1：`small_cap_custom_state=压力` 的日均收益、胜率和状态内最大回撤显著弱于 `有利` 或 `中性/混合`。
- 预测 2：小市值 -1%/-2% 坏日应集中在流动性压力、质量风险或拥挤踩踏状态，坏日覆盖率高于 R010-A `unfavorable`。
- 预测 3：相对 `B3-gate-tiered-v2`，小市值正超额应主要来自 `有利` 或 `混合但无质量压力` 状态；若正超额主要来自压力状态，则标签解释方向需要修订。
- 预测 4：2021-05、2024-01、2024-06 至 2024-09、2024-11 至 2024-12、2025-04 等关键窗口中，至少 3 个窗口能被某一标签族解释为风险、低暴露互补或风格有利。
- 预测 5：去掉收益绝对值最大的 1% 交易日后，标签分组方向仍不反转。
- 预测 6：四段中至少 3/4 分段的压力状态坏日率不低于非压力状态；否则不得进入组合层预注册。

## 竞争性解释

| 竞争性解释 | 为什么可能成立 | 如何区分 |
| --- | --- | --- |
| 标签只是在复述小市值收益本身 | 本地代理含小市值滞后收益和坏日计数 | 分开报告本地代理标签和数据库 schema 标签；做去滞后收益代理消融 |
| 质量风险覆盖是符号覆盖，不是日频风险 | 财报更新慢，且公告日存在阶梯变化 | 输出 as-of 财报年龄和标签变化频率；若变化太少，只能降为背景标签 |
| 流动性压力只是成交活跃度下降 | 低换手可能既代表压力，也代表低暴露或低交易 | 同时报告持仓/成交上下文，不能直接解释为可交易性恶化 |
| 拥挤踩踏需要事后下跌才知道 | 真正踩踏常在未来回撤后确认 | 首轮只定义事前拥挤压力，不使用未来最大回撤或事后低点 |
| 状态差异来自少数极端日 | 小盘收益分布厚尾，少数日贡献很大 | 输出去极端日、分段稳定性和月度贡献 |
| 标签能解释但不能交易 | 状态标签可能只有解释力，缺少稳定提前量 | 本轮只做归因，不进入路由、权重或实盘 |

### promote

满足全部条件才允许进入“组合层风险预算/权重预注册”的前置观察候选：

- 标签面板覆盖率不低于 95%，只读和未来函数审计全部通过。
- `small_cap_custom_state` 的压力/有利状态在全周期和至少 3/4 分段方向一致。
- 压力标签对 -1% 坏日覆盖高于 R010-A `unfavorable`，且不是只由单个标签族或单一年份贡献。
- 相对 `B3-gate-tiered-v2` 至少存在一个稳定正互补状态。
- 去极端日后主要结论不反转。

结果含义：

```text
promote_to_combination_prereg_candidate
```

这只表示可新建组合层预注册，不表示可以自动路由、调权或修改实盘。

### observe

满足任一情况则保持观察：

- 标签能解释部分坏日或关键窗口，但分段稳定性不足。
- 质量风险或拥挤踩踏更像背景标签，不能形成可操作状态。
- 小市值常开输入仍有价值，但状态标签只适合诊断面板。

### revise

出现任一情况则修订：

- 标签效果主要来自本地滞后收益代理，schema-only 指标贡献弱。
- 财报质量标签变化频率太低，只能作为季度背景，不适合作为日频状态。
- 流动性压力与低暴露/低交易混淆，需加入持仓和成交上下文。
- 关键窗口依赖后验阈值或事后下跌确认。

### kill

出现任一情况则停止本轮标签构造：

- 需要未来收益、未来最大回撤或事后谷底才能定义标签。
- 无法保证 `ann_date <= signal_date`、`effective_date <= signal_date` 或 `signal_date <= trade_date`。
- 标签面板与 R039 日收益无法可靠对齐。
- 脚本需要写库、更新数据、运行回测或修改配置/实盘。

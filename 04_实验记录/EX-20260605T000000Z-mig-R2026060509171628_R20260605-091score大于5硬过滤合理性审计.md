# R20260605-091 score 大于 5 硬过滤合理性审计

## 问题

用户澄清：当前策略在评分排名时，若动量得分大于 5 会被直接过滤，这个 `score >= 5` 硬过滤是否合理。

实盘 MiniQMT 当前规则：

```python
MAX_SCORE = 5
# 原版条件：0 < score < MAX_SCORE 才有效
if score is None or score <= 0 or score >= MAX_SCORE:
    continue
```

得分定义：

```python
annual_ret = exp(slope * 250) - 1
score = annual_ret * r2
```

因此 `score > 5` 一般代表 25 日窗口内非常陡的趋势，若 `r2` 接近 1，约等价于年化收益超过 500%，也就是 25 日平滑涨幅约 20% 以上。

## 结论

`score >= 5` 作为“异常强势/可能过热”的识别信号有合理性，但“直接从候选池删除”这个实现不够稳健。

当前不建议立刻废除实盘 cap5，原因是：

1. 它是原版策略的一部分，长周期复现和 MiniQMT 实盘脚本都以此为准。
2. 正式 V2 组合级消融尚未全段完成；close-to-close 组合代理曾显示 cap6 优于 cap5，但 2024 与 2025_20260519 正式段已经反证，不能直接采纳代理结论。
3. 分数公式含指数年化，极端高分容易来自短期暴涨、次新、复权/当前价口径敏感或不可持续趋势。

但也不建议把 `score >= 5` 永久当成硬删除规则，原因是：

1. 贴近 5 的候选会因为一跳当前价差异被完全剔除，工程上很脆。
2. 长周期复现剩余分叉中有 6 个 `score_cap_edge`，说明该边界确实会改变目标。
3. 部分样本中被 cap5 边界影响的是接近 Top1 的强标的，直接删除可能造成机会成本。

2026-06-05 补充：数据库恢复后完成 A11 事件级扫描。结果显示，`score >= 5` 并不应被简单视为纯负面；尤其 2025 与 2026 年内，放宽到 5.5/6 在未来 3/5/10 日有正边际。但无上限结果高度依赖少数极端行情。因此事件级结论不是“废除上限”，而是“硬删除过粗，需要进入组合级正式验证”。

随后新增 close-to-close 组合代理。该代理显示 cap6 在全样本、0bps/2bps/10bps 三档换仓成本下均显著优于 cap5，并且换仓次数更少；但 2022-2024 的分段表现仍提示不能直接无条件上线。后续正式 V2 路径进一步显示，代理结论不能替代正式组合回测。

## 判断

### 为什么 cap5 有逻辑

`score = annual_ret * r2` 中的 `annual_ret` 是指数年化。`score > 5` 不是普通高分，而是短窗口里非常陡的趋势。它可能意味着：

1. 标的短期涨幅过快，追入后均值回归风险上升。
2. 次新 ETF 或新纳入动态池标的窗口不稳定。
3. 当前价一跳、复权口径、盘中价格源会把分数推过阈值。
4. 高分可能与放量、拥挤、短期主题高潮同时出现。

所以 cap5 不是完全拍脑袋。

### 为什么硬删除不理想

硬删除把 `4.9999` 与 `5.0001` 当成完全不同的资产：

- `4.9999` 可以成为 Top1。
- `5.0001` 直接从排名中消失。

这会造成两个问题：

1. 边界噪声被放大，current_data/tick 的一跳差异会改变交易目标。
2. 若高分来自平滑强趋势而不是跳涨/放量，直接删除可能错过主升段。

因此，当前更准确的评价是：cap5 信号合理，硬过滤形态偏粗。

## 建议

### 当前实盘

暂不直接改 `MAX_SCORE=5`。

原因是正式 V2 组合级消融尚未完成，且 cap5 属于当前可复现策略的一部分。直接放开会同时改变排序、A2 输入、B3 输入、买入替补序列和成交失败后的顺位逻辑，影响面比看起来大。

但事件级证据、close-to-close 组合代理和替代机制扫描仍足够反对“永久保留 score>=5 直接删除且不再研究”。正式 V2 两段结果已经反对直接把 `max_score=6` 或 `quality_gate6` 上线；下一步应优先做 hard6/quality_gate6 误选归因，并继续跑剩余正式分段。winsor 仅作为极端分数解释性方案保留，softlog 类单调压缩降级。

### 下一轮研究

优先做四个版本的组合级消融：

| 版本 | 规则 | 目的 |
|---|---|---|
| cap5_base | `0 < score < 5` | 当前基线 |
| cap6 | `0 < score < 6` | 待归因候选；正式两段暂不支持替代 cap5 |
| cap7 | `0 < score < 7` | 判断 cap6 以上是否开始引入不稳定 |
| quality_gate6 | `score<5` 保留；`5<=score<6` 叠加 R2/量比/MA20 乖离门控 | 待归因候选；正式两段暂未确认低回撤优势 |
| winsor_score | `score_adj = min(score, 5或6)`，不删除 | 解释性候选，保留强标的但压缩极端分数 |

不再推荐 `softlog` 作为独立候选，因为单调变换不会改变排序，等价于无上限。若做 soft penalty，必须叠加非单调的过热惩罚或质量门控。

### 替代判断

不要再用单一 `score >= 5` 做硬否决。建议把它改成二段式：

1. `score >= 5` 只打上 `score_cap_hot` 标签。
2. 再用以下条件决定删除、降权或保留：
   - 近 5 日涨幅是否过大。
   - 当日/盘中成交量比是否接近或超过 2.5。
   - R² 是否足够高，高 R² 代表平滑趋势，低 R² 代表跳涨噪声。
   - 价格是否远离 MA20/MA60。
   - 是否为新进入动态池或历史窗口不足边缘标的。

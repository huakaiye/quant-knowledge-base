## 最佳只读候选

候选规则：

AND raw Top1 ret5 <= 0
AND raw Top1 slope10 <= 0
candidate = Top2-Top5 中 no_rank_quality 最高者

## 解释

1. R010-A4 的可交易边际不在“常规软排序”，而在少数 Top1 近端衰竭日。
2. `raw_top1_gap` 小并不是充分条件；小 gap 的 no-rank quality 规则在 T+5/T+10 为负。
3. 更合理的机制是：Top1 中期仍可能强，但 ret5 与 slope10 同时转弱时，Top2-Top5 中质量更平衡的候选有较高概率接力。
4. 这个机制与旧 A1 candidate-only 不同：它不恢复 A1 动作，也不依赖 R010-B1 动作层，只在 R010-A4 gate-pass 且 Top1 近端衰竭时做默认关闭候选。

## 决策

```text
promote_to_s5_preregister_default_off_smoke
```

允许：

- 实现 R010-A4S5 默认关闭候选。
- 先跑 2024 baseline vs S5 smoke。
- 审计 selected_code 差异、收益、回撤、换手和触发日清单。

禁止：

- 直接进入 2025/2026 样本外。
- 修改 MiniQMT 或实盘默认开关。
- 恢复 A1/A3/A4 旧动作。
- 扩大为无门控 Top2 替换或小 gap 通用替换。

## S5预注册规格

```text
name: R010-A4S5_top1_fast_weak_no_rank_quality
default: off
gate: 沿用 R010-A4 v1 signal_rotational_not_broad gate
trigger:
  top1_ret5 <= 0
  top1_slope10 <= 0
selector:
  在 raw_rank 2-5 中选择 no_rank_quality 最高者
fallback:
  若候选缺失或得分不可算，保持原 Top1
scope:
  只影响 filter_etfs 的排序结果，不启用 A1/A3/A4
```

S5 通过条件：

```text
1. 2024 至少产生可解释 selected_code 差异。
2. 总收益不低于 baseline，或回撤/换手改善足以解释收益代价。
3. 触发日与 S4 预注册逻辑一致，不出现额外未解释分支。
```

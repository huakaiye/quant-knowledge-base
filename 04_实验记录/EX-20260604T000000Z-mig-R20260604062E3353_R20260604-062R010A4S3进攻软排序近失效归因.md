# R20260604-062 R010A4S3 进攻软排序近失效归因

## 研究目的

承接 [[R20260604-061_R010A4S2_进攻软排序2024Smoke行为审计]]。S2 已确认 `v1_signal_gate` 与默认关闭基线在 2024 年资金曲线、选标和交易笔数完全一致。本轮只读归因要回答：

1. v1 为什么触发了门控和 applied，却没有产生交易差异。
2. 去掉 `rank_part`、增强 multiplier 或改成 additive 后，是否足以形成可回测的 v2 候选。
3. 是否应转向其他进攻收益路线。

## v2候选预审

本轮只做 top5-only 预审，不等同真实回测。候选包括 current multiplicative、no-rank multiplicative、放大 scale、no-rank additive。

结论：简单分数形态无法提供足够交易样本。直接开 v2 2024 smoke 预期价值很低，且容易变成后验调参。

## A1递补路线排除

日志复查发现 A1 `selected_rank=2` 有时没有真实启用，是因为当前 R010-A4 配置只启用 `A2`，不恢复 `A1`。这不是新机会。

既有 [[R20260530-021_R010B1动作消融首轮实现]] 已记录：`A1-candidate-only` 跨周期不稳定，不单独采纳；当前路线也多次要求不混入 A1/A3/A4 旧动作。因此本轮不把“恢复 A1 递补候选”作为 R010-A4 v2。

## 决策

```text
revise_to_ex_ante_quality_label_before_backtest
```

允许：

- 继续做 R010-A4S4 只读标签发现。
- 用 gate_pass 日构造原 Top1 与 Top2-Top5 的未来收益差面板。
- 寻找能提前解释“什么时候应该替换 Top1”的质量标签。

禁止：

- 直接跑这些简单软分数候选的交易回测。
- 恢复 A1/A3/A4 旧动作。
- 修改 MiniQMT 或实盘默认开关。

## 下一步

R010-A4S4 应改成只读 counterfactual 标签发现：

```text
1. 样本：2024 v1 gate_pass 日的 raw top5。
2. 标签：Top2-Top5 中是否有人在 T+1/T+5/T+10 超过原 Top1。
3. 特征：ret5、slope10、r2、score gap、raw_top1_gap、top1_top5_gap、dispersion、breadth。
4. 约束：只允许预注册少量可解释标签；不允许把旧 A1 candidate-only 变体重新包装成新规则。
```

只有当只读标签能在 2024 内产生足够且可解释的 Top1 替换样本，才允许进入下一轮默认关闭 smoke。

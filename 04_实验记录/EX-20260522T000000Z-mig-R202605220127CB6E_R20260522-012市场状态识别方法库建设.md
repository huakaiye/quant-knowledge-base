# R20260522-012 市场状态识别方法库建设

## 背景

回测平台与数据库正在升级，当前不适合继续启动新回测。本研究记录用于沉淀市场状态识别和状态转变信号的方法资料，为后续 R010-A/R010-B/R010-C 做准备。

## 本次新增内容

| 类型 | 文档 | 用途 |
| --- | --- | --- |
| 方法库 | [[市场状态识别方法库]] | 汇总状态识别方法族、代表文献、研究顺序 |
| 信号清单 | [[状态转变信号候选清单]] | 管理候选信号、数据口径、可用时点和未来函数风险 |
| 文献笔记 | [[2003_Pastor_Stambaugh_流动性风险]] | 补充流动性状态变量 |
| 文献笔记 | [[2008_Campbell_Thompson_样本外预测约束]] | 补充样本外预测与经济约束原则 |
| 文献笔记 | [[2016_Baker_经济政策不确定性]] | 补充外部宏观不确定性候选 |

## 方法结论

第一阶段应优先使用价格内部、可解释、可审计的状态信号：

- ETF池趋势广度。
- ETF池中位20日收益。
- Top1绝对动量。
- Top5动力确认。
- Top1-Top5分差。
- 横截面离散度。
- 市场实现波动率。
- 急跌后急反弹。

流动性、政策不确定性、HMM 过滤概率可以进入第二阶段，但不应进入首版 R010。

## 后续研究顺序

| 阶段 | 内容 | 是否需要回测 |
| --- | --- | --- |
| R010-A | 状态日志，不改变交易 | 需要平台恢复 |
| R010-A 扩展 | 用已有结果对齐状态做归因 | 可在结果文件完整时做 |
| R010-B | 弱势无主线和动量崩盘风险防守门控 | 需要平台恢复 |
| R010-C | 原始动量和 R9 平滑软排序状态切换 | 需要平台恢复 |
| R010-D | Top1 到 Top5 动力确认 | 需要平台恢复 |

## 无未来函数要求

| 要求 | 说明 |
| --- | --- |
| 信号时点 | T日交易前默认只用T-1及以前收盘数据 |
| 阈值 | 使用训练期固定阈值或滚动窗口，不用全样本分位数 |
| 标签 | 不用未来收益给当天打状态标签 |
| 状态模型 | 如使用 HMM，只允许过滤概率，不允许平滑概率 |
| 外部数据 | 必须记录发布时间和修订规则 |

## 决策

暂不回测。先将市场状态识别作为独立研究专题维护，等平台和数据库升级完成后，从 R010-A 状态日志开始实现。

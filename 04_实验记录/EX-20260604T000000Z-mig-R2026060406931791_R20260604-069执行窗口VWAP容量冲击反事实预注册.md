# R20260604-069 执行窗口 VWAP 容量冲击反事实预注册

## 研究信念

关联决策：[[D20260604-INTRADAY-001-日内成交头重脚轻进入执行模型候选]]。

R068 已验证弱版本日内形态：多数标的、多数交易日中，开盘 30 分钟成交强度和价格波动高于午盘窗口。但该结论还不能回答真实执行问题，因为执行窗口选择至少包含三个互相冲突的目标：

- 容量：能否在参与率上限内完成目标订单。
- 价格偏移：窗口 VWAP 相对窗口到达价的偏离是否更小。
- 冲击代理：单位成交额对应的窗口内价格扰动是否更低。

本实验只做执行窗口反事实，不做策略收益、不做买卖方向判断、不做回测。

## 核心假设

主假设分为三部分：

| 编号 | 假设 | 预期方向 |
| --- | --- | --- |
| H1 | `open_30` 的可成交容量显著高于 `lunch_30` 和 `lunch_60` | 开盘容量更高 |
| H2 | `lunch_30/lunch_60` 的窗口内 VWAP 偏移、实现波动和振幅低于 `open_30` | 午盘更稳 |
| H3 | 当目标订单按 `open_30` 的 3% 参与率容量设定时，午盘窗口经常无法在同等参与率内完成 | 午盘容量约束明显 |

竞争假设：

| 竞争解释 | 如何区分 |
| --- | --- |
| 午盘低波动只是因为成交少 | 同时报告容量和冲击代理，不只看 VWAP 偏移 |
| 开盘容量高但价格太不稳定 | 分开输出容量优势和窗口内价格偏移，不用单一综合分强行排序 |
| 尾盘也可能有较高容量 | 将 `close_30` 作为独立对照，不把日内结构简化成开盘 vs 午盘 |
| 固定金额订单与比例订单结论不同 | 同时输出固定名义金额和锚定参与率订单两类 fill ratio |

## 样本与窗口

样本日沿用 R068 的四段等距样本，默认每段 8 个交易日，共 32 个样本日。这样本实验只改变执行反事实定义，不改变市场样本。

窗口固定为：

| 窗口 | 时间 | 说明 |
| --- | --- | --- |
| `open_30` | 09:30-09:59 | 开盘主窗口 |
| `lunch_30` | 13:01-13:30 | 午盘 30 分钟；分钟表无 13:00 行 |
| `lunch_60` | 13:01-14:00 | 用户指定 13:00-14:00 的可用分钟口径 |
| `close_30` | 14:30-14:59 | 尾盘对照窗口 |

若 09:30 历史覆盖不足，不补齐、不前推、不作为开盘前可见特征；只按实际分钟行聚合。

## 指标定义

### 窗口成交与 VWAP

| 指标 | 定义 | 用途 |
| --- | --- | --- |
| `window_turnover` | 窗口成交额 | 容量基数 |
| `window_volume` | 窗口成交量，平台分钟表口径为手 | VWAP 分母 |
| `window_vwap` | `window_turnover / (window_volume * 100)` | 窗口成交均价代理 |
| `arrival_price` | 窗口第一根分钟 K 的 open | 窗口到达价 |
| `vwap_arrival_signed_bps` | `log(window_vwap / arrival_price) * 10000` | 买入方向窗口内价格漂移 |
| `vwap_arrival_abs_bps` | `abs(vwap_arrival_signed_bps)` | 无方向执行偏移 |
| `range_bps` | `(max(high)/min(low)-1)*10000` | 窗口振幅 |
| `rv_bps` | 窗口分钟 close-to-close 实现波动 * 10000 | 窗口波动 |

### 容量与填充率

参与率上限固定为：

```text
1% / 3% / 5% / 10%
```

固定名义金额订单：

```text
50,000 / 100,000 / 300,000 / 1,000,000 元
```

锚定反事实订单：

```text
anchor_open30_3pct = open_30 window_turnover * 3%
```

对每个窗口输出：

| 指标 | 定义 |
| --- | --- |
| `capacity_at_p` | `window_turnover * p` |
| `fixed_fill_ratio_at_p` | `min(1, capacity_at_p / fixed_notional)` |
| `anchor_fill_ratio_at_p` | `min(1, capacity_at_p / anchor_open30_3pct)` |
| `required_participation_for_anchor` | `anchor_open30_3pct / window_turnover` |

### 冲击代理

本轮没有逐笔订单簿，因此只使用保守代理：

```text
impact_proxy = absret_sum / sqrt(window_turnover)
```

该指标只用于窗口比较，不视作真实冲击成本。

## 决策规则

### promote 条件

满足全部条件，才允许进入“执行窗口规则候选”下一预注册：

- `open_30` 相对 `lunch_30` 的容量中位比值不低于 2.0。
- `open_30` 锚定 3% 参与率订单，在 `lunch_30` 以 3% 参与率完成的中位 fill ratio 低于 0.60。
- `lunch_30` 或 `lunch_60` 的 `vwap_arrival_abs_bps`、`rv_bps`、`range_bps` 至少两个指标显著低于 `open_30`。
- 以上方向在至少 3/4 分段成立。
- 小盘池、小市值 T0 实际交易标的和 ETF 分层不出现完全相反结论。

决策结果：

```text
promote_to_execution_window_rule_prereg_candidate
```

### revise 条件

出现任一情况则修订：

- 午盘更稳但容量不足，只允许作为小订单或回补窗口候选。
- 开盘容量高但 VWAP 偏移过大，只允许用于容量上限估计，不作为优先执行窗口。
- 尾盘容量或偏移优于开盘/午盘，需要三窗口规则而非二分规则。
- 固定金额订单和锚定订单结论冲突，需要按订单规模分层。

### kill 条件

出现任一情况则停止该执行前提：

- 开盘容量优势不稳定，少于 3/4 分段成立。
- 午盘稳定性不成立，VWAP 偏移和波动均不低于开盘。
- 结论依赖未来收益、后验买卖方向或策略净值。

### 当前结论

结论状态：

```text
promote_to_execution_window_rule_prereg_candidate_sample_supported
```

执行含义：

- 大订单或容量敏感订单：`open_30` 仍是主要容量释放窗口，但要承受更高 VWAP 偏移和波动。
- 小订单或低扰动优先订单：`lunch_30` 可以作为低扰动窗口，但容量明显不足。
- 中等订单：`lunch_60` 和 `close_30` 需要进入下一轮三窗口规则比较，不能只做开盘/午盘二分。
- 不允许把本结论接成交易信号；它只服务执行窗口设计。

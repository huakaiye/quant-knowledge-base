# R20260522-011 市场状态识别文献库与 R010 设计

## 研究目的

市场状态识别和市场转变信号是 ETF 双池动量轮动的关键中间层。本记录把文献启发升级为项目研究库模块，为 R010 实验提供明确的信号规格和防未来函数约束。

## 新增文献笔记

| 文献 | 核心启发 | 项目落点 |
| --- | --- | --- |
| [[2004_Cooper_市场状态与动量]] | 动量收益依赖过去市场状态 | 用滞后市场收益和ETF池状态做门控 |
| [[2010_Stivers_截面离散度与动量溢价]] | 横截面离散度是状态变量 | 用ETF池离散度判断轮动是否有选标价值 |
| [[2002_Ang_Bekaert_状态转换资产配置]] | 状态有持续性，切换要有概率和成本 | 状态机需要确认、退出阈值和滞后 |
| [[2002_Chordia_商业周期与动量]] | 动量收益有周期性和状态依赖 | 用价格状态替代低频宏观周期，控制模块开关 |

## 新增模块想法

- [[市场状态识别与切换门控]]

该模块成为 R010 的核心，不再把市场环境判断散落在 R009 分析和策略参数里。

## 无未来函数约束

| 约束 | 规则 |
| --- | --- |
| 可用数据 | T日买入信号只能用T日信号生成时已经可见的数据 |
| 日线口径 | 开盘前/盘前信号默认使用T-1收盘；盘中信号必须明确使用实时价 |
| 阈值 | 禁止全样本分位数；使用训练期固定阈值或滚动历史分位 |
| 状态模型 | 若未来使用HMM，只能用filtered probability，不能用smoothed probability |
| 切换 | 状态切换必须有确认规则，避免单日噪音来回跳 |
| 实验 | 阈值选择和样本外验证必须分离 |

## R010 研究设计

R010 不直接优化收益，而是验证“状态门控是否减少大类错误”。

状态动作：

| 状态 | 策略动作 |
| --- | --- |
| 宽基/ETF池共振上行 | 原始动量 |
| 窄主线强趋势 | 原始动量 + 轻质量惩罚 |
| 中等离散轮动 | R9平滑软排序 |
| 低离散震荡 | 提高TopN保留，减少换仓 |
| 弱势无主线 | 防守ETF或空仓 |
| 恐慌后修复 | 降仓，避免追旧赢家 |

## 预期实验序列

| 实验 | 内容 | 目的 |
| --- | --- | --- |
| R010-A | 只用广度 + Top动量强度 | 验证最简状态机 |
| R010-B | 加入横截面离散度 | 验证是否减少低离散误交易 |
| R010-C | 加入动量崩盘风险 | 验证是否减少恐慌后反弹误伤 |
| R010-D | 完整状态门控 | 与原始基准、R9软排序对比 |

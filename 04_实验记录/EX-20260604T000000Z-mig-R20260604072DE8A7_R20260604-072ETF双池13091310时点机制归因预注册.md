# R20260604-072 ETF双池 13:09/13:10 时点机制归因预注册

## 问题

当前 ETF 双池在 13:09 卖出、13:10 买入。需要回答：

| 问题 | 需要的证据 |
| --- | --- |
| 为什么是 13:09/13:10 | 是历史经验、信号可见性约束、价格漂移、成交量，还是其他因素 |
| 其他时间点是否更好 | 对信号生成后所有可执行分钟做扫描 |
| 13:10 收益高的来源 | 价格优势、回撤控制、容量，还是统计噪声 |
| 是否要正式回测 | 只有分钟扫描出现稳定候选时才进入 WSL 正式回测 |

## 先验事实

代码事实：

```text
09:20 update_dynamic_pool
13:09 sell_routine -> get_cached_targets -> filter_etfs
13:10 buy_routine  -> reuse targets cache
14:59 sync_positions
```

因此当前策略的信号可见性边界是：

- 卖出：13:09 才知道当前目标列表，13:09 及之后才是当前信号可执行卖出时点。
- 买入：13:10 复用 13:09 缓存目标，13:10 及之后才是当前信号可执行买入时点。
- 13:09 之前的买卖只属于 T-1 信号版本或诊断，不属于当前策略。

## 竞争性假设

| 假设 | 内容 | 可证伪条件 |
| --- | --- | --- |
| H1 信号可见性假设 | 13:09/13:10 不是神奇分钟，而是目标生成后的最早可执行时点 | 延后若稳定更好，则不能仅靠最早可执行解释 |
| H2 价格漂移假设 | 新目标在 13:10 后继续上涨，等待会付出价格漂移成本；旧标的卖出延后也无稳定收益 | 后续分钟买入 edge 为正且路径改善，则证伪 |
| H3 成交量假设 | 成交量只影响容量，不决定收益；高成交量窗口不必然更高收益 | edge 与 turnover 高相关且高容量分钟路径更优，则证伪 |
| H4 低波动假设 | 午盘/尾盘低波动只降低扰动，不保证买卖收益更好 | 低波动分钟若收益和回撤都更优，则证伪 |
| H5 统计噪声假设 | 13:09/13:10 只是样本偶然点，邻近分钟无显著差异 | 13:10 周边分钟等价，且更晚分钟不差，则支持 |

## 实验 1：分钟级时点扫描

输入：

- R010-B4 `b3_gate_tiered_v2` 四分段真实 filled trades。
- ClickHouse `jq_bar_minute_v2`，只读 SELECT。

对每笔实际订单，构造候选分钟：

```text
09:30-11:30
13:01-14:59
```

候选价格按实际成交分钟缩放到回测成交价口径：

```text
sell candidate price = raw_minute_close * actual_sell_price / raw_13:09_close
buy candidate price  = raw_minute_close * actual_buy_price  / raw_13:10_close
```

卖出 edge：

```text
sell_edge_bps = ln(candidate_price / actual_sell_price) * 10000
```

买入 edge：

```text
buy_edge_bps = ln(actual_buy_price / candidate_price) * 10000
```

正数代表候选分钟比当前实际执行更好。

## 数学解释

对同一笔订单、同一交易日、同一标的，若数量暂时视作不变：

卖出替代时点收益差：

```text
cash_delta_sell(t) = shares * (P_t - P_13:09)
edge_sell_bps(t)  = ln(P_t / P_13:09) * 10000
```

买入替代时点收益差：

```text
cash_delta_buy(t) = shares * (P_13:10 - P_t)
edge_buy_bps(t)  = ln(P_13:10 / P_t) * 10000
```

因此收益差的直接决定项是**相对决策时点的价格漂移**。成交量影响的是能否成交、冲击和滑点，不会自动带来更高收益。

本轮归因也支持这一点：

| 方向 | 因素 | 相关/系数结论 |
| --- | --- | --- |
| 卖出 | 价格漂移 | 与 edge 相关约 `+1.000`，数学同义 |
| 买入 | 价格漂移 | 与 edge 相关约 `-1.000`，数学同义 |
| 卖出 | `log1p(turnover)` | 相关约 `+0.012` |
| 买入 | `log1p(turnover)` | 相关约 `-0.018` |
| 卖出 | 等待分钟数 | 标准化 beta `-0.052` |
| 买入 | 等待分钟数 | 标准化 beta `+0.019` |

解释：成交量不是本轮收益差的主导解释。价格相对 13:09/13:10 的漂移才是核心；成交量更多用于容量约束。

## 为什么 13:09/13:10 不是“数学最优点”

当前证据不支持把 13:09/13:10 描述为统计学意义上的全局最优分钟。更准确的解释是：

1. `13:09` 是当前信号首次生成并可执行卖出的时间。
2. `13:10` 是卖出后买入的最早可执行时间。
3. 它们的优势首先来自**信号可见性和执行顺序约束**，不是来自成交量最大。
4. 延后时点中确实存在局部路径收益更好的候选，尤其 `14:45` 买入，但统计稳定性不足。

## 是否需要正式回测

本轮没有运行正式策略回测。原因是预注册门槛没有通过：

- 卖出 `13:21`：路径近似较好，但胜率低、中位 edge 为 0。
- 买入 `14:45`：路径近似较好，但胜率仅 `47.11%`，sign test 不显著，回撤不稳定。

严格按本实验规则：不直接进入正式回测，不修改策略。

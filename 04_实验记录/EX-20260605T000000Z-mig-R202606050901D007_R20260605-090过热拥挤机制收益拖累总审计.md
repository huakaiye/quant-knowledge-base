# R20260605-090 过热拥挤机制收益拖累总审计

## 2026-06-05 澄清

用户后续澄清，本轮真正问题是评分排名中 `score >= 5` 直接过滤是否合理，而不是 A2/B3/tiered-v2 这类泛化过热/拥挤机制。

因此，本记录只保留为 A2/B3/A9 的泛化机制审计，不作为 `score < 5` 上限过滤的主答案。`score < 5` 专项结论转入 [[R20260605-091_score大于5硬过滤合理性审计]] 与 [[D20260605-SCORECAP-001-score大于5硬过滤保留但进入替代研究]]。

## 本次问题

用户问题：当前策略的“过热机制”是否严重拖累收益能力，是否应当废除，或者改用其他方式做过热/拥挤判断。

本记录把研究库内的当前方向页、概念页、决策链路与平台已有回测结果合并复核。这里的“过热机制”必须拆成三类，不应混为一个开关：

1. A2-slope004：进攻端的持仓抗抖和追高约束。
2. B3 gate + tiered-v2：防守端的全弱拥挤/系统性退潮仓位上限。
3. R010-A9/S3/S5/S6：过滤层稀薄、Top3 替换、容量/二次保护的 shadow 候选。

## 结论

当前实盘默认组合不应废除过热/拥挤机制。

A2-slope004 没有严重拖累收益。它相对 B0 的总复合收益提升约 +314.71pct，但在 2020-2021 与 2025-2026 两段存在轻微机会成本，且几乎不改善最大回撤。因此 A2 应继续作为“抗抖/少追高”模块保留，但不能被解释成主要风控模块。

B3 gate + tiered-v2 是当前更有证据的拥挤/退潮防守模块。它相对 A2-slope004 同时提升总收益与最大回撤，且 live-enabled 审计已经通过。废除这层会把系统退回到更高回撤、更低复合收益的状态。

R010-A9 的 S3 过滤层稀薄 Top3 替换仍只能保持 shadow/default-off。S3 的事件归因有正边际，但高热窄主线分支 S4 已被证伪，S5/S6 也未通过实盘就绪。因此不能把 A9 作为新的实盘过热机制。

- A2 对最大回撤几乎没有贡献，所以后续文档与实盘解释中应把它归类为进攻端 anti-churn，而不是拥挤风险控制。

## 是否应当废除

不应废除。

废除 A2 会回到 B0，历史总复合收益少约 314.71pct，交易次数也上升。虽然 A2 在部分阶段有机会成本，但证据不支持把它定义为严重收益拖累。

废除 B3/tiered-v2 更不合理。当前证据显示它相对 A2 同时提升收益和降低最大回撤，是更接近“过热/拥挤防守”的有效模块。

需要废除或继续禁止的是旧的、证据不足的过热判断方式：

- 禁止恢复 A2-current-momentum 的朴素旧仓保留逻辑。
- 禁止把 S4 高热窄主线分支接入实盘。
- 禁止把单一热度阈值当成拥挤判断的主开关。

## 建议的过热/拥挤判断框架

后续不建议问“过热机制开不开”，而应拆成三层职责。

### 1. 进攻端抗抖层

保留 A2-slope004：

- 只在旧仓短斜率相对候选有足够优势时减少换仓。
- 目标是降低追高和噪声换仓。
- 不承担系统性退潮风控职责。

后续可做 read-only 研究：

- 在 A2 负贡献事件中，观察候选与旧仓的 `slope_short` 差、`ret5` 差、池内离散度、median_ret 与 breadth。
- 只研究“何时不该保留旧仓”，不直接扩大 A2 的自由度。

### 2. 防守端拥挤/退潮层

继续使用 B3 gate + tiered-v2：

- raw signal 先识别 all-weak persist。
- gate 使用 median_ret 或 ma20_breadth 过滤掉伪全弱。
- tiered-v2 再按 `cum_days` 与 `top5_exhausted` 分层到 cap80/cap70/cap60。

这层是当前唯一通过收益、回撤、动作日胜率和 live-enabled 审计的拥挤机制。

### 3. 过滤层稀薄替代层

R010-A9/S3/S5/S6 继续 shadow：

- S3 事件库继续积累。
- 容量 guard 必须继续纳入真实可交易性观察。
- 在 live-readiness 前不得替代 B3/tiered-v2。

## 决策

记录决策：[[D20260605-OVERHEAT-001-过热拥挤机制保留并拆分职责]]

本次研究结论为 revise：

1. 保留当前 A2-slope004 + B3-gate-tiered-v2 默认组合。
2. 不废除现有过热/拥挤机制。
3. 将 A2 从“过热风控”重新表述为“进攻端抗抖/少追高”。
4. 将 B3/tiered-v2 作为正式拥挤/退潮防守机制。
5. A9/S3/S5/S6 继续 default-off/shadow。
6. 后续禁止用单一热度阈值或 S4 高热分支替代当前机制。

## 后续研究动作

1. 建立 A2 负贡献事件面板，重点看 2020-2021 与 2025-2026 的机会成本来源。
2. 对 tiered-v2 的 cap60 极端分支继续做实盘日志审计，不扩大规则。
3. A9/S3 只扩展事件样本和容量审计，不进入实盘默认。
4. 每次 live 配置变化必须重新生成 live-enabled audit，并写入决策链路。

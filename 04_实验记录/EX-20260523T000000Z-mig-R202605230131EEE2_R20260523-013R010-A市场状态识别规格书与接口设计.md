# R20260523-013 R010-A市场状态识别规格书与接口设计

## 背景

回测平台和数据库仍在升级，当前不启动新回测。本研究用于把 R010-A 的需求提前固化，避免平台恢复后直接进入参数尝试。

## 本次产物

| 文档 | 用途 |
| --- | --- |
| [[R010-A_市场状态识别规格书]] | 定义 R010-A 的目标、非目标、状态、日志和验收标准 |
| [[市场状态信号接口设计]] | 定义输入、输出、信号函数和状态判定接口 |
| [[未来函数审计清单]] | 定义所有状态信号必须满足的时点约束 |

## 关键决策

| 决策 | 说明 |
| --- | --- |
| R010-A 只记录日志 | 不改变交易，先验证状态识别质量 |
| 默认信号日为 T-1 | T 日交易前只用 T-1 及以前数据 |
| 第一阶段使用规则状态机 | 不使用 HMM 或机器学习分类器 |
| 先做 Top5 动力确认 | Top1 弱时继续看 Top2 到 Top5，全弱再防守 |
| 日志字段必须完整 | 后续归因依赖每日状态日志 |

## 待平台恢复后执行

1. 新建独立市场状态模块。
2. 实现信号计算函数。
3. 写构造数据单元测试。
4. 接入 ETF 双池策略，只输出日志。
5. 用状态日志对齐已有回测收益做归因。
6. 再决定是否进入 R010-B 防守门控。

## 暂不做

- 不跑回测。
- 不接入交易。
- 不调阈值。
- 不引入外部宏观数据。
- 不用全样本分位数。

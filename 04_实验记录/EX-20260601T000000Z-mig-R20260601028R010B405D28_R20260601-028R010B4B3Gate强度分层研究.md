# R20260601-028 R010B4 B3Gate强度分层研究

> 编号说明：文件名沿用接手任务指定的 `R20260601-028`，研究子编号使用 `R010-B4`，避免与同日 Top1 斜率放松记录混淆。

## 研究信念

关联信念：[[BLF20260529-006_ETF阴跌防守与动量轮动研究信念]]

当前相信的机制：

[[B3 gate]] 已把全弱持续性信号中一部分噪音过滤掉。若 gate 通过后的风险状态更干净，则在部分 gate 日把风险仓上限从 cap80 进一步降到 cap70 或 cap60，可能继续改善回撤；但若所有 gate 日统一加重防守，可能错过趋势修复并牺牲收益。

先验置信度：中。

## 本次研究假设

在 `[[A2-slope004]] + B3-gate-median-or-breadth-cap80` 的基础上，B3 gate 通过后的风险日可以按风险强度分层：普通 gate 保持 cap80，强风险 gate 使用 cap70，极端风险 gate 使用 cap60。相较统一 cap80，分层强度应提升回撤控制，同时不显著牺牲复合收益。

## 可观察预测

- 预测 1：固定 cap70/cap60 应给出强度边界；若 cap60 明显降低回撤但收益大幅下降，说明强防守只能分层使用。
- 预测 2：tiered-v1 在 2024 的最大回撤应优于 B3-gate-cap80 的 -30.00%，复合收益不应低于 B3-gate-cap80 的 95%。
- 预测 3：动作日胜率应不低于 75%，且误伤日主要集中在风险快速修复阶段，而不是长期趋势延续阶段。
- 预测 4：若假设成立，至少 3/4 分段不明显劣于 B3-gate-cap80，2025-20260519 不出现灾难性收益拖累。

## 竞争性解释

| 竞争性解释 | 为什么可能成立 | 如何区分 |
| --- | --- | --- |
| 2024 特定回撤过拟合 | B3 gate 在 2024-06 到 2024-09 回撤窗口表现突出，强降仓可能只适配该窗口 | 必须做 2020-2021、2022-2023、2024、2025-20260519 四段验证 |
| 趋势修复误伤 | gate 日之后可能快速反弹，cap70/cap60 会少吃修复收益 | 输出误伤日列表、动作日平均超额和累计 log edge |
| 分层阈值只是强度代理 | `[[median_ret（中位收益）]]`、`[[ma20_breadth（广度）]]` 可能只是与全弱持续天数重复，不提供新增信息 | 比较固定 cap70/cap60 与 [[tiered-v2（分层强度防守）]] 的触发天数、收益保留率和回撤改善 |
| 交易路径扰动 | 降仓改变后续持仓路径，单日动作 edge 正但组合收益下降 | 同时审计动作日和非动作日贡献，分段看收益拖累 |
| 数据或未来函数问题 | 分层字段若使用未来窗口，会虚增效果 | 使用已有当日日志字段，做未来函数审计和 WSL dry-run |

## 偏差诊断

最大意外：固定 cap70/cap60 在 2022-2023、2024、2025-20260519 都改善整体收益和回撤，但动作日胜率在 2022-2023 和 2025-20260519 低于预注册 kill 线。

可能原因：

- 机制偏差：更强降仓并非每个 gate 日都更干净，而是少数冲击窗口贡献足够大，抵消了大量小误伤。
- 信号偏差：`median_ret` 与 `ma20_breadth` 阈值能够捕捉 2024 深回撤和 2025-04 冲击，但没有稳定刻画 2022-2023 的弱势震荡。
- 样本偏差：2024 与 2025-04 的冲击窗口权重较大，容易掩盖动作日胜率不足。
- 交易偏差：固定强度改变持仓路径，2020-2021 出现单日 edge 为正但全段收益和回撤变差。
- 统计偏差：强度升级天数少，特别是 tiered-v1 在 2025 只升级 2 天，单个大日会显著影响平均超额。

更简单的解释：B3 gate 后加大降仓强度确实能在深回撤和冲击窗口降低风险，但当前规则并不能稳定区分“应强降仓的 gate 日”和“可能快速修复的 gate 日”。因此固定 cap60/cap70 只适合作为强度边界参考，tiered-v1 需要修订后再测。

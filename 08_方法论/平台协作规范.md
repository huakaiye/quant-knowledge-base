# 平台协作规范

回测平台和实盘系统根路径不在共享文档中写死，统一使用逻辑根：

```text
${QUANT_PLATFORM_ROOT}
${LIVE_TRADING_ROOT}
```

本机路径通过环境变量或 `.research.local.json` 配置。解析回测平台路径：

```powershell
powershell -ExecutionPolicy Bypass -File tools/Get-QuantPlatformRoot.ps1 -Target Platform -Format All
```

解析实盘系统路径：

```powershell
powershell -ExecutionPolicy Bypass -File tools/Get-QuantPlatformRoot.ps1 -Target Live -Format All
```

回测平台默认示例为 `E:\量化平台_V1.4.0`，但这只是本机默认值，不是团队共享事实。实盘路径默认留空，必须由各自电脑本地配置后才能使用。

## 路径可移植规则

共享文档中的平台和实盘路径分两层：

| 层级 | 写法 | 是否入库 | 用途 |
| --- | --- | --- | --- |
| 平台逻辑根 | `${QUANT_PLATFORM_ROOT}` | 是 | 表示“本机回测平台根目录” |
| 实盘逻辑根 | `${LIVE_TRADING_ROOT}` | 是 | 表示“本机实盘系统根目录” |
| 平台相对路径 | `configs/research/...`、`results/v2/research/...` | 是 | 跨电脑稳定定位平台内资产 |
| 实盘相对路径 | `config/...`、`logs/...`、`reports/...` 等实盘系统内路径 | 是，但不得含账号、token 或券商私密配置 | 跨电脑说明实盘资产位置和审计证据 |
| 本机真实路径 | `E:\...`、`D:\...`、`/mnt/e/...` | 否，除历史证据外 | 只用于本机执行命令 |

新增实验、策略档案、台账和命令说明默认写平台或实盘系统相对路径。需要展示完整命令时，先用 `tools/Get-QuantPlatformRoot.ps1` 解析本机路径，再拼接相对路径。

历史迁移记录和已完成实验中已经存在的绝对路径，属于当时证据的一部分，可以原样保留；不要为了适配某台电脑批量改写历史路径。

## 职责边界

| 内容 | 放研究库 | 放平台 |
| --- | --- | --- |
| 研究方向、假设、决策 | 是 | 否 |
| 文献卡、因子灵感、术语 | 是 | 否 |
| 策略代码 | 否 | 是 |
| 回测配置 | 登记路径 | 是 |
| 回测结果 | 登记摘要和路径 | 是 |
| 大型 CSV、净值、交易明细 | 否 | 是 |
| 质量门禁报告 | 登记摘要和路径 | 是 |

## 回测命令

Agent/CLI 回测必须在 WSL 内运行：

```powershell
$platformWsl = powershell -ExecutionPolicy Bypass -File tools/Get-QuantPlatformRoot.ps1 -Format WSL
wsl -- bash -lc "cd '$platformWsl' && PYTHONPATH=src python3 src/run_v2_backtest.py --config configs/research/<rd_id>/<ex_id>/<config>.json"
```

如果是平台现有迁移配置，可使用平台已有入口，但实验记录必须写清配置路径。

## 可见进度

回测默认必须保持过程可见，尤其是子代理执行回测时。用户应能看到命令正在运行到哪一步、是否报错、日志写到哪里，以及如何判断进度是否正常。

长回测使用 `tee` 保存日志并保持进度可见：

```powershell
$platformWsl = powershell -ExecutionPolicy Bypass -File tools/Get-QuantPlatformRoot.ps1 -Format WSL
wsl -- bash -lc "cd '$platformWsl' && PYTHONUNBUFFERED=1 PYTHONPATH=src python3 src/run_v2_backtest.py --config configs/research/<rd_id>/<ex_id>/<config>.json 2>&1 | tee results/v2/research/<rd_id>/<ex_id>/<run_id>.run.log"
```

子代理回测必须在最终回复和实验记录中写明：

- 运行命令。
- 日志路径。
- 进度是否正常。
- 异常或卡住时的判断依据。
- 若后台运行，查看进度和停止进程的命令。

不允许无说明地使用过程不可见命令。确需后台运行时，必须先记录“后台回测豁免”：

```text
后台回测豁免：<原因>
进程标识：<pid或任务名>
日志路径：<path>
查看进度：<command>
停止方式：<command>
预计耗时：<duration>
```

## 禁止事项

- 不自动更新数据。
- 不无说明地使用过程不可见、无日志、无法监控的长回测命令。
- 不自动修改 MiniQMT 或实盘默认开关。
- 不把实盘账号、券商配置、Token、授权文件、客户端缓存或交易凭据写入研究库。
- 不把研究中的 shadow、observe、probe 标签升级为实盘信号。
- 不为追平结果而使用评分 override、交易 override 或后验行情修补。
- 不把平台结果文件复制进研究库。

## 实验记录中的平台字段

每份实验记录必须保留以下字段：

```yaml
platform_project: ${QUANT_PLATFORM_ROOT}
config_paths:
result_paths:
summary_paths:
quality_gate:
```

`config_paths`、`result_paths` 和 `summary_paths` 优先填写平台相对路径；只有引用历史证据、外部截图或不可移植旧记录时，才允许保留绝对路径，并写清“历史路径，仅作证据，不要求本机存在”。

没有运行的实验写 `未执行`，不要伪造结果。

# 平台协作规范

回测平台路径：

```text
E:\量化平台_V1.4.0
```

## 职责边界

| 内容 | 放研究库 | 放平台 |
| --- | --- | --- |
| 研究方向、假设、决策 | 是 | 否 |
| 文献卡、因子灵感、术语 | 是 | 否 |
| 策略代码 | 否 | 是 |
| 回测配置 | 登记路径 | 是 |
| 回测结果 | 登记摘要和路径 | 是 |
| 大型 CSV、净值、交易明细 | 否 | 是 |
| 质量门禁报告 | 登记摘要和路径 | 是 |

## 回测命令

Agent/CLI 回测必须在 WSL 内运行：

```bash
wsl -- bash -c "cd /mnt/e/量化平台_V1.4.0 && PYTHONPATH=src python3 src/run_v2_backtest.py --config configs/research/<rd_id>/<ex_id>/<config>.json"
```

如果是平台现有迁移配置，可使用平台已有入口，但实验记录必须写清配置路径。

## 可见进度

长回测建议使用 `tee` 保存日志并保持进度可见：

```bash
PYTHONUNBUFFERED=1 PYTHONPATH=src python3 src/run_v2_backtest.py --config configs/research/<rd_id>/<ex_id>/<config>.json 2>&1 | tee results/v2/research/<rd_id>/<ex_id>/<run_id>.run.log
```

## 禁止事项

- 不自动更新数据。
- 不自动修改 MiniQMT 或实盘默认开关。
- 不把研究中的 shadow、observe、probe 标签升级为实盘信号。
- 不为追平结果而使用评分 override、交易 override 或后验行情修补。
- 不把平台结果文件复制进研究库。

## 实验记录中的平台字段

每份实验记录必须保留以下字段：

```yaml
platform_project: E:\量化平台_V1.4.0
config_paths:
result_paths:
summary_paths:
quality_gate:
```

没有运行的实验写 `未执行`，不要伪造结果。

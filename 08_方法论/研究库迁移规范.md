# 研究库迁移规范

旧研究库路径：

```text
E:\【笔记库】\量化研究库
```

旧库有价值的是研究事实、实验结果、文献线索、策略经验和决策脉络；不复用旧目录架构。

## 迁移原则

1. 先建映射卡，再搬内容。
2. 只迁移经过复核的事实。
3. 不把旧编号当作新唯一编号。
4. 不复制旧库的目录结构和视觉符号。
5. 不把旧结论直接升级为新库当前结论。

## 迁移步骤

1. 在 `11_迁移暂存/` 新建 `MIG` 迁移卡。
2. 记录旧文件路径、旧编号、主题、涉及策略、涉及平台结果路径。
3. 判断应迁入哪类新资产：`RD`、`EX`、`DEC`、`LIT`、`FAC`、`DATA`、`TERM`。
4. 新建对应资产，保留旧路径作为 `source_old_path`。
5. 更新对应台账。
6. 如果影响当前路线，更新驾驶舱和方向页。

## 迁移状态

| 状态 | 含义 |
| --- | --- |
| `todo` | 发现旧资产，未复核 |
| `mapped` | 已建立迁移卡和目标类型 |
| `migrated` | 已迁入新库 |
| `verified` | 已核对平台路径和结果摘要 |
| `discarded` | 不迁移，并写清原因 |

## 禁止迁移方式

- 整个旧目录直接复制到新库。
- 只按文件名猜测结论。
- 只迁移收益指标，不迁移假设和证伪条件。
- 迁移后不更新台账。

# 命名与编号规范

## 为什么不用顺序号

多人或多个 Agent 并行研究时，如果使用 `001、002、003` 这类全局顺序号，两个 PR 很容易创建同一个编号。合并后再修编号会破坏链接和台账。

本库所有核心资产使用不依赖中心分配的唯一 ID。

## 通用 ID 格式

```text
<类型前缀>-<UTC时间戳>-<agent标识>-<随机码>
```

示例：

```text
RD-20260605T115651Z-main-DP00
EX-20260605T115651Z-main-BASE
DEC-20260605T115651Z-main-SYS0
```

字段解释：

| 字段 | 说明 |
| --- | --- |
| 类型前缀 | 资产类型，例如 `RD`、`EX`、`DEC` |
| UTC 时间戳 | 创建瞬间的 UTC 时间，格式 `YYYYMMDDTHHMMSSZ` |
| agent 标识 | 创建者或子代理标识，例如 `main`、`zicha001`、`zyan001` |
| 随机码 | 4 到 8 位大写字母或数字，用于防止同秒冲突 |

## 类型前缀

| 前缀 | 类型 | 保存位置 |
| --- | --- | --- |
| `RD` | 研究方向或模块子方向 | `02_研究方向/` |
| `STRAT` | 策略档案 | `03_策略档案/` |
| `EX` | 实验记录 | `04_实验记录/` |
| `DEC` | 研究决策卡 | `05_研究决策/` |
| `LIT` | 文献卡片 | `06_文献资料/` |
| `IDEA` | 模块想法 | `07_因子数据灵感/04_模块/` |
| `FAC` | 因子灵感 | `07_因子数据灵感/01_因子/` |
| `DATA` | 数据灵感 | `07_因子数据灵感/02_数据/` |
| `MECH` | 机制灵感 | `07_因子数据灵感/03_机制/` |
| `TERM` | 术语词条 | `09_术语库/` |
| `MIG` | 迁移卡片 | `11_迁移暂存/` |
| `RUN` | 平台运行标识 | 平台 `results/v2/research/` |

## 文件名格式

```text
<ID>_<中文短标题>.md
```

标题要求：

- 中文优先。
- 20 个汉字以内更好。
- 不使用 `/ \ : * ? " < > |` 等 Windows 非法字符。
- 不把结论写进标题，除非是决策卡。

示例：

```text
RD-20260605T115651Z-main-DP00_双池轮动策略.md
EX-20260605T115651Z-main-BASE_双池轮动基线资产盘点预注册.md
DEC-20260605T115651Z-main-SYS0_研究库V2架构立项通过.md
```

## 研究方向与子方向

策略级方向：

```yaml
rd_id: RD-...
parent_rd_id:
scope: 策略
```

模块级子方向：

```yaml
rd_id: RD-...
parent_rd_id: RD-父方向ID
scope: 模块
module_type: 进攻模块
```

模块可以作为研究方向，但必须满足两个条件：

1. 它有独立假设、独立实验和独立决策价值。
2. 它通过 `parent_rd_id` 连接到所属策略，不能漂浮成孤立文档。

## 平台配置和结果命名

研究库只登记平台路径，不复制大文件。推荐平台路径：

```text
E:\量化平台_V1.4.0\configs\research\<rd_id>\<ex_id>\<run_id>.json
E:\量化平台_V1.4.0\results\v2\research\<rd_id>\<ex_id>\<run_id>\
```

`run_id` 推荐格式：

```text
RUN-<UTC时间戳>-<agent标识>-<随机码>_<英文短名>
```

英文短名只服务于 shell、JSON 和结果目录，研究解释仍写中文。

## 台账更新规则

新增核心资产时必须同步更新对应台账：

| 新增内容 | 必改台账 |
| --- | --- |
| 研究方向 | `01_台账/研究方向台账.csv` |
| 策略档案 | `01_台账/策略资产台账.csv` |
| 实验记录 | `01_台账/实验台账.csv` |
| 决策卡 | `01_台账/决策台账.csv` |
| 文献卡 | `01_台账/文献台账.csv` |
| 术语 | `01_台账/术语台账.csv` |

台账是检索入口，不替代正文。正文负责解释，台账负责快速定位。

## 归档规则

满足任一条件时，应考虑归档：

- 方向已 `kill`，且短期不会恢复。
- 实验记录是历史版本，被新实验完全取代。
- 文献灵感已转化为因子卡、数据卡或模块卡。
- 旧库迁移卡已经完成映射和复核。

归档不等于删除。归档后的文档仍保留 ID，台账状态改为 `archived`，并在正文写清归档原因。

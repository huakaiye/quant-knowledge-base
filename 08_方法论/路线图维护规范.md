# 路线图维护规范

Markdown Mermaid 图不适合承载长期增长的全局路线图。节点变多后，Obsidian 预览区无法方便缩放，阅读体验会迅速变差。

本库采用“Canvas 看全局，Markdown 讲规则，台账做检索”的路线图方案。

## 推荐工具分工

| 工具 | 用途 | 节点规模 |
| --- | --- | --- |
| Obsidian Canvas | 全局路线、方向关系、关键决策路径 | 10 个以上节点 |
| Markdown Mermaid | 小型缩略图、局部流程示意 | 10 个以内节点 |
| 台账 CSV | 精确检索、状态、路径、ID | 不限 |
| 方向页 | 解释研究信念、证据和下一步 | 单方向 |
| 决策卡 | 解释为什么改变路线 | 单决策 |

## 全局 Canvas 规则

全局 Canvas 路径：

```text
00_入口/研究路线图.canvas
```

全局 Canvas 只放：

- 研究驾驶舱。
- 策略级研究方向。
- 模块级研究方向。
- 当前关键实验。
- 当前关键决策。
- 必要的方法论入口。

不要把每一次参数扫描、每一个结果文件、每一个文献卡都塞进全局 Canvas。那会让图重新失控。

## 分层拆图规则

满足任一条件时，应新建子 Canvas：

- 单个策略方向超过 20 个实验节点。
- 单个模块方向超过 10 个实验节点。
- 全局 Canvas 中同一块区域需要缩放到很大才能读清。
- 某条路线已经有多个分叉，例如进攻、防御、执行各自持续演化。

子 Canvas 推荐命名：

```text
02_研究方向/<rd_id>_路线图.canvas
```

全局 Canvas 中只保留该子 Canvas 的入口节点。

## 节点粒度

| 层级 | 全局 Canvas 是否展示 | 说明 |
| --- | --- | --- |
| 策略级方向 | 展示 | 全局主干 |
| 模块级方向 | 展示 | 策略下的主要分叉 |
| 关键决策 | 展示 | 改变路线或状态的节点 |
| 当前关键实验 | 展示 | 当前正在推进或刚影响方向的实验 |
| 普通实验 | 不展示 | 放实验台账和子 Canvas |
| 文献和因子灵感 | 通常不展示 | 只有变成关键假设时才展示 |
| 平台结果目录 | 不展示 | 放策略档案和实验记录 |

## 维护流程

新增方向或关键决策后：

1. 更新对应正文。
2. 更新 `01_台账/`。
3. 更新 `00_入口/研究驾驶舱.md`。
4. 更新 `00_入口/研究路线图.canvas` 或对应子 Canvas。
5. 若节点太多，拆分子 Canvas，而不是继续压缩全局图。

## 给 Agent 的要求

- 不要把 Canvas 当唯一事实源。
- Canvas 节点必须能链接到正文文件。
- 大图只表达路线关系，不承载完整实验结论。
- 结论仍以实验记录、决策卡和台账为准。

# 子代理调度规范

子代理用于降低高等级模型用量，但不能降低研究质量。

## 模型分工

| 层级 | 模型 | 适合任务 | 禁止事项 |
| --- | --- | --- | --- |
| 子查 | `gpt-5.3-codex-spark` | 检索、文件清单、路径核对、格式检查、字段摘录 | 不做路线判断，不写最终结论 |
| 子研 | `gpt-5.4` | 阅读一组材料、归纳证据链、草拟实验记录或决策卡 | 不独立决定 promote/revise/park/kill |
| 子执 | 视任务选择 | 受控执行脚本或回测，必须展示日志和进度 | 不自动更新数据，不改实盘 |
| 主控 | 主线程或 `gpt-5.5` | 研究假设、证伪、未来函数审计、实盘边界、最终决策 | 不把关键判断完全外包 |

## 任务代号

委派提示词第一行必须写：

```text
任务代号：<前缀>-<三位序号>-<中文短名>
```

示例：

```text
任务代号：子查-001-R010结果检索
任务代号：子研-001-双池证据归纳
任务代号：子执-001-可见进度回测
任务代号：主控-001-路线终审
```

## 强制触发条件

以下场景默认必须调用至少一个子代理，除非主控写明豁免原因：

- 复核旧库迁移实验、决策或平台结果。
- 核对平台配置路径、结果路径、字段可见性或样本区间。
- 整理多篇文献、多条实验记录或多张决策卡的证据链。
- 修改会影响研究路线、驾驶舱、策略档案或台账的文件。
- 发布到 GitHub、处理合并风险或检查 Obsidian/Git 配置。
- 对未来函数、过拟合、成本、样本外、消融或负控做前置材料检查。

## 子执回测可见性

子代理执行回测时，默认必须使用过程可见命令。主控委派时必须要求子代理提供：

- 实际运行命令。
- 日志文件路径。
- 当前进度或阶段输出。
- 失败时的错误输出。
- 用户或主控如何查看进度。
- 必要时如何停止进程。

推荐命令形态：

```bash
wsl -- bash -lc 'cd /mnt/e/量化平台_V1.4.0 && PYTHONUNBUFFERED=1 PYTHONPATH=src python3 src/run_v2_backtest.py --config configs/research/<rd_id>/<ex_id>/<config>.json 2>&1 | tee results/v2/research/<rd_id>/<ex_id>/<run_id>.run.log'
```

禁止默认使用以下不可见方式：

- 只返回最终结果、不展示中间日志的封装命令。
- 无日志路径的后台运行。
- 静默重定向到黑洞或只保留最终摘要。
- 让主控或用户无法判断是否卡住的长时间命令。

如果确需后台运行，子代理必须先写：

```text
后台回测豁免：<原因>
进程标识：<pid或任务名>
日志路径：<path>
查看进度：<command>
停止方式：<command>
预计耗时：<duration>
```

允许豁免的场景：

- 纯文字错别字、格式或链接修复。
- 只读回答一个非常小的事实问题。
- 用户明确要求不要调用子代理。
- 当前工具环境没有可用子代理工具。

豁免必须写成：

```text
子代理豁免：<原因>；主控：<agent>；时间：<UTC时间>
```

## 主控调度计划

开始复杂任务前，主控必须先写一句调度计划：

```text
子代理计划：调用；任务代号：子查-001-旧库复核队列；模型：gpt-5.3-codex-spark；交付物：候选文件清单。
```

或：

```text
子代理计划：不调用；豁免原因：纯格式修复，不涉及实验、决策、路径或发布。
```

## 子代理最终回复必须包含

- 任务代号。
- 使用模型。
- 读取文件。
- 修改文件。
- 执行命令。
- 结论边界。
- 风险点。

## 调用记录

凡影响实验、决策、驾驶舱、策略档案、平台路径或 Git 发布的子代理调用，主控必须在相关文档中保留“子代理调用记录”区块，并同步 `01_台账/子代理调用台账.csv`。

文档内记录格式：

```markdown
## 子代理调用记录

- 任务代号：
- 模型：
- 发起时间：
- 读取文件：
- 修改文件：
- 执行命令：
- 结论边界：
- 风险点：
- 主控复核：
- 结果对决策影响：
```

台账字段：

```csv
call_id,task_code,model,caller,target_docs,inputs,outputs,parent_ex_id,parent_dec_id,status,reviewed_by,risk_flag,timestamp_utc,novice_summary
```

`completed` 实验或影响路线的 `completed` 决策必须能找到子代理调用记录，或写明子代理豁免原因。

## 可并行的任务

可以并行拆给 `子查`：

- 检索某个 `rd_id`、`ex_id`、`dec_id`。
- 列出某个方向的全部实验记录。
- 核对平台配置路径和结果路径是否存在。
- 检查模板字段是否缺失。
- 统计台账重复 ID。

不应并行拆分给多个子代理的任务：

- 同一份正式决策的最终判断。
- 是否进入实盘或影子观察。
- 是否确认无未来函数。
- 是否杀死或提升一个研究方向。

## 主控复核清单

使用子代理结果前，主控必须复核：

- 路径是否真实存在。
- 引用是否来自当前文件，而不是记忆。
- 是否混淆正式回测、代理审计、样本外、shadow 和实盘。
- 是否遗漏反证、成本、换手、消融和未来函数。
- 是否把后验最优参数包装成机制结论。

质量优先于节省用量。关键判断必须由主控承担。

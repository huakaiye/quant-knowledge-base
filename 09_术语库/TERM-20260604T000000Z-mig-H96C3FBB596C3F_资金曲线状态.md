# 资金曲线状态

## 定义

资金曲线状态是指：根据策略组合净值本身的路径，判断当前是否处于健康上涨、正常回撤、深回撤、修复中或低仓锁定等状态。

它不同于单纯的组合回撤。组合回撤只描述当前净值离历史高点有多远；资金曲线状态还必须描述恢复速度、是否正在修复、是否因为降仓导致回本变慢，以及何时应解除低仓。

## 当前研究边界

在 R010-B5S1 中，资金曲线状态对应非线性仓位公式中的 `S_t`：

```text
w_t = (sigma_target / sigma_t) * tanh(k * m_t) * exp(-a * D_t) * S_t
```

当前平台已有 `portfolio_value`、`portfolio_peak`、`portfolio_drawdown` 等字段，但没有独立的 `capital_curve_state` 或 `capital_curve_scale`。因此 `S_t` 只能作为未来预注册模块，不得直接接入仓位公式。

R010-B5S2 已经从 B3-gate-tiered-v2 拼接净值中只读派生 T-1 资金曲线状态诊断，并覆盖 27/27 个 R010-B5 counterfactual 事件。该状态当前只有诊断含义：24 个事件处于 `deep_drawdown_not_recovering`，3 个事件处于 `recovering_deep_drawdown`。由于后者样本极小，且没有 `capital_curve_scale`，不得把它写成补仓规则。

R010-B5S3 进一步固定分组诊断后，`recovering_deep_drawdown` 仍只有 3 个事件，不能形成补仓规则；`deep_drawdown_not_recovering` 24 个事件 5日与10日 edge 均值偏负。因此资金曲线状态当前不支持进入完整公式回测或交易化。

## 必须区分

| 对象 | 含义 |
| --- | --- |
| 组合回撤 `D_t` | 净值相对历史高点的回撤幅度 |
| 资金曲线状态 `S_t` | 净值路径所处状态，包括修复、锁仓和恢复条件 |
| 风险窗口 | B3 gate / tiered-v2 对市场风险的识别 |
| 实盘止损 | 账户或持仓亏损触发的执行保护 |

## 主要风险

| 风险 | 说明 |
| --- | --- |
| 与回撤重复 | 若 `S_t` 只是回撤的另一个写法，会重复 `D_t`，不能提供新信息 |
| 低仓锁定 | 深回撤后继续压仓，可能导致组合修复速度下降 |
| 后验恢复规则 | 看到反弹后再定义恢复条件，容易过拟合 |
| 实盘误用 | 没有日志字段、负控制和前向样本前，不得接入 MiniQMT |

## 下一步要求

若未来研究 `S_t`，必须先预注册：

- `capital_curve_state` 的状态定义；
- `capital_curve_scale` 的输出范围；
- 恢复和解除低仓锁定的条件；
- `source_lag_days`，证明状态只使用 T-1 可见信息；
- 负控制，证明它不是简单复述 `portfolio_drawdown`。

# 离线归因

## 定义

离线归因是指：**在不启动新回测、不访问数据库、不改变交易逻辑的前提下，复用既有回测日志、状态面板、动作记录和净值数据，分析某个信号或动作在历史样本中的命中率、误伤成本、覆盖率和分段稳健性**。它是一种"用已有数据回答问题"的研究方法。

## 与状态归因的区别

| 维度 | 离线归因 | [[状态归因]] |
|------|---------|------------|
| 分析对象 | 信号或动作的效果 | 策略在不同市场状态下的表现 |
| 核心问题 | "这个信号真的有用吗？" | "策略在哪种市场中赚钱/亏钱？" |
| 数据来源 | 既有回测日志 | 市场状态标签 + 策略收益序列 |
| 分组方式 | 按信号/动作是否触发分组 | 按市场状态标签分组 |
| 典型输出 | 命中率、edge、误伤率 | 各状态下收益、夏普、最大回撤 |

两者配合使用：状态归因发现"策略在阴跌中持续亏损"，离线归因验证"若在阴跌中降仓到 60%，能省下多少回撤？"

## 具体操作步骤

### 步骤 1：收集数据源
回测日志（trade_log）、状态面板（state_panel）、净值序列（nav_series）、动作记录（action_log）、影子日志（shadow_log）

### 步骤 2：定义分析
```
分析问题：如 "A 信号触发的日子，后续 10 日净值表现如何？"
反事实：
  - 路径 A（触发）：触发日的实际净值
  - 路径 B（基准）：非触发日的平均净值
  - 比较指标：log edge = ln(路径A净值 / 路径B净值)
```

### 步骤 3：提取事件窗口
提取信号触发日列表 → 提取前后 N 日净值 → 检查事件重叠 → 提取控制变量

### 步骤 4：计算诊断
| 指标 | 期望 |
|------|------|
| 命中率（edge > 0 的占比） | > 50% |
| 累计 edge | > 0 |
| 误伤成本 | 越低越好 |
| 分段稳健性 | 各分段不方向翻转 |

### 步骤 5：形成假设
- 正面（命中率 > 70%）→ 进入正式消融测试或预注册回测
- 负面（命中率 < 40%）→ 信号可能无效
- 模糊（命中率 ~50%）→ 样本不足或信号无区分力

## 离线归因的边界

| 限制 | 说明 |
|------|------|
| 只能产生假设 | 不能直接用于实盘上线 |
| 选择性偏差 | 既有回测参数已固定，不能探索"如果参数不同" |
| 不能替代正式回测 | 正式回测需重新跑完整交易模拟 |
| 必须证明数据新鲜度 | 如果使用的是优化时"见过"的数据，则无独立性 |

## 在本研究库中的实践

离线归因在 R010-B5 系列中被频繁使用：
- **R010-B5S2**：从拼接净值派生波动率项和资金曲线状态
- **R010-B5R5**：严格反事实审计本身就是离线归因——复用影子日志，逐事件重建反事实净值

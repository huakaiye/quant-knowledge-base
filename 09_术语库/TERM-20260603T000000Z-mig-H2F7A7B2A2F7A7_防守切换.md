# 防守切换

## 定义

防守切换是指：**当市场趋势信号、动量优势或广度条件恶化时，策略从满仓进攻状态切换到一个更保守的仓位或资产配置状态**。切换对象可以是低仓位（降 cap）、防守 ETF、现金，或者降低换手频率。

## 当前策略中的切换链路

```
[A2-slope004] 选股端输出候选池
       ↓
[B3全弱信号] 原始风险检测：Top5 ETF 短/中/长动量是否全线走弱？
       ↓
[B3 gate] 二次确认：用 median_ret 和 ma20_breadth 过滤假信号
       ↓
[tiered-v2] 强度分层：根据全弱持续天数和恶化程度决定 cap
       ↓
最终仓位上限
```

设计哲学是 **"信号从宽到严，动作从轻到重"**：B3 raw 发出大量假警报但不可漏过风险，gate 过滤约 36%（44/122 天）的噪音触发，tiered-v2 在确认的风险日内分三档降仓而非一刀切。

## 与 B3 gate / tiered-v2 的层级关系

防守切换在本策略中表现为**三层金字塔**：

```
第一层：防守切换（抽象概念层）
        描述"什么情况下从进攻切换到防守"

第二层：B3 gate（确认层，决定"是否切换"）
        用 median_ret 和 ma20_breadth 做实时的风险确认

第三层：tiered-v2（执行层，决定"切换多少"）
        分三档降仓：cap80 / cap70 / cap60
```

## 过度防守风险

核心陷阱：轻微走弱期间频繁降仓导致错过反弹、交易成本累积、低仓锁定（降仓后回本更慢）。tiered-v2 的三档设计（cap80 只降 20% 仓位）部分缓解了这个问题。

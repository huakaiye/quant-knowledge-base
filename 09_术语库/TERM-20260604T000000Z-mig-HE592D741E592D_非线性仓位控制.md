# 非线性仓位控制

> 本页是概念词条。当前研究方向页见 [[非线性仓位控制]]。

## 定义

非线性仓位控制是指：仓位不是按几个固定阈值硬切换，而是通过连续函数随风险状态变化。常见形式包括指数衰减、双曲正切、S 型函数和目标波动率缩放。

在当前 ETF 双池研究中，非线性仓位控制只作为组合层风控研究问题，不是默认实盘开关。

## 与硬分档的区别

| 对象 | 说明 |
| --- | --- |
| 硬分档仓位 | 回撤到某个阈值后直接切换到固定 cap，例如 80%、60% |
| 非线性仓位 | 仓位随回撤、波动率或趋势状态连续变化，减少跳变 |
| 主要优势 | 降低低仓锁定和反弹误伤概率 |
| 主要风险 | 参数更多，容易过拟合；必须预注册和分段验证 |

## 当前研究用法

R010-B5R1 只使用回撤反馈部分：

```text
risk_scale = max(floor, exp(-decay * max(drawdown_abs - threshold, 0)))
```

该公式只在 B3/tiered-v2 已确认风险的窗口内作为二级预算候选，不单独驱动交易。

## R010-B5S1 公式组件化

R010-B5S1 将用户提出的完整非线性仓位公式拆成四个组件：

```text
w_t = (sigma_target / sigma_t) * tanh(k * m_t) * exp(-a * D_t) * S_t
```

| 组件 | 当前边界 |
| --- | --- |
| `sigma_target / sigma_t` | 当前只有 `realized_vol` 风险代理，不等同组合级 `sigma_t`；`sigma_target` 缺失 |
| `tanh(k*m_t)` | 有 `median_ret`、`ma20_breadth` 等代理，但低 `median_ret` 缺独立样本外 |
| `exp(-aD_t)` | 已实现为 R010-B5 回撤反馈，但严格 counterfactual 后仍观察级 |
| `S_t` | 对应[[资金曲线状态]]，当前没有独立字段或模块 |

因此完整乘法公式当前只能作为组件级审计规格，不得直接回测、不得进入 R010-B5R6、不得实盘。

## R010-B5S2 只读组件审计

R010-B5S2 进一步确认：

| 组件 | S2 结果 | 当前边界 |
| --- | --- | --- |
| 组合级 `sigma_t` | 可从 B3-gate-tiered-v2 拼接净值派生 T-1 `sigma_t20_ann_t1` 与 `sigma_t60_ann_t1`，覆盖 27/27 事件 | 仍没有 `sigma_target`，不能形成 `vol_scale` |
| `S_t` 资金曲线状态 | 可派生 T-1 `capital_curve_state_t1`，覆盖 27/27 事件 | 仅诊断，不决策；没有 `capital_curve_scale` |

S2 的判断是 `observe_component_readonly_audit_inputs_ready_no_formula_backtest`。这表示输入可用于组件诊断，但完整乘法公式仍不得回测、不得进入 R010-B5R6、不得实盘。

## R010-B5S3 组件诊断结果

R010-B5S3 对 27 个事件做固定分组诊断后，主组件 watch 组为 0。自然波动率分组和资金曲线修复状态均不足以推动完整公式。

| 线索 | 结果 | 边界 |
| --- | --- | --- |
| 自然波动率分组 | `sigma20 > sigma60` 10日正率不足；短长波动率分组未通过 | 不支持波动率组件交易化 |
| 资金曲线修复 | `recovering_deep_drawdown` 仅 3 事件 | 不能写成补仓规则 |
| rolling sigma mid | 参考分组偏正 | 只能等待新增样本确认 |
| 负控制 | 未污染 | 说明诊断未被明显 placebo 反转，但不足以升级 |

S3 的判断是 `observe_no_component_support_no_formula_backtest`。完整乘法公式路线应暂停。

## R010-B5S4 前向样本可用性

R010-B5S4 进一步审计现有结果是否已经有新增前向或独立事件样本，结论是否定的。

| 审计项 | 结果 | 含义 |
| --- | --- | --- |
| 审计资源数 | 9 | 覆盖 S2/S3/R5/R5R2/R4 shadow/R5R3 等事件资源 |
| 合格独立资源数 | 0 | 没有可用于确认 rolling sigma mid 的独立样本 |
| S3 后新增结果目录数 | 0 | 没有新的平台结果目录可作为前向验证 |
| 事件日期重叠 | 全部重叠 S3 27 个事件 | 同日复核只能复现旧结论 |

S4 的判断是 `park_forward_confirmation_until_new_independent_samples_no_formula_backtest`。这意味着非线性仓位控制当前不能顶替任何防御模块；后续只有新增事件不少于 8 个、覆盖不少于 2 个分段且不与 S3 日期重叠时，才允许只读确认 `sigma20_rank_state=mid` 与 `sigma60_rank_state=mid`。

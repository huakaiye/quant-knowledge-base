# 状态切换

## 定义

状态切换是指：**策略根据实时识别的市场状态，在不同的运行模式（如进攻满仓、防守降仓、低换手、不同排序模型）之间进行切换**。它是 [[市场环境过滤]] 和 [[市场状态识别]] 的结果输出层。

## 切换边界条件

| 要素 | 说明 | 例子 |
|------|------|------|
| 进入条件 | 满足什么条件切换到目标状态 | B3 gate active → 进入防守 |
| 退出条件 | 满足什么条件切回正常状态 | B3 gate inactive → 恢复进攻 |
| 最短停留期 | 进入后至少停留多少天 | tiered-v2 的 cum_days ≥ N |
| 冷却期 | 退出后至少等待多久才允许再次切换 | 防止反复切换的 hysteresis |
| 渐进/跳变 | 逐步过渡还是一次到位 | tiered-v2 三档分级 vs 直接空仓 |

## 过度切换风险

### 1. 来回摇摆（Whipsaw）
市场环境在边界附近反复波动 → 每天在"进攻"和"防守"间切换 → 交易成本和信号摩擦吞噬收益。

### 2. 低仓锁定
进入防守后退出条件过严 → 长期停留于低仓 → 错过反弹 → 回本周期被拖长。

### 3. 过度分层
同时切换多个维度 → 组合效应无法审计 → 不知道哪个动作真正贡献了改善。

### 缓解措施
- **Hysteresis**：退出条件比进入条件更严格
- **最短停留期**：进入后至少停留 N 天
- **渐进切换**：tiered-v2 沿 80→70→60 梯度下降

## 与市场状态识别的层级关系

```
市场状态识别（研究诊断层）→ 输出状态标签，事后归因

状态切换（决策执行层）→ 输入实时可见指标，输出仓位上限
```

关键设计原则：**状态识别用于研究诊断（用事后信息），状态切换用于实盘决策（只用 T-1 可见信息）**。两者不应混淆。

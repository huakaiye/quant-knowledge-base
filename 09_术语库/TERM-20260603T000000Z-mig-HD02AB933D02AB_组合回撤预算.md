# 组合回撤预算

## 定义

组合回撤预算是指：不只看单个信号或单个 ETF，而是把当前策略组合净值相对历史高点的回撤作为风险预算输入。当组合回撤加深时，降低风险资产仓位上限；当组合回撤修复时，再逐步恢复风险仓。

在 R010-B5 中，它被定义为 B3-gate-tiered-v2 之上的组合层风控覆盖，而不是替代 B3 gate 或 tiered-v2 的新主触发器。

## 关键区别

| 对象                  | 含义                    |
| ------------------- | --------------------- |
| B3 gate / tiered-v2 | 由 ETF 池内状态决定是否降仓、降到多强 |
| 组合回撤预算              | 由策略净值路径决定风险仓上限        |
| 波动率缩放               | 由波动率或目标风险决定仓位         |
| 实盘止损                | 由账户亏损或持仓损失触发保护动作      |

## 主要风险

| 风险 | 说明 |
| --- | --- |
| 低仓锁定 | 降仓后组合回本更慢，可能长期停留在低仓状态 |
| 错过反弹 | 大幅反弹常发生在深回撤后，机械低仓会削弱修复速度 |
| 与既有防守重复 | B3 gate/tiered-v2 已经在弱势日降仓，组合层再降仓可能过度防守 |
| 代理回测偏差 | 净值层缩放不是正式路径回测，不能替代持仓、交易和防守 ETF 路径验证 |

## R010-B5 首轮结论

首轮净值层代理审计显示，机械回撤预算可以改善最大回撤，但收益保留率明显不足。最激进版本将最大回撤改善约 5.77pp，但收益保留率仅约 37.20%；极保守版本收益保留率约 85.87%，但回撤改善仅约 0.93pp。

因此当前结论不是上线，而是修订研究问题：若继续，应优先研究“只在 B3/tiered 已确认风险窗口内叠加组合回撤预算”，而不是全周期机械按净值回撤降仓。

## R010-B5R1 非线性修订

R010-B5R1 将硬分档修订为条件化非线性缩放：

```text
risk_scale = max(floor, exp(-decay * max(drawdown_abs - threshold, 0)))
```

核心差异：

| 项目 | R010-B5硬分档 | R010-B5R1非线性 |
| --- | --- | --- |
| 触发范围 | 全周期按组合回撤触发 | 只在 B3/tiered 风险窗口内触发 |
| 仓位变化 | 100/90/80/60/40 硬切换 | 指数函数连续缩放 |
| 主要风险 | 长期低仓锁定 | 样本较少、真实路径未验证 |
| 首轮代理结论 | revise | promote_proxy_candidate_for_true_path_no_live |

首轮代理审计中，`nl_endo_gate_any_d10_a5_f70` 复合收益 +1581.80%，最大回撤 -25.99%，收益保留率 107.73%，回撤改善 +1.14pp，动作日胜率 81.48%。该结果只允许进入真实路径回测预注册，不允许实盘。

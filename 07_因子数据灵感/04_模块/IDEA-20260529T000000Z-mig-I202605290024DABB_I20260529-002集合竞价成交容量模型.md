# I20260529-002 集合竞价成交容量模型

## 一句话描述

为依赖[[集合竞价]]买卖的短线策略建立真实可成交约束，防止把一字板、排队优势和过大下单金额产生的虚假收益纳入回测。

## 来源

来源文献：[[2025_权仓量化_价量因子核心窗口涨停股应用]]

来源观察：原文章的策略逻辑高度依赖次日早盘集合竞价买入和固定持有两日后集合竞价卖出，因此如果没有成交容量模型，收益可信度不足。

## 双标签

收益来源：[[事件驱动类策略]]

执行方式：[[集合竞价]]、[[风控叠加]]

## 模块定义

模型输入：

- 目标委托金额。
- T+1 集合竞价成交额或开盘首分钟成交额代理。
- T+1 开盘价、涨停价、跌停价。
- 标的是否一字涨停或一字跌停。
- 设定参与率上限，例如不超过可见成交额的 1%-5%。

初始成交规则：

| 情况 | 买入处理 | 卖出处理 |
| --- | --- | --- |
| 一字涨停 | 不成交 | 持仓不受影响 |
| 一字跌停 | 不买入 | 不可卖出或部分成交为 0 |
| 委托金额超过容量 | 按容量上限部分成交 | 按容量上限部分成交 |
| 正常开盘 | 按开盘价加滑点成交 | 按开盘价减滑点成交 |

## 适用策略

| 策略 | 适用方式 | 备注 |
| --- | --- | --- |
| [[涨停事件价量结构策略]] | 必备执行约束 | 未建模前不得采纳任何收益结论 |
| [[小市值日内做T]] | 可复用容量约束 | 尤其适合低流动性小票 |

## 数据需求

| 数据 | 当前是否具备 | 说明 |
| --- | --- | --- |
| 集合竞价成交额 | 待确认 | 最理想口径 |
| 开盘首分钟成交额 | 待确认 | 集合竞价缺失时的保守代理 |
| 涨跌停价格 | 待确认 | 判断不可买卖 |
| 分钟线成交额 | 待确认 | 判断开盘流动性 |

## 预期改善

- 收益：会降低纸面收益，但提高可信度。
- 回撤：可减少不可卖出或流动性踩踏的低估。
- 换手：不直接改变信号，但可能降低有效成交次数。
- 稳定性：能区分小资金可交易收益和大资金不可交易收益。

## 验证设计草案

| 项目 | 内容 |
| --- | --- |
| 基准策略 | 不加容量约束的涨停事件策略 |
| 实验版本 | 加一字板过滤、涨跌停不可成交、参与率上限、滑点 |
| 样本内 | 待平台恢复后设定 |
| 样本外 | 同涨停事件价量结构研究 |
| 消融方式 | 逐步加入一字板过滤、容量上限、滑点、延迟成交 |
